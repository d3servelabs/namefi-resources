매일 올라오는 합리적인 질문이 있습니다. *"저는 이미 `.eth` 이름(또는 `.crypto`, `.x`)을 갖고 있는데요. 왜 제 `.com`을 [토큰화](/ko/glossary/tokenize/)해야 하나요? 둘이 같은 것 아닌가요?"*

같지 않습니다. 분위기상 약간, 브랜딩 측면에서는 많이 겹치지만, 운영 방식으로 보면 서로 다른 문제를 해결합니다. 이 글에서는 각각이 어디에 맞는지를 정리합니다.

토큰화 도메인에 대해 자세히 알고 싶다면 [토큰화 도메인이란 무엇인가?](/ko/blog/what-are-tokenized-domains/)부터 시작하세요.

---

## 한 문장 정리

- **토큰화 도메인** = 실제 [ICANN](/ko/glossary/icann/) 도메인(`.com`, `.xyz`, `.io` 등)에 [온체인](/ko/glossary/on-chain/) 소유권 토큰을 추가한 것.
- [**Web3**](/ko/glossary/web3/) **도메인** = 오직 온체인에만 존재하는 이름(`.eth`, `.crypto`, `.x` 등). [DNS](/ko/glossary/dns/)와는 별개의 네이밍 시스템입니다.

토큰화 도메인은 기존 DNS 세계를 *확장*합니다. Web3 도메인은 그것을 *대체*하거나(또는 사용 방식에 따라 나란히 존재합니다).

---

## 혼동이 생기는 이유

둘 다 지갑 안에 NFT로 존재합니다. 둘 다 "도메인"이라 불립니다. 둘 다 어딘가에서 ICANN 이야기가 나옵니다 — 그런데 정반대 방향으로요. 두 범주 모두의 마케팅이 이 차이를 흐리게 만드는 경우가 많습니다.

가장 명확한 사고 모델은 다음과 같습니다.

- 일반 브라우저에 주소를 입력했을 때 확장 프로그램, 플러그인, 특수 리졸버 없이도 웹사이트로 연결된다면 — 그것은 **DNS 도메인**입니다. 토큰화해도 이 점은 바뀌지 않습니다.
- 브라우저 확장 프로그램, 특수 [지갑](/ko/glossary/wallet/) 기능, 또는 리졸버 게이트웨이가 있어야 작동한다면 — 그것은 **Web3 도메인**입니다.

둘 다 유효합니다. 하는 일이 다를 뿐입니다.

## 나란히 비교

| 기능 | 토큰화 ICANN 도메인 | Web3 도메인 (ENS, .crypto 등) |
|---|---|---|
| 일반 브라우저에서 해석 | 예, 기본 지원 | 아니요 (리졸버/확장 필요) |
| 기본 이메일 지원 | 예 | 아니요 (다른 메커니즘) |
| SSL/TLS 인증서 지원 | 예 (Let's Encrypt 등) | 아니요 (별도 신뢰 모델) |
| ICANN 인정 여부 | 예 | 아니요 |
| 온체인 존재 | 예 (소유권 레이어) | 예 (전체 정체성) |
| 지갑에서 NFT로 보유 | 예 | 예 |
| 지갑 별칭으로 사용 | 경우에 따라 (플러그인 통해) | 예, 기본 지원 |
| 레지스트라에서 연간 갱신 | 예 (실제 DNS 도메인) | 주로 일회성 또는 다른 모델 |
| 최종 사용자 브라우저 확장 불필요 | 예 | 아니요 |
| DNS 인프라와 호환 | 예 | 직접 호환 안 됨 |

## 각각이 *가장 잘 맞는* 상황

### 토큰화 ICANN 도메인

다음 상황에 적합합니다.

- 실제 웹사이트, 앱, 또는 비즈니스를 운영하면서 Web3 소프트웨어 설치 여부와 관계없이 **모든 사람**이 접근할 수 있기를 원할 때.
- 도메인에서 이메일을 사용하고, 표준 CA의 SSL 인증서를 발급받고, CDN 설정 등을 구성하고 싶을 때.
- 레지스트라 절차 없이 도메인 자체를 **지갑 기반으로 소유하고 이전**하고 싶을 때 — 판매, 선물, 대여 등.
- 도메인을 일반 웹사이트로 운영하면서도 [DeFi](/ko/glossary/defi/)에서 온체인 [담보물](/ko/glossary/collateral/)로 활용하고 싶을 때.

예시: 기업의 `.com`, SaaS 앱의 `.io`, 크리에이터의 `.xyz`, 브랜드의 `.art`. 실제 인터넷에서 작동해야 하는 모든 것.

### Web3 도메인 (ENS, Unstoppable, Freename 등)

다음 상황에 적합합니다.

- **지갑 정체성**이 필요할 때 — 크립토 앱이나 지갑에 입력하면 주소로 해석되는 이름. `0x...` 대신 `vitalik.eth`.
- Web3 네이티브 프로필 / 핸들이 필요하고, 이를 지원하는 dapp에서 활용하고 싶을 때.
- 표준 이메일, 플러그인 없는 브라우저, 또는 SSL에서 작동할 필요가 없을 때.
- 특정 TLD(`.eth`, `.crypto`, `.x`)의 커뮤니티 문화적 측면이 마음에 들 때.

예시: 개인 Web3 정체성, 지갑 프로필, 크립토 수령을 위한 기억하기 쉬운 주소, NFT 쇼케이스 페이지.

## 해석 방식: 각각이 실제로 작동하는 원리

### DNS (토큰화 도메인이 속한 세계)

`example.com`을 입력합니다. 컴퓨터가 [DNS 리졸버](/ko/glossary/dns-resolver/)에 요청합니다. 리졸버가 DNS 계층을 탐색합니다. [IP 주소](/ko/glossary/ip-address/)를 받습니다. 브라우저가 사이트를 가져옵니다. 도메인이 토큰화되었는지 여부와 관계없이 이 모든 과정은 동일하게 작동합니다. 토큰화는 *해석* 레이어가 아닌 *소유권* 레이어를 추가하기 때문입니다.

이 측면의 실제 내용은 [DNS는 여전히 작동합니다](/ko/blog/dns-on-tokenized-domains/)를 참조하세요.

### ENS / Web3 이름 해석

`vitalik.eth`를 입력합니다. Web3 지원 클라이언트(MetaMask, dapp, [ENS](/ko/glossary/ens/) 지원 브라우저)가 [이더리움](/ko/glossary/ethereum/)의 ENS [스마트 컨트랙트](/ko/glossary/smart-contract/)에 쿼리하여 연관된 주소 또는 콘텐츠 해시를 가져와 렌더링합니다. Web3를 지원하지 않는 클라이언트(확장 프로그램 없는 Chrome, 회사 이메일 서버, SSL CA)는 `.eth`의 의미를 모르고 해석하지 못합니다.

이것은 결함이 아니라 설계입니다. ENS와 유사한 시스템은 더 넓은 인터넷의 네이밍 레이어를 대체하기 위한 것이 아니라 Web3 네이티브 경험을 위해 만들어졌습니다. 기반 아키텍처에 대한 자세한 내용은 [공식 ENS 문서](https://docs.ens.domains/)를 참조하세요.

## 많은 사람들이 두 가지를 모두 보유하는 이유

하나를 선택할 이유가 없습니다. 역할이 다릅니다.

일반적인 패턴은 다음과 같습니다.

- **`mybrand.com`** (토큰화) — 실제 제품 / 웹사이트 / 이메일용.
- **`mybrand.eth`** (ENS) — 크립토 수령, Web3 프로필 구축, dapp 내 주소 지정용.

토큰화된 `.com`은 공개 인터넷에서 작동합니다. `.eth`는 지갑 별칭이자 크립토 네이티브 앱 내 정체성으로 작동합니다. 역할이 다르고, 둘 다 유용합니다.

---

## 하나만 선택하는 경우

- **토큰화 도메인만:** 실제 제품을 만들거나, 비즈니스를 운영하거나, 일반 브라우저와 이메일 클라이언트에서 작동해야 하는 모든 경우. 이 경우 `.eth`는 있으면 좋은 부가 요소입니다.
- **Web3 이름만:** 지갑 정체성만 필요하고 실제 웹사이트를 운영하지 않는 경우. (비크립토 용도를 위해 `.com`은 여전히 원할 수 있지만, 반드시 토큰화할 필요는 없습니다.)

## 흔한 오해

- **"ENS가 DNS를 대체할 것이다."** 아닙니다. ENS는 그런 목적으로 설계되지도 않았습니다. ENS는 크립토 정체성에 최적화된 병렬 네이밍 시스템입니다.
- **"토큰화된 `.com`은 'Web3 도메인'이다."** 그것은 *토큰화된 DNS 도메인*입니다. "Web3 도메인"이라는 표현은 보통 `.eth`/`.crypto` 방식의 이름에 사용됩니다. 범주가 다릅니다.
- **"이제 브라우저가 `.eth`를 기본 지원한다."** Brave와 일부 특정 확장 프로그램은 그렇습니다. 주류 브라우저는 아닙니다. 모든 사람에게 작동하는 최종 사용자 경험을 위해서는 DNS가 여전히 정답입니다.
- **"도메인을 토큰화하면 ICANN 인정을 잃는다."** 아닙니다. DNS / ICANN 측면은 그대로 유지됩니다. 온체인 소유권 레이어가 추가될 뿐입니다.
- **"Web3 도메인은 탈중앙화되어 있고, 토큰화 도메인은 그렇지 않다."** 둘 다 탈중앙화된 속성(온체인 소유권)과 중앙화된 속성(레지스트리, ICANN, 스마트 컨트랙트 업그레이드)을 모두 갖고 있습니다. 탈중앙화는 체크박스가 아니라 스펙트럼입니다.

## 요약

- **토큰화 도메인**은 온체인 소유권 토큰이 추가된 실제 ICANN 도메인입니다. 모든 브라우저에서 정상적으로 해석되고, 이메일을 지원하며, SSL과 호환되고, 일반적인 연간 갱신 비용을 냅니다.
- **Web3 도메인**(ENS, Unstoppable Domains, Freename)은 완전히 온체인에 존재하며 지갑 별칭 / Web3 정체성으로 기능하는 별개의 범주입니다.
- 두 범주는 경쟁 관계가 아닙니다. 서로 다른 문제를 해결하며, 많은 사람들이 두 가지를 모두 보유합니다.
- 인터넷 어디서나 이름이 작동해야 한다면 토큰화된 DNS 도메인이 필요합니다. Web3 네이티브 핸들과 주소가 필요하다면 ENS 방식의 이름이 필요합니다.
- 같은 지갑에서 둘 다 보유할 수 있습니다.

토큰화 분야의 플랫폼에 대해서는 [도메인 토큰화 플랫폼 선택하기](/ko/blog/choosing-a-domain-tokenization-platform/)를 참조하세요.
