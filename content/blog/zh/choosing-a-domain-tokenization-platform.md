## 快速概览

大致来说，这些平台可以分为三大阵营：

1. **面向所有者的代币化服务。** 你可以带着一个真实的 ICANN 域名（或注册一个新的），平台会将其代币化。例如：**Namefi、3DNS**。
2. **协议层与面向注册局的基础设施。** 平台构建标准、智能合约或与注册局建立合作伙伴关系，供其他平台（和注册商）在之上进行开发构建。例如：**Doma Protocol、D3 Global Inc**。
3. **专门的销售与流动性工具。** 拍卖、碎片化（分片）、借贷——这些工具建立在代币化域名之上，而不是直接生成它们。例如：**GBM**（拍卖）、**Doma Prime**（流动性）、**Domora**（主打碎片化）。

这些类别之间存在重叠，部分平台横跨多个阵营。但是，“这个平台属于哪个阵营？”是你首先应该思考的问题。

## 各平台的最佳优势

### Namefi

**最适合：** 希望将真实的 `.com`/`.xyz`/`.io` 域名在 Ethereum 或 Base 网络上[代币化](/en/glossary/tokenize/)的所有者，并期望获得广泛的 NFT 市场与 [DeFi](/en/glossary/defi/) 借贷支持，同时要求 DNS 管理体验不能比 Cloudflare 差。

**亮点：** 支持众多 [顶级域名 (TLD)](/en/glossary/tld/) 且受 [ICANN](/en/glossary/icann/) 认可的域名；通过标准 [NFT](/en/glossary/nft/)（[ERC-721](/en/glossary/erc-721/)）实现链上所有权，确保钱包、交易市场和链上工具“开箱即用”；包含 [DNSSEC](/en/glossary/dnssec/) 的全面 DNS 管理；内置应用内[交易市场](/en/glossary/marketplace/)；集成链上支付（[x402](/en/glossary/x402/)）。支持多链。从第一天起就支持自我托管（Self-custody）。

**不太适合：** 想要申请自己尚不拥有的全新 TLD 的用户，或是只想拥有纯 Web3 原生名称（如 `name.eth`）的用户。

### Doma Protocol

**最适合：** 开发者和协议层级的工作。Doma Protocol 是一个协议层——它为代币化 DNS 域名提供共享标准和底层原语，包括流动性原语（Doma Prime）以及全新代币化域名的发行平台（Mizu）。

**亮点：** 高度关注开发者体验；与多家域名注册商建立了合作关系；基于该协议构建的应用程序生态系统正在不断壮大。

**不太适合：** 只是想将手中现有域名代币化，而不需要协议技术栈的普通所有者。你通常不会直接使用 Doma Protocol，而是通过建立在它之上的平台与其进行间接交互。

### D3 Global Inc

**最适合：** 注册局层面的合作关系，以及专为 Web3 场景设计的全新 TLD。

**亮点：** D3 Global Inc 的定位围绕全新的 TLD 和注册局协议展开，重点强调 Web2 DNS 与 Web3 命名系统之间的互操作性。

**不太适合：** 如果你的目标是“将我已经拥有的 `.com` 代币化”，那么 D3 Global Inc 并不是最直接的路径——他们更关注注册局/TLD 层面，而非个人所有者的操作流程。

### 3DNS

**最适合：** 想要简化代币化流程且不需要支持所有 TLD 的 Web3 原生所有者。

**亮点：** 受 ICANN 认可的域名代币化，提供简洁且极具主见的用户体验。拥有活跃的生态系统合作伙伴。

**不太适合：** 需要广泛的 TLD 覆盖，或需要 Namefi/Doma 提供的特定集成的所有者。在做出决定前，值得对比一下其支持的 TLD 范围。

### Domora

**最适合：** 碎片化（分片）应用场景——即优质域名的共同所有权（co-ownership）。

**亮点：** 明确以“将域名打造成碎片化资产”为理念驱动，而不仅仅是作为一个可交易的 NFT。

**不太适合：** 直截了当的“我想代币化并使用我的域名”的流程。Domora 更侧重于特定的市场结构，而不是一个通用平台。

### WebUnited

**最适合：** 主要对在 Web2 和 Web3 之间建立 DNS 镜像桥接感兴趣的所有者。

**亮点：** 传统 DNS 与链上命名系统之间的镜像式集成模式。

**不太适合：** 如果你需要完整的交易市场、借贷技术栈，或是广泛的 NFT 市场兼容性。

### GBM (GBM Auctions)

**最适合：** 通过“竞价赚取（bid to earn）”的拍卖机制出售代币化域名。

**亮点：** 它是拍卖基础设施，而非代币化平台本身。通常与上述某个平台结合使用，专门处理销售环节。

**不太适合：** 除销售环节以外的任何其他操作。

## 对比清单（实用检查表）

当你在不同平台之间做决定时，营销页面无法告诉你哪个最合适。但以下问题可以帮到你：

- **TLD 覆盖范围。** 平台是否支持你关注的特定顶级域名（如 `.com`、`.io`、`.xyz`、`.art`，或 `.de`/`.uk` 等 ccTLDs）？
- **支持的区块链。** Ethereum 主网、Base、Polygon 还是其他？NFT 存在于哪条链上？在哪里支付 Gas 费？
- **NFT 标准。** 是否为标准的 ERC-721？这对各大交易市场（如 OpenSea、Blur、Magic Eden 等）的兼容性以及链上借贷至关重要。
- **DNS 管理。** 能否在平台内管理 DNS？是否可以使用外部名称服务器（如 Cloudflare、Route53）？是否支持 DNSSEC？
- **借贷 / 抵押。** 你是可以将代币化域名作为抵押品在现有的货币市场上借贷，还是只能在平台特定的封闭生态内进行？
- **市场兼容性。** Blur、OpenSea 等平台会显示你的上架信息，还是只在该平台自己的交易市场中可见？
- **托管模式。** 是自我托管（你的钱包，你的私钥，自己负责），还是平台托管？两者各有取舍。
- **续费流程。** 谁向注册商付款？年度续费如何计费？如果你停止支付会发生什么？
- **退出路径。** 如果某天你想“解除代币化（de-tokenize）”并退回到普通的域名注册商设置，是否可行？
- **费用。** 铸造费、市场交易费、转让费、续费、Gas 费。在做出承诺之前，请根据你的具体场景将它们加总计算。

如果一个平台的文档中没有把这些讲清楚，这本身就是一个信号。

## 关于“最佳”的一点看法

世界上没有放之四海而皆准的“最佳”平台。正确的选择取决于：

- 你关注哪些 TLD。
- 你是所有者、开发者还是交易市场。
- 你有多看重自我托管。
- 你是希望将域名作为 DeFi 抵押品，还是仅仅想持有它。
- 你对单一平台的信任程度。

我们显然认为对许多所有者来说，Namefi 是正确的答案。但你能做的最好决定是：**除了我们，至少再去尝试一家其他平台**。如果我们更好，在横向对比后你自然会知道。如果其他平台更符合你的使用场景，那你完全应该选择它。

## 总结

- 域名代币化平台分为三大类：面向所有者的服务、协议层以及专门的销售/流动性工具。
- 对于想要将手头现有域名代币化的所有者来说，**Namefi** 和 **3DNS** 是最直接的选择。
- **Doma Protocol** 是一个协议层；你通常会与它进行间接交互。
- **D3 Global Inc** 以注册局为中心，主要围绕新的 TLD 展开。
- **Domora**、**WebUnited** 和 **GBM** 分别专注于域名碎片化、DNS 桥接以及拍卖。
- 做出正确选择取决于你所需的 TLD、托管偏好、市场兼容性，以及你打算对代币化域名*做些什么*。
- 在将你的域名资产组合完全投入之前，请尝试多个平台。

要想专门了解 Namefi 的更多信息，请访问 [namefi.io](https://namefi.io)。要了解更广泛的类别概念，请阅读[什么是代币化域名？](/en/blog/what-are-tokenized-domains/)。
