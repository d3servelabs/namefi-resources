您可能听说过“代币化域名（tokenized domain）”、“将域名代币化（tokenize a domain）”或“域名代币化（domain tokenization）”等短语，并好奇它们的真正含义。代币化域名是一种新型域名吗？是一种纯区块链名称吗？是 `.com` 的替代品吗？首先，“代币化”一个域名到底意味着什么？

本文将直接回答**“是什么”**的问题：什么是代币化域名，域名代币化意味着什么，代币化域名*不是*什么，以及这整个概念与您已经熟知的域名有什么联系。

> 如果您想了解*为什么*域名代币化很重要，请参阅 [为什么要将域名在链上代币化？](/en/blog/why-tokenize-domains/)。本文主要关注*是什么*。

## 简明定义

**代币化域名**是一个常规的、受 [ICANN](/en/glossary/icann/) 认可的[域名](/en/blog/what-is-domain/)（例如 `mybrand.xyz` 或 `example.com`），其所有权同时作为**区块链上的代币**（通常是 [NFT](/en/glossary/nft/)）来呈现。创建这种基于代币的表现形式的过程被称为**域名代币化**，而这个行为本身就是人们常说的*将域名代币化*。

换句话说：

> 代币化域名是具有**两个同步所有权层**的同一个域名：传统的 [DNS](/en/glossary/dns/) 注册局记录，*以及*镜像该记录的链上代币。**将域名代币化**就是将这第二个链上层添加到现有或新注册的域名中。

当您转移代币时，底层的域名也会随之转移。当域名过期或续费时，代币也会反映该状态。

## 两层结构，一个域名

将代币化域名想象成具有两条同步记录会有助于理解：

| 层级             | 是什么                                              | 由谁维护                                  |
|------------------|-----------------------------------------------------|-------------------------------------------|
| DNS / 注册局     | 位于[注册商](/en/glossary/registrar/)和注册局的官方记录 | 受 [ICANN](/en/glossary/icann/) 认证的[注册商](/en/glossary/registrar/) |
| 链上代币         | 位于您的[钱包](/en/glossary/wallet/)中代表所有权的 [NFT](/en/glossary/nft/) | 公有链上的[智能合约](/en/glossary/smart-contract/) |

这两层由域名代币化平台（在 Namefi 的案例中，由 Namefi 协议及其注册商集成）保持同步。每当我们谈论*将域名代币化*或*域名代币化*时，我们指的都是为特定域名建立和维护这种双层关系。

这有别于*仅仅*在注册商处拥有域名（传统模式），也有别于*仅仅*在链上拥有名称（ENS 风格的模式）。代币化域名是有意将两者结合在了一起。

## 代币化域名*不是*什么

关于域名代币化，有几个常见的误解值得澄清：

### 不是新的顶级域名（TLD）

代币化域名不是 `.crypto`、`.eth` 或 `.x` 风格的名称。当您通过 Namefi 将域名代币化时，您使用的是您已经熟悉的顶级域名（如 `.com`、`.xyz`、`.io`、`.art` 等），这些域名可以在全球任何浏览器、电子邮件客户端或 DNS 解析器中解析。

### 不同于 ENS 或“区块链名称”

[ENS](/en/glossary/ens/) 名称（如 `vitalik.eth`）完全存在于链上，如果没有桥接器或特殊的解析器，它们无法在标准 DNS 中解析。相比之下，代币化域名是**真正的 DNS 域名**，只是*同时*具有链上表现形式。域名代币化是在真实的 DNS 名称上添加了链上层；它并不是用平行的命名系统取代 DNS。

| 功能特性                         | 传统域名           | ENS / 区块链名称       | 代币化域名       |
|----------------------------------|--------------------|------------------------|------------------|
| 可在任何浏览器中工作             | 是                 | 需要解析器             | 是               |
| 受 ICANN 认可                    | 是                 | 否                     | 是               |
| 保存在您的钱包中                 | 否                 | 是                     | 是               |
| 可以在链上转移                   | 否                 | 是                     | 是               |
| 可与智能合约组合                 | 否                 | 是                     | 是               |

### 不是抗审查的或“法外之地”

由于底层资产是真正的 DNS 域名，代币化域名仍需遵守续费、[ICANN](/en/glossary/icann/) 政策、[UDRP](/en/glossary/udrp/) 争议以及适用法律。代币反映了所有权；它并不能使域名豁免于现实世界的规则。

## 将域名代币化在实践中是如何运作的

当您在 Namefi 上将域名代币化（或注册一个全新的代币化域名）时，实际发生的过程如下：

1. **注册** — 通过受认证的[注册商](/en/glossary/registrar/)注册（或转入）一个真正的 DNS 域名。
2. **铸造** — 作为域名代币化的一部分，代表该域名的 [NFT](/en/glossary/nft/) 会被铸造到您的[钱包](/en/glossary/wallet/)中。
3. **同步** — 平台使每个代币化域名的 DNS 级所有权与链上所有权保持一致。如果您转移了 NFT，DNS 记录也会随之更改。
4. **使用** — 您可以将代币化域名指向某个网站、设置 DNS 记录，或在链上应用程序（市场、身份、[DeFi](/en/glossary/defi/) 等）中使用该 NFT。

最终用户的体验是：*一个域名，两种交互方式*——熟悉的 DNS 世界，以及由域名代币化解锁的可编程链上世界。

## 您可以用代币化域名做什么

由于两层同时存在，您可以获得各项功能的结合：

- **作为常规域名使用** — 托管网站、设置电子邮件、配置 DNS 记录。
- **保存在您自己的钱包中** — 拥有所有权无需托管账户。
- **秒级转移** — 将 NFT 发送至另一个钱包；DNS 记录会随之更改。
- **在 NFT 市场上挂售** — OpenSea、Blur 等。
- **在智能合约中使用** — 抵押、[拍卖](/en/glossary/auction/)、[租赁](/en/glossary/leasing/)、[碎片化所有权](/en/glossary/fractional-ownership/)等。
- **与链上身份绑定** — 链接到 [Farcaster](/en/glossary/farcaster/)、[Lens](/en/glossary/lens/) 或 [DID](/en/glossary/did/) 系统。

## 兄弟类别：ENS、Unstoppable Domains、Freename 和 Web3 域名

代币化域名的一个近亲是 **Web3 域名**家族——这一类别由 ENS、Unstoppable Domains 和 Freename 等优秀项目开创。我们希望明确这一区别，这并不是为了贬低它们的工作（它们为链上命名和身份做出了巨大贡献），而是为了帮助读者根据自己的目标选择合适的工具。

Web3 域名的设计经过了深思熟虑，与代币化的 ICANN 域名有所不同。您可以这样理解它们：

- **在设计上属于不同的命名空间。** Web3 域名（`.eth`、`.crypto`、`.x`、`.nft` 以及用户创建的 TLD）有意独立于 [ICANN](/en/glossary/icann/) 根区之外，这使得它们能够快速迭代并尝试新的命名模式。其代价是它们与传统的 DNS 层次结构是平行的，而不是位于其内部。
- **浏览器和电子邮件解析需要额外步骤。** 在普通浏览器中访问 Web3 域名或向其发送电子邮件，通常需要解析器、扩展程序或桥接器。支持它们的钱包、dApp 和加密原生浏览器的生态系统正在稳步增长——但与标准浏览器、邮件服务器、CDN、SEO 工具和 SSL/TLS 证书颁发机构的同等支持仍在进展中。
- **真正新颖的钱包原生用例。** 这是 Web3 域名大放异彩的地方：取代冗长的 `0x…` 地址为人类可读的名称、简化代币转移、为 dApp 登录提供支持，并作为链上身份原语。这些模式中的许多在 ENS 及其同行出现之前根本不存在，而代币化域名正是建立在这些理念之上的。
- **采用情况不同于真实的 DNS / ICANN 域名。** 真实域名（也称为 *DNS 域名*、*ICANN 域名*或*真实域名*——例如 `.com`、`.org`、`.xyz`、`.io`）受益于数十年来的普遍支持，横跨各类浏览器、电子邮件提供商、CDN 和证书颁发机构。Web3 域名在加密原生生态系统中拥有令人瞩目且不断增长的覆盖率，而更广泛的互联网普及仍在迎头赶上。

领先的 Web3 域名平台，感谢它们各自的贡献：

- [ENS](https://ens.domains) — 基础的以太坊原生命名系统 (`.eth`)，也是 Web3 中最重要的原语之一。ENS 还通过 [DNSSEC](/en/glossary/dnssec/) 提供了通向真实 DNS 名称的深思熟虑的桥梁。
- [Unstoppable Domains](https://unstoppabledomains.com) — 早期且具有影响力的区块链原生名称（如 `.crypto`、`.x` 和 `.nft`）先驱，具有广泛的钱包和 dApp 集成。
- [Freename](https://freename.io) — 一种用于用户创建的 Web3 TLD 和命名空间的创新方法。

如果您的主要目标是**链上身份**或 **Web3 命名**，这些平台都非常出色且值得探索。如果您的主要目标是一个**同样能**在任何浏览器、任何电子邮件客户端、任何 CDN 和任何 SSL 证书颁发机构中工作的名称——即一个您可以额外在钱包中持有和编程的真实 ICANN 域名——那么上面的代币化域名平台（Namefi、D3 Global Inc、Doma Protocol、Domora、WebUnited、GBM）就是为该用例设计的。这两个类别可以很好地共存，且许多用户同时持有两者。

## 一个简单的思维模型

如果传统域名是**由第三方代您持有的契约**，那么代币化域名就是**相同的契约，并在您的口袋中有一份密码学副本**——而且两者保持同步联动。

您不会失去法律/注册局层。您只是在其之上获得了一个可编程层。

## 总结

- **代币化域名**是拥有链上代币（通常是 NFT）作为其所有权镜像的真实 DNS 域名。
- **域名代币化**（也称为*域名名称代币化*）是创建和维护这种链上表现形式的过程。
- **将域名代币化**（或批量*代币化域名*）是在不放弃传统 DNS 层的前提下，将这种钱包原生的所有权层添加到真实的 ICANN 域名上。
- 代币化域名**不是**新的 TLD，不是 ENS 风格的名称，也不是绕过 DNS 或法律的途径。
- 它为您提供了传统域名的所有功能，**外加**钱包原生所有权和与链上应用程序的可组合性。
