这是一个合情合理且每天都会被问到的问题：*“我已经拥有一个 `.eth` 域名（或 `.crypto`、`.x`）了。为什么我还要把我的 `.com` [代币化](/en/glossary/tokenize/)？它们难道不是一回事吗？”*

它们并不一样。虽然两者在氛围上有些重叠，在品牌建设上也有很多相似之处，但在实际操作中，它们解决的是截然不同的问题。本文将详细为您梳理它们各自的适用场景。

如果您想专门深入了解代币化域名的详细内容，可以从[什么是代币化域名？](/en/blog/what-are-tokenized-domains/)开始阅读。

---

## 一句话总结

- **代币化域名 (Tokenized domain)** = 一个真正的 [ICANN](/en/glossary/icann/) 域名（如 `.com`、`.xyz`、`.io` 等），并在其之上添加了[链上](/en/glossary/on-chain/)所有权代币。
- [**Web3**](/en/glossary/web3/) **域名 (Web3 domain)** = **仅**存在于链上的名称（如 `.eth`、`.crypto`、`.x` 等）。它是一个独立的命名系统，并不属于 [DNS](/en/glossary/dns/) 的一部分。

代币化域名是对现有 DNS 世界的**延伸**。而 Web3 域名则是对其的**替代**（或者说与它并存，取决于您的使用方式）。

---

## 混淆源自何处

两者都涉及存放在钱包中的 NFT，都被称为“域名”，并且在讨论中都会涉及 ICANN——但方式却截然相反。这两个领域的市场营销往往会模糊它们之间的界限。

这里提供一个最清晰的心智模型：

- 如果您在普通浏览器中输入该名称，无需任何扩展程序、插件或特殊的解析器就能直接解析到一个网站——这就是一个 **DNS 域名**。将其代币化不会改变这一点。
- 如果您需要浏览器扩展程序、特殊的钱包功能或解析网关才能让它工作——这就是一个 **Web3 域名**。

两者都是有价值的，只是用途不同。

## 横向对比

| 特性 | 代币化 ICANN 域名 | Web3 域名 (ENS, .crypto 等) |
|---|---|---|
| 在任何浏览器中解析 | 是，原生支持 | 否（需解析器/扩展程序） |
| 开箱即用的电子邮件支持 | 是 | 否（机制不同） |
| 支持 SSL/TLS 证书 | 是 (Let's Encrypt 等) | 否（独立的信任模型） |
| 获 ICANN 认可 | 是 | 否 |
| 存在于链上 | 是（所有权层） | 是（整个身份） |
| 作为 NFT 存放在钱包中 | 是 | 是 |
| 用作钱包别名 | 有时（通过插件） | 是，原生支持 |
| 在注册商处按年续费 | 是（真正的 DNS 域名） | 通常为一次性付费或采用其他模型 |
| 终端用户无需浏览器扩展 | 是 | 否 |
| 兼容 DNS 基础设施 | 是 | 否（非直接兼容） |

## 各自的优势所在

### 代币化 ICANN 域名

最佳适用场景：

- 您正在运营一个真正的网站、应用程序或业务，并希望它能为**所有人**服务，无论他们是否安装了任何 Web3 软件。
- 您希望在您的域名上使用电子邮件、获取标准 CA 的 SSL 证书、配置 CDN 等。
- 您希望域名本身具备**钱包原生的所有权和可转让性**——能够进行出售、赠予、借贷——而无需受限于注册商的繁琐流程。
- 您希望该域名能在 DeFi 中用作链上抵押品，同时仍能作为一个正常网站运行。

示例：一家公司的 `.com`，一个 SaaS 应用的 `.io`，一位创作者的 `.xyz`，一个品牌的 `.art`。任何需要在现实互联网中发挥作用的场景。

### Web3 域名 (ENS, Unstoppable, Freename 等)

最佳适用场景：

- 您需要一个**钱包身份**——在加密应用或钱包中输入该名称时，它会解析为您的地址。例如使用 `vitalik.eth` 代替 `0x...`。
- 您希望在支持它的 DApp 中拥有一个 Web3 原生的个人资料或句柄（handle）。
- 您不需要该名称在标准电子邮件、无插件的普通浏览器或 SSL 环境中工作。
- 您喜欢某个特定顶级域（如 `.eth`、`.crypto`、`.x`）所附带的文化和社区属性。

示例：您的个人 Web3 身份，钱包上的个人资料展示，用于接收加密货币的易记地址，NFT 展示页面。

## 解析：它们实际是如何工作的

### DNS（代币化域名所在的世界）

当您输入 `example.com` 时。您的计算机会向 DNS 解析器发出请求。解析器会遍历 DNS 层级结构，最终返回一个 IP 地址，然后浏览器获取该网站的内容。无论域名是否被代币化，这一切的工作原理都是相同的，因为代币化添加的是**所有权**层，而不是**解析**层。

了解关于这方面的实际细节，请参阅 [DNS 依然有效](/en/blog/dns-on-tokenized-domains/)。

### ENS / Web3 域名解析

当您输入 `vitalik.eth` 时。一个支持 Web3 的客户端（如 MetaMask、某个 DApp、或支持 [ENS](/en/glossary/ens/) 的特定浏览器）会在以太坊上查询 ENS [智能合约](/en/glossary/smart-contract/)，获取关联的地址或内容哈希，并据此进行渲染。而一个不支持 Web3 的客户端（如没有安装扩展程序的 Chrome、您办公室的邮件服务器、您的 SSL CA 机构）并不知道 `.eth` 是什么意思，因此也无法解析它。

这并不是一个缺陷——而是系统设计使然。ENS 及类似系统是为 Web3 原生体验而构建的，并非为了取代更广泛的互联网命名层。有关底层架构，请参阅 [ENS 官方文档](https://docs.ens.domains/)。

## 为什么许多人选择同时持有两者

没有理由非得二选一。它们扮演着不同的角色。

一种常见的模式是：

- **`mybrand.com`**（代币化）：用于实际的产品 / 网站 / 电子邮件。
- **`mybrand.eth`**（ENS）：用于接收加密货币、建立 Web3 个人资料，以及在 DApp 中作为地址使用。

代币化的 `.com` 服务于开放式互联网。`.eth` 则作为钱包别名和加密原生应用中的身份证明。各司其职，相辅相成。

---

## 什么时候您只需要选择其中之一

- **只选代币化域名：** 如果您正在构建一个真正的产品、运营一项业务，或者进行任何需要在普通浏览器和电子邮件客户端中正常工作的事情。在这种情况下，`.eth` 只是一个锦上添花的加分项。
- **只选 Web3 域名：** 如果您只需要一个钱包身份，并且并不打算运营一个实际的网站。（对于非加密领域的事务，您可能依然想要一个 `.com` 域名，但并非必须将其代币化。）

## 常见误区

- **“ENS 将取代 DNS。”** 不，它也没有试图这样做。ENS 是一个平行且独立运作的命名系统，专门针对加密身份进行了优化。
- **“代币化的 `.com` 就是‘Web3 域名’。”** 它是一个*代币化 DNS 域名*。“Web3 域名”这一标签通常用于描述 `.eth`/`.crypto` 风格的名称。这两者的分类是不同的。
- **“浏览器现在原生支持 `.eth` 了。”** 对于 Brave 浏览器和一些特定的扩展程序来说，是的；但对于主流浏览器，并非如此。如果想要一种适用于所有人的终端用户体验，DNS 依然是确切的答案。
- **“如果我把域名代币化，就会失去 ICANN 的认可。”** 否。DNS / ICANN 方面没有任何改变。您只是添加了一个链上所有权层而已。
- **“Web3 域名是去中心化的，代币化域名则不是。”** 两者都具有某些去中心化的属性（如链上所有权），同时也都包含某些中心化的属性（如注册局、ICANN、智能合约升级）。去中心化是一个光谱，而不是一个非黑即白的复选框。

## 总结

- **代币化域名**是添加了链上所有权代币的真正 ICANN 域名。它们在每一个浏览器中都能正常解析，支持电子邮件，兼容 SSL，并需要正常缴纳年费续期。
- **Web3 域名** (ENS、Unstoppable Domains、Freename 等) 则属于完全不同的类别——它们完全存在于链上，用作钱包别名 / Web3 身份。
- 这两类域名并非竞争对手。它们解决的是不同的问题，所以许多人会同时持有两者。
- 如果您需要该名称在互联网的各个角落都能正常使用，您需要的是代币化 DNS 域名。如果您想要一个 Web3 原生的句柄和地址，您需要的是 ENS 风格的域名。
- 同一个钱包可以同时持有这两种域名。
