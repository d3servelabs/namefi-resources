拥有域名是容易的部分。出售域名才是真正挑战的开始。

基本交易听起来很简单：买方付款，卖方转移域名，完成。但在实际操作中，大多数域名销售往往在更早的阶段就停滞不前。卖方不知道域名值多少钱，合适的买家根本没看到它，推销邮件听起来像垃圾邮件，或者交割步骤模糊不清导致双方都感到紧张。

本指南专为那些手里有不再需要的域名的普通所有者，或是试图将一个好域名转化为实际销售的域名投资人而写。这并非承诺每个域名都有买家——许多域名其实并没有。它是一份实用的检查清单，旨在为您手中真正的域名提供一个真正的出售机会。

## 简要概述

如果您只记住一个流程，请记住以下几点：

1. 评估该域名是否有可靠的潜在买家。
2. 根据可比销售记录进行定价，而不是凭空期望。
3. 在推销之前，检查法律、转移、过期和 DNS 风险。
4. 选择您的销售路径：入站（Inbound）、出站（Outbound），或两者结合。
5. 使用 [Namefi Outbound](https://namefi.io/outbound) 研究潜在买家、买家角度、可用的公开联系方式，以及可编辑的初始信息草稿。
6. 只向可能真正感兴趣的人发送简短、具体的推销信息。
7. 以书面形式进行谈判：价格、费用分摊、转移方式、包含的资产以及时间表。
8. 通过安全的支付或担保（Escrow）工作流完成交易，转移域名，然后妥善保存记录。

转移是最后一步，买家匹配度才是第一步。

## 1. 认清您实际拥有的域名

首先，用通俗的语言描述这个域名。不要用“精品（Premium）”或“稀有（Rare）”等词汇。它到底是什么？

- 一个简短的、适合做品牌的名称。
- 一个单拼/单词或双拼/双词的类别名称。
- 城市名加上服务名。
- 与某个旧项目关联的域名。
- 一个拥有直接输入流量、反向链接或收入的名称。
- 一个您觉得有潜力而注册的投机性名称。

每种类型的销售方式都不同。城市服务类域名可能只适合少数能用得上的本地企业。简短的品牌名称可能适合初创公司、基金、应用程序或社区。有真实流量的域名可能会吸引买家，因为它已经具备了实际需求。而一个随机、未使用的域名可能根本没有市场。

诚实的测试标准是：**您能否不假思索地列出 10 到 30 个合理的潜在买家？**

如果可以，这个域名就值得您主动去出售。如果不能，它可能有一天会通过买家主动找上门（入站）售出，但主动推销（出站）可能会让您感到沮丧。一个理论上很巧妙的域名，与一个企业实际能用的域名是两回事。

还要尽早检查一些可能令人不快的问题：

- 该域名是否与他人的[商标](/zh-CN/glossary/trademark/)存在令人混淆的相似性？
- 您注册它主要是因为某个特定公司的品牌吗？
- 当前的网站、停放页面或广告内容是否会造成混淆？
- 域名是否即将过期？
- 它是否因为最近刚注册或转移而处于禁止转移锁定状态？

商标风险不容小觑。根据 [ICANN 的 UDRP（统一域名争议解决政策）](https://www.icann.org/resources/pages/policy-2024-02-21-en)，投诉人必须证明域名存在令人混淆的相似性、域名持有人不享有权利或合法利益，以及存在[恶意](/zh-CN/glossary/bad-faith/)注册和使用。ICANN 列出的一个恶意证据是：获取域名的主要目的是为了将其出售给商标所有者或其竞争对手，且售价高于有记录的自掏腰包的成本。如果这听起来与您的计划非常接近，请在推销任何人之前暂停，并咨询律师。

这并不是让您恐慌的理由，而是提醒您通过合规方式出售无争议域名的原因。

## 2. 根据市场定价，而不是凭感觉

域名估值是很复杂的，因为域名不是商品。对于“五字母的 `.com`”或“AI 关键词”并没有一个通用的标价。同一个域名对某个买家可能毫无价值，但对另一个买家却具有战略意义。

不过，一个站得住脚的价格通常来自五个因素：

| 考量因素 | 检查内容 |
| --- | --- |
| 可比销售记录 | 长度、字数、类别和[顶级域名](/zh-CN/glossary/tld/)（TLD）相似的近期销售记录 |
| TLD 强度 | 该后缀是全球流通性好，还是主要在特定垂直领域有用 |
| 买家使用场景 | 该名称在多大程度上直接助力实际业务、活动、产品或类别 |
| 现有价值 | 流量、反向链接、收入、域名年龄、干净的历史记录或已开发的资产 |
| 您的限制条件 | 续费成本、紧迫性、最低可接受价格和您的耐心 |

自动估值工具可以作为合理性检查的参考。但它们不是最终价格。它们不知道买家的紧迫程度、您的时间表、买家的预算，或者该名称是否解决了真实的商业问题。

在上架或推销之前，写下三个数字：

- **底价（Floor）：** 您毫无遗憾地能接受的最低价格。
- **目标价（Target）：** 如果买家非常匹配，您认为合理的价格。
- **期望价（Stretch）：** 如果这是一个对资金充足的买家具有战略意义的域名，您会索要的价格。

然后选择定价方式：

- **固定价格 / 立即购买（Buy Now）：** 适合您追求速度并希望减少反复沟通的情况。
- **提供报价（Make offer）：** 适合域名不同寻常或买家价值差异很大的情况。
- **[拍卖](/zh-CN/glossary/auction/)（Auction）：** 适合有多个买家展现出明显需求的情况。
- **私下谈判（Private negotiation）：** 适合买家群体较小且具有战略眼光的情况。

域名的价值越高，在收到第一封回复之前了解您的底线数字就越重要。如果您不需要在谈判中间现编价格，谈判会变得从容得多。

## 3. 在与买家交谈前准备好域名

在外发推销之前完成这部分工作，这能省去后来发尴尬邮件的麻烦。

### 确认控制权

确保您可以登录[注册商](/zh-CN/glossary/registrar/)账户，访问[注册人](/zh-CN/glossary/registrant/)/管理员邮箱，并证明您控制该域名。如果该域名属于某家公司，请确认谁有权出售它。

### 检查过期时间

如果域名即将过期，请在认真推销之前续费。买家肯定不愿意听到他们即将购买的域名下周可能就会被删除。

### 检查转移状态

[ICANN](/zh-CN/glossary/icann/) 解释说，注册商在某些情况下可能会拒绝转移，包括当域名处于锁定状态、初次注册后 60 天内或上次转移后 60 天内。对于许多跨注册商的转移，您还需要一个 [AuthInfo/转移密码（auth code）](/zh-CN/glossary/auth-code/)，ICANN 规定注册商必须允许您自行生成密码，或在您提出请求后的五个日历日内提供该密码。

这并不意味着每笔交易都必须等待。如果买家愿意在当前的注册商处接收域名，那么进行同注册商账户推送（Push）或所有权变更可能是可行的。但您在承诺交割日期之前，必须了解清晰的转移路径。

### 决定包含哪些内容

交易仅包含域名吗？还是说您也在出售网站、内容、邮箱账户、社交媒体账号、代码、商标、客户列表或分析历史数据？

请把这些写下来。“域名”与“围绕域名的业务”并不是一回事。

### 保持 DNS 稳定不出错

如果该域名支撑着一个实时运转的网站或电子邮件系统，请小心处理 [DNS](/zh-CN/glossary/dns/) 更改。注册商转移并不一定意味着 DNS 必须更改，但如果没有人规划好交接，注册商托管的 DNS 可能会中断。对于有价值的运营中域名，请确认 DNS 托管位置、哪些记录是关键的，以及交割后由谁来管理连续性。

## 4. 选择入站、出站或两者结合

出售域名有两大类方式。

**入站（Inbound）** 意味着您让域名易于被发现并等待买家。这可能意味着建立一个简单的“出售中”落地页、发布固定价格的列表，或设置一个联系表单。当域名足够优质，买家很可能会搜索它或直接输入它时，入站策略最为有效。

**出站（Outbound）** 意味着您主动研究可能的买家并与他们联系。当域名明显适合特定的买家群体时，出站策略最为有效：比如本地服务类别、产品关键词、公司现有域名的简短版本，或是与可见的扩张计划相匹配的名称。

大多数卖家应该双管齐下：

- 在该域名上设置一个简单、可信的落地页。
- 保持价格或询价路径清晰可见。
- 研究出一份可能的潜在买家短名单。
- 仅联系匹配度确实很高的潜在客户。

让域名推销看起来像垃圾邮件的最快方法，就是把“他们的网站上某处有这个关键词”视为足够匹配的理由。这远远不够。买家匹配度才是真正的硬通货。

[Namefi Outbound](https://namefi.io/outbound) 专为整个流程中的买家研究环节而设计。

从您拥有或代理的域名开始。Outbound 帮助您探索：

- 可能适合该名称的买家类型和使用场景。
- 匹配的潜在公司。
- 供您参考的买家切入角度。
- 可用的公开联系方式。
- 在有足够上下文情况下的可编辑推销草稿。
- 可导出用于您自身流程的研究笔记。

将其作为研究工具，而不是自动驾驶系统。优秀的推销流程仍然需要人类的判断。

在审查潜在客户时，问问自己：

- 这家公司是否已经在该类别中运营？
- 这个域名是否明显比他们现在使用的更短、更简洁或更易记？
- 是否有产品发布、融资、品牌重塑、地域扩张或活动，使得这个时机变得合理？
- 您能用一句话解释这种匹配关系且不显得牵强吗？
- 是否存在任何商标或品牌混淆的风险？

如果答案很勉强，请隐藏该潜在客户并继续看下一个。我们的目标不是做出一份庞大的电子表格，而是一份真实可靠的候选短名单。

## 6. 编写不像域名垃圾邮件的推销信

一封优秀的初次邮件应该简短、具体，且能让对方得体地忽略。它不应该向买家说教域名的理论知识；不应包含附件；不应假装“有多方表示兴趣”（除非这是真的）。

最简单的模板模式：

```text
主题：example.com

你好 [姓名]，

我是 example.com 的所有者，我注意到 [公司] 正在围绕 [特定产品、市场或使用场景] 进行开发。

如果您希望为 [明确的买家收益] 寻找一个更简短的名称，这个域名可能会很有用。您觉得值得简单聊聊吗？

谢谢，
[您的姓名]
```

作为首次接触，这已经足够了。如果该域名是相关的，买家会理解的。如果您需要 400 个字来证明这个名字的价值，要么是买家找错了，要么是这个域名比您想象的要弱。

优秀的出站推销规则：

- 尽可能发给真正的决策者。
- 使用一个该域名匹配的具体理由。
- 邮件保持纯文本格式。
- 让请求变得简单。
- 包含真实的签名和清晰的退订（Opt-out）方式。
- 在进行了少数几次有礼貌的跟进后即可停止。

如果您在发送商业邮件，请遵守适用于您的法规。在美国，[FTC 的 CAN-SPAM 指南](https://www.ftc.gov/business-guidance/resources/can-spam-act-compliance-guide-business)涵括了准确的发件人信息、非欺骗性的主题行、有效的邮寄地址以及清晰的退订方式等要求。其他国家的规定可能更加严格。“我只是卖一个域名”并不是一种合规策略。

## 7. 稳妥谈判，把握节奏

当买家回复时，请放慢节奏。回复代表了兴趣，而不是交易完成。

在深入讨论价格之前，请弄清楚：

- 谁在购买：个人、公司、经纪人、代理商还是员工。
- 他们是只想要域名，还是也想要其他资产。
- 他们需要同注册商推送还是跨注册商转移。
- 谁来支付担保、市场平台、转移或付款手续费。
- 是否有验收期。
- 是否需要税务表格、发票或公司文件。

不要跟自己砍价。如果买家说“太高了”但没有还价，问问他们心中的价格范围是多少。如果他们需要内部审批，给他们一份清晰的书面摘要以便转发。

一份有用的交易摘要如下所示：

```text
域名：example.com
购买价格：$____
包含的资产：仅限域名
付款/担保路径：____
费用支付方：____
转移方式：同注册商推送 / 跨注册商转移 / 代币化转移
预计时间表：____
验收期：____
```

对于初次购买域名的买家，您可能需要平静地向他们解释转移流程。这是销售过程的一部分。许多优秀的买家以前从未从其他所有者那里购买过域名。

## 8. 安全交割

传统域名销售的核心问题很简单：

- 卖家不想在收款之前转移域名。
- 买家不想在收到域名之前付款。

这就是为什么严肃的私下交易通常使用中立的担保（Escrow）或受信任的结账流程。通常的流程是：

1. 买卖双方就书面条款达成一致。
2. 买方向交易提供资金。
3. 资金得到核实。
4. 卖方转移域名。
5. 买方或担保方确认收到域名。
6. 资金释放给卖方。

小心诈骗。[联邦贸易委员会（FTC）](https://consumer.ftc.gov/articles/how-avoid-scam) 警告说，骗子通常会施压让人们立即行动、坚持要求特定的付款方式，或发送事后证明是假支票的付款。对于域名卖家来说，常见的诈骗套路屡见不鲜：

- “我多付了钱，请退还差价。”
- “我的律师会处理担保。”
- “使用我发给你的这个担保链接。”
- “先去买这个估值报告，然后我再购买域名。”
- “今天先转移，我之后会电汇。”
- “让我们脱离平台交易，以便获得更好的价格。”

不要点击买家发送的担保链接并理所当然地认为它们是真的。请亲自输入担保或结账的 URL。在您自己的账户内确认交易。在付款路径显示资金已有保障之前，不要转移域名。

对于跨注册商转移，卖家通常先解锁域名并提供转移密码（auth code），然后买家在接收注册商处启动转移。对于同注册商推送，双方都在当前的注册商处使用各自的账户。Escrow.com 指出，当 ICANN 的 60 天[转移锁定](/zh-CN/glossary/transfer-lock/)限制了跨注册商的移动时，同注册商推送可能会非常有用。

交割后：

- 保存协议、担保记录、电子邮件、转移确认、发票和付款记录。
- 移除已失效的上架信息。
- 将任何未完成交易的域名重新锁定。
- 更新您的财务记账记录。
- 如果金额较大，请咨询税务专业人士。

在美国，美国国税局（IRS）根据资产的持有和使用方式，采用相应的税务规则来处理资产销售；[IRS 第 544 号出版物](https://www.irs.gov/publications/p544) 是一个起点，但不能替代注册会计师（CPA）。其他司法管辖区也有自己的规则。

## 特殊情况：代币化域名

如果您的域名是一个[代币化域名](/zh-CN/blog/what-are-tokenized-domains/)，交割机制可能会有所不同。

在传统销售中，担保之所以存在，是因为付款和注册商转移分在不同的步骤进行。在代币化域名的销售中，所有权可能由 [NFT](/zh-CN/glossary/nft/) 表示，并通过[链上](/zh-CN/glossary/on-chain/)交易进行转移。这可以压缩结算时间，但并不能消除所有的风险。您仍然需要买家匹配度、干净的法律背景、[钱包](/zh-CN/glossary/wallet/)安全、税务记录，以及对买家期望收到的内容的清晰理解。

有关代币化销售如何改变担保机制的更多信息，请阅读[《从上架到结算：代币化市场如何取代传统担保机制》](/zh-CN/blog/how-tokenized-marketplaces-replace-escrow/)。

## 避坑指南（切勿做的事）

- 不要假设每一个未使用的域名都有转售价值。
- 不要仅凭自动估值来定价。
- 不要将明显的商标名称推销给商标所有者。
- 不要群发数以百计的通用邮件。
- 不要贬低潜在客户当前的域名。
- 不要制造虚假的紧迫感或暗示不存在的其他买家。
- 不要在资金得到保障之前转移域名。
- 不要仅仅因为是买家发送的担保链接就直接使用。
- 不要忘记转移锁定、过期日期或 DNS 连续性。
- 不要在售出后保留失效的上架信息。

大多数域名销售的成功并不是靠巧妙的伎俩赢得的。它们是靠将合适的名字匹配给合适的买家，然后让交易变得平淡无奇（安全稳定）而赢来的。
