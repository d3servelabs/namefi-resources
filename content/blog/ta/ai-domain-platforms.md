ஓராண்டுக்கு முன்பு, "AI மற்றும் டொமைன்கள்" என்றால் பெயர் உருவாக்கும் கருவி என்று பொருள்: ஒரு பெட்டியில் உங்கள் வணிக யோசனையை உள்ளிடுவீர்கள்; அது `.com`, `.ai` பரிந்துரைகளின் பட்டியலை வழங்கும்; பிறகு மனிதர்கள் வழக்கமாகப் பயன்படுத்தும் செக்-அவுட் பக்கத்துக்குச் சென்று நீங்கள் வாங்குவீர்கள். அது இன்றும் உண்மையான, பயனுள்ள ஒரு வகைதான். ஆனால் இப்போது கதை அதோடு முடிவதில்லை.

2026-இன் தொடக்கத்திலிருந்து இரண்டாவது வகையும் நடைமுறைக்கு வந்துள்ளது: சுட்டியைப் பயன்படுத்தும் ஒரு நபருக்குப் பதிலாக ஓர் [AI முகவரே](/ta/glossary/ai-agent/) கிடைப்புநிலையைத் தேடி, விலையைப் பார்த்து, பதிவையும் தானே முடிக்கக்கூடிய தளங்கள். "இந்த யோசனைக்காக ஒரு லேண்டிங் பக்கத்தை உருவாக்கி, உண்மையான டொமைனில் வெளியிடு" போன்ற நீண்ட பணியின் ஒரு படியாகவே இது நடைபெறலாம். மேம்பட்ட பரிந்துரைப் பெட்டியிலிருந்து இது கணிசமாக வேறுபடுகிறது. இருந்தாலும், இவற்றைப் பற்றிய பல சந்தைப்படுத்தல் வாசகங்கள் உட்பட எல்லா இடங்களிலும் இந்த இரு வகைகளும் அடிக்கடி குழப்பப்படுகின்றன.

இந்த வழிகாட்டி அதற்கான வரைபடம். ஒரு தளத்தை முகவர் பயன்படுத்தக்கூடியதாக மாற்றும் இடைமுக முறைகள், இன்று முகவர் வழிப் பதிவை ஆதரிக்கும் குறிப்பிட்ட தளங்கள் (ஒவ்வொன்றாலும் உண்மையில் என்ன செய்ய முடியும், முடியாது என்பதை அவற்றின் சொந்த ஆவணங்களுடன் சரிபார்த்து), அதற்குப் பதிலாக பெரிய, நிலைபெற்ற ரெஜிஸ்ட்ரார்கள் வழங்குவது என்ன என்பவற்றை இது விளக்குகிறது. இறுதியில் முடிவெடுக்கும் அட்டவணையும் அடிக்கடி கேட்கப்படும் கேள்விகளும் உள்ளன. நேரடி எண்ணிக்கை சார்ந்த ஒப்பீடுதான் உங்களுக்கு வேண்டும் என்று ஏற்கெனவே தெரிந்தால், [Cloudflare vs Name.com vs Namefi](/ta/blog/cf-namecom-namefi/) கட்டுரைக்குச் செல்லுங்கள்.

தொடங்குவதற்கு முன் ஒரு குறிப்பு: கீழேயுள்ள பல தளங்கள் பொது beta நிலையில் உள்ளன; beta அம்சங்கள் மாறக்கூடும். இந்த வழிகாட்டி வெளியிடப்பட்ட தேதியின்படி நேரடி ஆவணங்களை வைத்து இங்குள்ள தகவல்கள் சரிபார்க்கப்பட்டுள்ளன. எனவே, குறிப்பிட்ட திறனைப் பற்றிய எந்தக் கூற்றையும் அந்தத் தேதியில் நிலவிய தகவலாகவே கருதுங்கள்; நிரந்தர விவரக்குறிப்பாக அல்ல.

## டொமைன் பதிவு ஏன் முகவர் அடுக்குக்குள் வந்தது

இருபது ஆண்டுகளுக்கும் மேலாக, டொமைனைப் பதிவுசெய்வது என்பது உலாவியில் நடக்கும் செயலாக இருந்தது: தேடல் பெட்டி, வாங்கும் கூடை, பணம் செலுத்தும் படிவம், மேலும் உண்மையில் மனிதர்தான் பயன்படுத்துகிறார் என்பதை உறுதிப்படுத்தப் பெரும்பாலும் CAPTCHA. அந்தக் காலத்தின் பெரும்பகுதியில் ரெஜிஸ்ட்ரார்களிடம் நிரலாக்க API-கள் இருந்துள்ளன. ஆனால் அந்த API-கள் ஹோஸ்டிங் கட்டுப்பாட்டுப் பலகம் அல்லது மொத்தப் புதுப்பிப்பு நிரல் போன்ற பிற மென்பொருள் அமைப்புகளுக்காக உருவாக்கப்பட்டவை; உரையாடலின் நடுவில் ஒரு திட்டத்துக்குப் பெயர் தேவை என்று முடிவெடுக்கும் மொழி மாதிரிக்காக அல்ல.

அடுத்தடுத்து இரண்டு மாற்றங்கள் நிகழ்ந்தன. முதலில், 2025 ஜூலையில் Name.com, முதல் AI-native டொமைன் தளம் என்று அது அழைத்த ஒன்றை அறிவித்தது. [Model Context Protocol](https://modelcontextprotocol.io) (MCP) மற்றும் OpenAPI schema-களை அடிப்படையாகக் கொண்ட அந்த API, "என் செயலியில் டொமைன் பதிவைச் சேர்" போன்ற இயல்பான மொழிக் கோரிக்கையிலிருந்து விவரக்குறிப்பைப் படித்து, செயல்படும் பதிவுக் குறியீட்டை ஒரு coding முகவர் எழுதக்கூடிய வகையில் வடிவமைக்கப்பட்டிருந்தது ([Name.com](https://www.name.com/blog/the-first-ai-native-domain-platform#:~:text=the%20first%20registrar%20to%20bring%20together%20intelligent%20domain%20capabilities%20and%20seamless%20integration%20for%20AI%20agents)). அடுத்து, ஏப்ரல் 15, 2026 அன்று Cloudflare ஒரு Registrar API-ஐ பொது beta-வில் வெளியிட்டது. "டொமைன்களைத் தேடவும், கிடைப்புநிலையைச் சரிபார்க்கவும், நிரல் வழியாக அவற்றைப் பதிவுசெய்யவும் Registrar API உதவுகிறது" என்பதே அதன் தெளிவான அறிமுகமாக இருந்தது (Cloudflare Blog; [துறைச் செய்தி](https://webhosting.today/2026/04/22/ai-agents-can-now-register-domains-no-human-required/) வழியாக). குறிப்பிடத்தக்க வகையில், Cursor மற்றும் Claude Code முகவர்கள் ஏற்கெனவே அணுகக்கூடிய Cloudflare MCP server-உடனும் அது நேரடியாக இணைக்கப்பட்டது.

இந்த இரண்டாவது மாற்றமே பரவலாக எழுதப்பட்டது. Cloudflare ஒரு பெரிய, நன்கு அறியப்பட்ட ரெஜிஸ்ட்ரார் என்பதும், அதன் விளக்கம் நேரடியாக இருந்ததும் காரணங்கள்: "நான் ஒப்புக்கொள்கிறேன்" என்பதைக் கிளிக் செய்து, அட்டை எண்ணை மனிதர் உள்ளிட வேண்டியிருந்ததால் தானியக்கமாக்க முடியாமல் இருந்த டொமைன் பதிவு, இப்போது ஒரு முகவர் துணைச் செயலாகச் செய்யக்கூடிய ஒன்றாக மாறியிருந்தது. 2026-இன் நடுப்பகுதியில் டொமைன் துறையை ஆய்வு செய்த CircleID இதை நேரடியாகக் கூறியது: "AI முகவர்கள் கிடைப்புநிலையைச் சரிபார்த்து, பெயர்களைப் பதிவுசெய்து, மனிதத் தலையீடு இல்லாமல் DNS-ஐ உள்ளமைக்கும் டொமைன் மறுவிற்பனையாளர்களாக அதிகரித்த அளவில் செயல்படுகின்றன" ([CircleID, ஏப்ரல் 2026](https://circleid.com/posts/the-domain-universe-in-2026-ai-security-market-maturity-and-the-new-gtld-frontier#:~:text=AI%20agents%20are%20increasingly%20acting%20as%20domain%20resellers%20checking%20availability%2C%20registering%20names%2C%20and%20configuring%20DNS%20without%20human%20intervention)).

Registry-கள் தங்கள் விதிகளை மாற்றியதால் இவை எதுவும் நடக்கவில்லை. உலாவிக்கு மட்டுமே புரியும் வகையில் இருந்த தற்போதைய கொள்முதல் செயல்முறையை, இயந்திர அழைப்பாளரும் புரிந்துகொள்ளும்படி சில தளங்கள் மாற்றத் தீர்மானித்ததே காரணம். அதற்கு "ஓர் API-ஐ வெளியிடுவதை" விட அதிகம் தேவைப்படுகிறது.

## மூன்று இடைமுக முறைகள்: மூல API, MCP server மற்றும் llms.txt

ஒவ்வொரு API-யையும் முகவரால் பயன்படுத்த முடியாது; அந்த வேறுபாட்டைத் துல்லியமாகப் பெயரிட்டு விளக்குமளவுக்கு அது முக்கியமானது. முழுச் சரிபார்ப்புப் பட்டியலுக்கு [AI முகவருக்கேற்ற ரெஜிஸ்ட்ரார் என்றால் என்ன?](/ta/blog/agent-native/) என்பதைப் பாருங்கள். சுருக்கமாகச் சொன்னால், இந்த வழிகாட்டியிலுள்ள தளங்களில் ஒன்றோடொன்று இணைந்து செயல்படும் மூன்று முறைகள் காணப்படுகின்றன.

- **மூல REST API.** இதுவே பழமையான முறை. Developer API வைத்துள்ள எந்த ரெஜிஸ்ட்ராரும், தொழில்நுட்ப ரீதியில் மென்பொருளுக்கு டொமைனைப் பதிவுசெய்ய வழி தருகிறது. ஆனால் அதைக் கண்டறிவதுதான் சிக்கல்: அந்த API இருப்பதை முகவர் ஏற்கெனவே அறிந்திருக்க வேண்டும்; அதன் context-இல் ஆவணங்கள் ஏற்கெனவே இருக்க வேண்டும்; அதற்கான client-உம் முன்கூட்டியே எழுதப்பட்டிருக்க வேண்டும். REST API மட்டும் இருப்பது, பொது நோக்கமுள்ள முகவருக்கு அது எங்கே உள்ளது அல்லது அதைச் சரியாகப் பயன்படுத்துவது எப்படி என்பதைச் சொல்லாது.
- **MCP server.** [MCP](https://modelcontextprotocol.io) என்பது திறந்த, எந்த மாதிரியையும் சாராத ஒரு நெறிமுறை. இதன் பராமரிப்பாளர்கள் இதை "AI பயன்பாடுகளை வெளிப்புற அமைப்புகளுடன் இணைப்பதற்கான தரப்படுத்தப்பட்ட வழி" என்றும், "AI பயன்பாடுகளுக்கான USB-C port" போன்றது என்றும் விவரிக்கின்றனர் ([modelcontextprotocol.io](https://modelcontextprotocol.io#:~:text=Think%20of%20MCP%20like%20a%20USB%2DC%20port%20for%20AI%20applications)). Claude, Cursor, Windsurf போன்ற எந்த இணக்கமான AI client-க்கும் அழைக்கக்கூடிய வரையறுக்கப்பட்ட கருவித் தொகுப்பை இது வழங்குகிறது. ஒரு ரெஜிஸ்ட்ரார் MCP server வழங்கும்போது, பிரித்தறிய வேண்டிய நீண்ட REST ஆவணச் சுவருக்குப் பதிலாக, துல்லியமான செயல்களின் பட்டியலை (`search_domain`, `register_domain`, `set_dns_record`) முகவரிடம் அளிக்கிறது.
- **llms.txt மூலம் கண்டறியக்கூடிய API.** [llms.txt](https://llmstxt.org) என்பது 2024-இல் முன்மொழியப்பட்ட எளிய உரை மரபு: ஒரு தளத்தின் root-இல் உள்ள `/llms.txt` கோப்பு. தளத்தின் முக்கிய ஆவணங்கள் மற்றும் திறன்களின் சுருக்கமான, தேர்ந்தெடுக்கப்பட்ட அட்டவணையை மொழி மாதிரிகளுக்கு வழங்குவதே இதன் நோக்கம்; crawlers-க்கு `robots.txt` அனுமதி விதிகளை வழங்குவது போல. உதாரணமாக `namefi.io/llms.txt`-இல் இதை வெளியிடும் ஒரு ரெஜிஸ்ட்ரார், முன்பு அந்தத் தளத்தைப் பார்த்திராத முகவருக்குக் கூட, மனிதர் API ஆவணங்களை உரையாடலில் ஒட்டாமலேயே அதன் திறன்களைக் கண்டறிய உதவுகிறது.

இவை ஒன்றுக்கொன்று போட்டியான தரநிலைகள் அல்ல. வலுவான தளங்கள் மூன்றையும் அடுக்குகின்றன: கண்டறிதலுக்கு llms.txt, உண்மையான கருவி அழைப்புகளுக்கு MCP server, இரண்டிற்கும் அடியில் REST API.

## தளம் வாரியான விவரம்

### Cloudflare Registrar API (beta)

ஏப்ரல் 15, 2026 முதல் இயங்கும் Cloudflare beta மூன்று செயல்களை வழங்குகிறது: தேடல், கிடைப்புநிலை மற்றும் விலைச் சரிபார்ப்பு, பதிவு. Cloudflare இதை "டொமைன் வாழ்நாளின் முதல் முக்கியமான தருணம்" என்று விவரிக்கிறது; இடமாற்றங்கள், புதுப்பிப்புகள், தொடர்பு விவரப் புதுப்பிப்புகள் ஆகியவை ஆண்டின் பிற்பகுதியில் வரும் என்று கூறியுள்ளது (Cloudflare Blog). விலையிடல் Cloudflare-இன் நீண்டகால ரெஜிஸ்ட்ரார் முறையைப் பின்பற்றுகிறது: கட்டுப்பாட்டுப் பலகம், API அல்லது முகவர் என எந்த வழியில் அழைப்பு வந்தாலும், markup எதுவுமின்றி "registry வசூலிப்பதையே நாங்களும் அப்படியே வசூலிக்கிறோம்" என்று அது கூறுகிறது (Cloudflare Blog).

முகவருக்கான பகுதி தனித் தயாரிப்பு அல்ல; ஒருங்கிணைப்புதான். "Registrar API முழு Cloudflare API-இன் ஒரு பகுதி; அதனால் Cloudflare MCP வழியாக முகவர்கள் இன்றே அதை அணுகலாம்" என்றும், "Cursor, Claude Code அல்லது MCP-உடன் இணக்கமான எந்தச் சூழலிலும் செயல்படும் முகவர் Registrar endpoint-களைக் கண்டறிந்து அழைக்கலாம்" என்றும் Cloudflare கூறுகிறது (Cloudflare Blog). அதன் நோக்கமுள்ள செயல்முறை விளக்கத்தில் ஒரு சரிபார்ப்புப் படி உள்ளது: முகவர் "பெயர்களைப் பரிந்துரைத்து, எதை உண்மையில் பதிவுசெய்ய முடியும் என்று உறுதிசெய்து, ஒப்புதலுக்காக விலையைக் காட்டி, பின்னர் கொள்முதலை முடிக்கலாம்" (Cloudflare Blog). ஆனால் ஆவணங்களின்படி, இது API-யிலேயே கட்டாயப்படுத்தப்பட்ட செலவு வரம்பு வசதி அல்ல; ஒரு வடிவமைப்புப் பரிந்துரை மட்டுமே.

இதை அடிப்படையாகக் கொண்டு திட்டமிடுவதற்கு முன் இரண்டு வரம்புகளை அறிந்துகொள்ள வேண்டும். beta இன்னும் Cloudflare-இன் முழு TLD பட்டியலை வழங்கவில்லை; தொடக்கத்தில் Cloudflare கூறும் "தேர்ந்தெடுக்கப்பட்ட பிரபலமான TLD-கள்" மட்டுமே உள்ளன (Cloudflare Blog). மேலும், தற்போது உள்ள Cloudflare கணக்குக்கே கட்டணம் விதிக்கப்படுகிறது. எனவே முகவர் API-ஐ அழைத்தாலும், இது fiat பணத்தையும் மனிதர் தொடங்கிவைத்த கணக்கையும் சார்ந்த உறவாகும்.

### Name.com AI-native API

2025 ஜூலையில் அறிவிக்கப்பட்ட Name.com தளம், அதே இயல்பான மொழியிலிருந்து code உருவாக்கும் எண்ணத்தை அடிப்படையாகக் கொண்டது. Developer அல்லது முகவர் தங்களுக்கு என்ன வேண்டும் என்பதை ("என் செயலியில் டொமைன் பதிவைச் சேர்") விவரித்தால், MCP மற்றும் OpenAPI-ஐ அடித்தளமாகக் கொண்டு, AI client அதைச் செயல்படும் ஒருங்கிணைப்புக் குறியீடாக மாற்றும் வகையில் அதன் ஆவணங்கள் அமைக்கப்பட்டுள்ளன. சுயசேவை developer அணுகலும் Claude, Cursor போன்ற கருவிகளுக்கான ஆதரவும் உள்ளது ([Name.com](https://www.name.com/blog/the-first-ai-native-domain-platform#:~:text=leverages%20modern%20standards%20including%20Model%20Context%20Protocol)). விலைகள் வெளிப்படையானவை, பயன்பாட்டு அளவை அடிப்படையாகக் கொண்டவை; ரெஜிஸ்ட்ரார் API-களில் பொதுவாகக் காணப்படும் மறுவிற்பனையாளர் பாணி markup அமைப்பை இது பயன்படுத்துகிறது.

Name.com-இன் அறிவிப்பில் crypto அல்லது பணப்பை வழிப் பணம் செலுத்துதல் பற்றியோ, API-க்குள்ளேயே அமைக்கப்பட்ட வெளிப்படையான human-in-the-loop உறுதிப்படுத்தல் படி பற்றியோ ஆவணம் இல்லை. வழக்கமான developer கணக்கு முறையில் இவை இரண்டுமே சாத்தியமானவை என்றாலும், மூலத்தில் தெளிவாகக் குறிப்பிடப்படவில்லை. எனவே "fiat, கணக்கு அடிப்படையிலான கட்டணம்" என்பதைத் தற்காலிகச் செயல்பாட்டு ஊகமாகவே கருதுங்கள்; முழுமையாக உறுதிப்படுத்தப்பட்ட விவரமாக அல்ல.

### Namefi: MCP server மற்றும் பணப்பை செக்-அவுட்

Namefi-இன் இயந்திரம் படிக்கக்கூடிய சொந்தத் திறன் அட்டவணையான [namefi.io/llms.txt](https://namefi.io/llms.txt), மேலே கூறிய மூன்றாவது இடைமுக முறைக்கே ஓர் எடுத்துக்காட்டு; கீழேயுள்ள தகவல்களுக்கு அதுவே ஒரே அதிகாரப்பூர்வ மூலம். Namefi, `api.namefi.io/mcp` முகவரியில் Streamable HTTP வழியாக MCP server-ஐ இயக்குகிறது. பதிவு, கிடைப்புநிலைச் சரிபார்ப்பு, DNS நிர்வாகம் ஆகியவற்றுக்கான வகைப்படுத்தப்பட்ட கருவிகளை அது வழங்குகிறது. ஒரே command மூலம் அதை Claude Code-இல் சேர்க்கலாம் (`claude mcp add --transport http namefi https://api.namefi.io/mcp`). இதன் அடியில் REST API (`api.namefi.io/v-next/`) உள்ளது; அது `x-api-key` header மூலம் அங்கீகரிக்கப்படுகிறது. டொமைனை வைத்திருக்கும் பணப்பையிலிருந்தே API விசையை உருவாக்க வேண்டும்; அதனால் தனிக் கணக்கு மீட்பு முறைக்குப் பதிலாக API அணுகல் நேரடியாக ஆன்-செயின் கட்டுப்பாட்டுடன் இணைக்கப்படுகிறது.

பணம் செலுத்தும் முறையே முக்கிய வேறுபாடு. Namefi இரண்டு வழிகளை ஆவணப்படுத்துகிறது: முன்பணம் செலுத்தப்பட்ட NFSC (Namefi Service Credits) இருப்பிலிருந்து கட்டணம் வசூலிக்கப்படும் வழக்கமான API விசை முறை; மேலும், அதன் ஆவணங்கள் Web3 பயனர்கள் மற்றும் "முகவர் சார்ந்த பணப்பைகள்" என்று குறிப்பிடுபவர்களுக்கான, SIWE (Sign-In With Ethereum) உட்பட பணப்பைக் கையொப்பங்களைப் பயன்படுத்தும் crypto-native முறை. இரண்டாவது முறையில் ரெஜிஸ்ட்ரார் கணக்கை உருவாக்காமலேயே வாங்குவதற்கான அங்கீகாரத்தை வழங்கலாம். பதிவுக்குப் பின், முழுமையான DNS record CRUD (A, AAAA, CNAME, MX, TXT மற்றும் பல), தானியங்கி புதுப்பித்தல், டொமைன் parking மற்றும் forwarding, தானியங்கி ENS record உருவாக்கம் ஆகியவற்றை Namefi ஆதரிக்கிறது. மேலும், இங்குள்ள மற்ற இரு தளங்களிலிருந்து கட்டமைப்பு ரீதியாக Namefi-ஐ வேறுபடுத்தும் அம்சமான [டோக்கனைஸ் செய்தல்](/ta/glossary/tokenized-domain/) உள்ளது: உண்மையான, ICANN-இல் பதிவுசெய்யப்பட்ட டொமைனை, ஆன்-செயினில் [பணப்பையில்](/ta/glossary/wallet/) வைத்திருக்கக்கூடிய சொத்தாக பிரதிநிதித்துவப்படுத்துதல். Claude, Codex, Cursor மற்றும் மேலும் மூன்று முகவர்களுக்கான படிப்படியான அமைப்பு [Namefi-இல் உங்கள் AI முகவர் மூலம் ஒரு டொமைனைப் பதிவுசெய்வது எப்படி?](/ta/blog/ai-agent-register/) என்ற கட்டுரையில் உள்ளது; Claude-க்கான விரிவான வழிகாட்டியை [Claude மூலம் டொமைன் வாங்குதல்: Namefi MCP படிப்படியான வழிகாட்டி](/ta/blog/claude-mcp-domains/) வழங்குகிறது. அந்த இயல்பான மொழிக் கோரிக்கை நடைமுறையில் எப்படித் தெரியும் என்பதை [இயல்பான மொழியில் டொமைன் வாங்குவது எப்படி (2026)](/ta/blog/nl-domain-purchase/) என்பதில் காணலாம்.

ஒரு குறையை நேரடியாகக் குறிப்பிட வேண்டும்: ஆதரிக்கப்படும் TLD-களின் நிலையான பட்டியலை Namefi-இன் llms.txt வெளியிடவில்லை. <!-- TODO: confirm with team — full supported TLD list --> உங்கள் பயன்பாட்டில் TLD ஆதரவுதான் முடிவெடுக்கும் காரணம் என்றால், உறுதி செய்வதற்கு முன் தற்போதைய ஆவணங்களில் நேரடியாகச் சரிபாருங்கள்.

## GoDaddy, Namecheap போன்ற நிலைபெற்ற நிறுவனங்கள் இதற்குப் பதிலாக வழங்குவது என்ன

பெரிய நுகர்வோர் [ரெஜிஸ்ட்ரார்கள்](/ta/glossary/registrar/) மேலேயுள்ள அட்டவணையில் ஏன் இல்லை என்பதைத் துல்லியமாகப் புரிந்துகொள்வது அவசியம். ஏனெனில் ["AI டொமைன் தேடல்" என்பது உண்மையில் இரண்டு வேறுபட்ட தயாரிப்புகளைக் குறிக்கப் பயன்படுத்தப்படுகிறது](/ta/blog/ai-search-meanings/). பெரிய, நிலைபெற்ற நிறுவனங்கள் AI உதவியுடன் பெயர் பரிந்துரைத்தல் மற்றும் தொடக்க அமைப்பு ஆகியவற்றில் கணிசமாக முதலீடு செய்துள்ளன: உங்கள் வணிகத்தை விவரித்தால், பிராண்டுக்கு ஏற்ற பெயர்களைப் பரிந்துரைக்கும் கருவிகள்; சில நேரங்களில் logo அல்லது தொடக்கத் தள உருவாக்கியுடன் தொகுப்பாகவும் கிடைக்கும். அது உண்மையான, பயனுள்ள தயாரிப்பு. ஆனால் மேலேயுள்ள தளங்களின் வகைக்கு அது சமமல்ல. அந்தச் செயல்முறையில் AI, மனிதரின் முடிவுக்கு உதவுகிறது; வெளிப்புற முகவர் ஒரு கருவியாக அழைக்கக்கூடிய வகையில், தானே தேடி, விலையை அறிந்து, பதிவை முடிக்கும் அதிகாரத்தை அது கொண்டிருக்கவில்லை. ஒரு நபர் இன்னமும் செக்-அவுட் பக்கத்துக்குச் சென்று வாங்கு என்பதைக் கிளிக் செய்ய வேண்டும். மேலேயுள்ள மூன்று தளங்களும் ஆவணப்படுத்தியுள்ள அதே அதிகாரத்துடன், முகவர் அழைக்கக்கூடிய API, MCP server அல்லது llms.txt கோப்பை ஒரு நிலைபெற்ற நிறுவனம் வெளியிடும் வரை, அது "மனிதர் தேர்வுசெய்ய AI உதவுகிறது" என்ற வகையில்தான் இருக்கும்; இந்த வகையில் அல்ல.

## தேர்ந்தெடுப்பது எப்படி

- **நீங்கள் ஏற்கெனவே Cloudflare சூழலில் இருக்கிறீர்கள்; இப்போது தேடல்-சரிபார்ப்பு-பதிவு மட்டும் தேவை.** உங்கள் டொமைன்களும் DNS-உம் ஏற்கெனவே Cloudflare-இல் இருந்தால் Registrar API-தான் குறைந்த உராய்வுள்ள தேர்வு. ஆனால் beta-வின் TLD பட்டியலும் அம்சத் தொகுப்பும் இன்னும் முழுமையான ரெஜிஸ்ட்ராரின் அளவை விட குறுகியவை.
- **டொமைன் பதிவின் மீது மறுவிற்பனையாளர் அல்லது பல வாடிக்கையாளர் தயாரிப்பை உருவாக்குகிறீர்கள்.** Name.com-இன் அளவு சார்ந்த விலையிடலும் சுயசேவை developer அணுகலும் மறுவிற்பனையாளர்களை மனதில் கொண்டு உருவாக்கப்பட்டவை.
- **முன்கூட்டியே உள்ள மனிதர் சொந்தக் கணக்கின்றி உங்கள் முகவர் பரிவர்த்தனை செய்ய வேண்டும், அல்லது டொமைனே இடமாற்றக்கூடிய, பணப்பையில் வைத்திருக்கக்கூடிய சொத்தாக இருக்க வேண்டும்.** இந்தத் தேவைக்காகவே [Namefi](https://namefi.io) உருவாக்கப்பட்டுள்ளது: signup படி இல்லாத, பணப்பைக் கையொப்பமிட்ட செக்-அவுட்; மேலும், மற்ற ஆன்-செயின் சொத்தைப் போல டொமைனை நகர்த்தவும் அதன் கட்டுப்பாடு யாரிடம் உள்ளது என்பதை நிரூபிக்கவும் விரும்பினால் [டோக்கனைஸ் செய்யப்பட்ட](/ta/glossary/tokenized-domain/) உரிமை.
- **முகவர் வழிக் கொள்முதல் அதிகாரம் உண்மையில் தேவையா என்று உறுதியாகத் தெரியவில்லை.** மனிதர் இன்னமும் "வாங்கு" என்பதைக் கிளிக் செய்யும்போது பெயரைத் தேர்வுசெய்வதற்கான உதவி மட்டுமே உங்களுக்கு வேண்டுமெனில், இந்த வழிகாட்டியிலுள்ள எந்தத் தளத்தையும் விட AI உதவியுள்ள பெயர் உருவாக்கும் கருவியே பொருத்தமானது. முழு வேறுபாட்டுக்கு [2026-இல் "AI டொமைன் தேடல்" என்பது இரண்டு வேறுபட்ட விஷயங்களைக் குறிக்கிறது](/ta/blog/ai-search-meanings/) என்பதைப் பாருங்கள்.

Claude போன்ற ஒரு மாதிரிக்கு டொமைனைப் பதிவுசெய்யும் உள்ளமைந்த திறன் கிடையாது. அது தேடி, விலையை அறிந்து, கொள்முதலை முடிக்க வேண்டுமெனில், ஒரு தளத்தின் MCP server அல்லது API-உடன் இணைக்கப்பட வேண்டும் (எடுத்துக்காட்டாக Namefi-இன் MCP server, அல்லது Cloudflare MCP வழியாக Cloudflare-இன் Registrar API). அந்த இணைப்பு இல்லாவிட்டால், நீங்களே பதிவுசெய்ய வேண்டிய பெயர்களை மட்டுமே AI உதவியாளர் பரிந்துரைக்க முடியும்.

### முதலில் என்னிடம் சரிபார்க்காமல் டொமைன்களைப் பதிவுசெய்து பணம் செலவிட AI முகவரை அனுமதிப்பது பாதுகாப்பானதா?
வேறு எந்தத் தானியங்கி கொள்முதல் அதிகாரத்தையும் அணுகுவது போலவே இதையும் அணுகுங்கள்: அதிகாரத்தை வழங்கும் முன்பே அதன் வரம்பை நிர்ணயியுங்கள். இந்தத் தளங்கள் ஆவணப்படுத்தியுள்ள பாதுகாப்பான முறைகள்: மொத்த அபாயத் தொகையை வரம்பிடும் முன்பணம் இருப்பு (Namefi-இன் API விசை முறை), மீண்டும் பயன்படுத்த முடியாத ஒவ்வொரு பரிவர்த்தனைக்குமான கையொப்பம் (பணப்பைக் கையொப்பமிட்ட செக்-அவுட்), அல்லது இறுதிக் கொள்முதல் அழைப்புக்கு முன் மனிதர் கைமுறையாக உறுதிப்படுத்தும் படி. இந்த வழிகாட்டியிலுள்ள எந்தத் தளமும் உங்கள் சார்பாகப் பொதுவான செலவு வரம்பைக் கட்டாயப்படுத்துவதில்லை. பொதுவாகக் கணக்கு நிதியளிப்பு வரம்புகள் அல்லது உங்கள் முகவரின் சொந்தச் செயல்முறையில் வெளிப்படையான உறுதிப்படுத்தல் படி மூலம், கட்டுப்பாட்டு வரம்பை நீங்களே அமைக்க வேண்டும்.

### API, MCP server, llms.txt ஆகியவற்றுக்கு இடையிலான உண்மையான வேறுபாடு என்ன?
REST API என்பது அழைக்கக்கூடிய அடிப்படைச் செயல்களின் தொகுப்பு. MCP server, அந்தச் செயல்களில் வரையறுக்கப்பட்ட சிலவற்றை தனிப்பயன் ஒருங்கிணைப்புக் குறியீடு இல்லாமலேயே MCP-உடன் இணக்கமான எந்த AI client-உம் நேரடியாக அழைக்கக்கூடிய தனித்தனி கருவிகளாக வழங்குகிறது. llms.txt கோப்பு என்பது கண்டறிதல் அடுக்கு: ஒரு தளத்தின் root-இல் உள்ள குறுகிய, தேர்ந்தெடுக்கப்பட்ட அட்டவணை; என்னென்ன ஆவணங்கள் மற்றும் திறன்கள் உள்ளன என்பதை முதலில் முகவருக்குச் சொல்கிறது. crawlers எதை index செய்யலாம் என்று robots.txt சொல்வதைப் போன்றது. ஒரு தளத்தில் இவற்றில் ஏதேனும் ஒன்று மட்டும் இருக்கலாம். ஆனால் வலுவான முகவர் சார்ந்த தளங்கள் மூன்றையும் இணைக்கின்றன: கண்டறியப்பட llms.txt, அழைக்கப்பட MCP, இரண்டிற்கும் அடியில் REST.

### இந்தத் தளங்களில் ஏதேனும் ஒன்றைப் பயன்படுத்த எனக்கு cryptocurrency பணப்பை தேவையா?
இல்லை. Cloudflare மற்றும் Name.com இரண்டும் வழக்கமான fiat, கணக்கு அடிப்படையிலான கட்டண முறையைப் பயன்படுத்துகின்றன. Namefi-உம் முன்பணம் இருப்பிலிருந்து கட்டணம் வசூலிக்கும் அதே வகையான API விசை முறையை ஆதரிக்கிறது. Namefi-இன் கணக்கு இல்லாத, பணப்பைக் கையொப்பமிட்ட செக்-அவுட் முறை அல்லது டோக்கனைஸ் செய்யப்பட்ட உரிமை அம்சம் குறிப்பாக உங்களுக்கு வேண்டுமெனில் மட்டுமே பணப்பை தேவை.

### இவற்றில் இன்று எந்தத் தளம் மிகவும் "முழுமையடைந்தது"?
இவற்றில் எதையும் முழுமையான, மாறாத விவரக்குறிப்பாகக் கருதக்கூடாது. Cloudflare-இன் சேவை வெளிப்படையாக beta என்று குறிக்கப்பட்டுள்ளது; அதன் TLD பட்டியல் முழுப் பட்டியலை விடக் குறுகியது. வரையறைப்படியே beta அம்சங்கள் மாறக்கூடியவை. ஒரு குறிப்பிட்ட அம்சத்தைச் சார்ந்து உருவாக்குவதற்கு முன், ஒவ்வொரு தளத்தின் நேரடி ஆவணங்களிலும் தற்போதைய திறன்களைச் சரிபாருங்கள்.

## உங்கள் அடுத்த டொமைனை Namefi-இல் வாங்கி டோக்கனைஸ் செய்யுங்கள்

உங்கள் செயல்முறைக்கு எந்த இடைமுக முறை பொருந்தினாலும், வாங்குபவர் ஒரு படிவத்தைக் கிளிக் செய்யும் நபராக இருப்பதைப் போலவே ஒரு முகவர், பணப்பை அல்லது script-ஆகவும் இருக்கக்கூடிய பயன்பாட்டிற்காக [Namefi](https://namefi.io) உருவாக்கப்பட்டுள்ளது. இது MCP server, ஆவணப்படுத்தப்பட்ட REST API, கணக்கு உருவாக்கத்தை முற்றிலும் தவிர்க்கும் பணப்பைக் கையொப்பமிட்ட செக்-அவுட் வழி ஆகியவற்றைக் கொண்ட [ICANN](/ta/glossary/icann/) அங்கீகாரம் பெற்ற [ரெஜிஸ்ட்ரார்](/ta/glossary/registrar/) ஆகும். விருப்பமான [டோக்கனைஸ் செய்தல்](/ta/glossary/tokenized-domain/) மூலம், உங்கள் முகவரின் பணப்பை வைத்திருக்கவும் நகர்த்தவும் கூடிய சொத்தாக டொமைனையே மாற்றலாம்.

**[Namefi-இல் டொமைனைத் தேடிப் பதிவுசெய்யுங்கள்](https://namefi.io).**
