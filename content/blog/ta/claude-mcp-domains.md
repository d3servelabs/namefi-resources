இந்த வழிகாட்டியின் முடிவில், [ICANN](/ta/glossary/icann/) அமைப்பில் பதிவுசெய்யப்பட்ட ஓர் உண்மையான டொமைன் உங்களிடம் இருக்கும்; அதன் DNS, நீங்கள் உருவாக்கிக்கொண்டிருப்பதைச் சுட்டும் வகையில் அமைந்திருக்கும்; இவை அனைத்தும் Claude உடனான ஓர் உரையாடலிலிருந்தே பதிவுசெய்யப்பட்டிருக்கும் — உலாவியில் செக்-அவுட் இல்லை, கார்ட் இல்லை, CAPTCHA இல்லை. இது [Namefi](https://namefi.io) MCP சேவையகத்திற்கான Namefi குழுவின் சொந்த அமைப்பு வழிகாட்டி — முகவர்களுக்காக [namefi.io/llms.txt](https://namefi.io/llms.txt) மற்றும் [docs.namefi.io](https://docs.namefi.io)-ல் நாங்கள் வெளியிடும் அதே API-ஐ மனிதர்கள் படித்துப் புரிந்துகொள்ளும் வகையில் விளக்குகிறது. ஒரு விவரம் இன்னும் இறுதிசெய்யப்படாமலோ வெளியிடப்படாமலோ இருந்தால், ஊகிக்காமல் அதை இந்த வழிகாட்டி வெளிப்படையாகக் குறிப்பிடுகிறது.

“உங்கள் [AI முகவர்](/ta/glossary/ai-agent/) மூலம் ஒரு டொமைனைப் பதிவுசெய்தல்” பற்றிய மூன்றாம் தரப்பு விளக்கங்களும் உள்ளன — [பிரபலமான ஓர் எடுத்துக்காட்டு](https://dev.to/marsheer/how-to-register-a-domain-name-with-your-ai-agent-no-human-needed-h26), Cloudflare-ன் Registrar API மீது மறுவிற்பனையாளராக உருவாக்கப்பட்ட வேறொரு MCP சேவையகத்தைப் பயன்படுத்தி இந்த முறையை விளக்குகிறது. வழங்குநர்கள் மாறினாலும் MCP-ன் அடிப்படை இயக்கமுறை ஒரே கருத்தைச் சார்ந்ததே; இந்த வழிகாட்டி Namefi-ன் சொந்த MCP சேவையகம், அதன் சொந்த அங்கீகார முறை மற்றும் அதன் சொந்த [டோக்கனாக்கப்பட்ட டொமைன்](/ta/glossary/tokenized-domain/) விருப்பத்திற்கே உரியது. இங்குள்ள தகவல் மூன்றாம் தரப்பின் விளக்கத்தை அல்ல, Namefi-ன் ஆவணங்களை ஒப்பிட்டுச் சரிபார்க்கப்பட்டது.

## சுருக்கமாக, MCP என்றால் என்ன

[Model Context Protocol](https://modelcontextprotocol.io) (MCP) என்பது ஓர் AI பயன்பாட்டை — இந்த வழக்கில் Claude-ஐ — வெளிப்புறக் கருவிகளுடனும் தரவு மூலங்களுடனும் இணைப்பதற்கான திறந்த தரநிலை. நெறிமுறையின் சொந்த ஆவணம் இதை [AI பயன்பாடுகளுக்கான USB-C போர்ட்](https://modelcontextprotocol.io/docs/getting-started/intro#:~:text=Think%20of%20MCP%20like%20a%20USB%2DC%20port%20for%20AI%20applications) என்று விவரிக்கிறது — ஒவ்வொரு கருவிக்கும் தனித்தனியான ஒருங்கிணைப்புக்குப் பதிலாக ஒரே தரநிலைப்படுத்தப்பட்ட இணைப்பான். Namefi-ன் MCP சேவையகத்துடன் இணைக்கப்பட்டதும், அரட்டையில் ஒட்டப்பட்ட ஆவணங்களிலிருந்து REST API-ஐ எதிர்பொறியமைக்க வேண்டியதற்குப் பதிலாக, கிடைக்கும்தன்மையைச் சரிபார்த்தல், டொமைனைப் பதிவுசெய்தல், DNS பதிவுகளைப் படித்தல் மற்றும் எழுதுதல் போன்ற வரையறுக்கப்பட்ட அழைக்கக்கூடிய செயல்பாடுகள் Claude-க்குக் கிடைக்கும்.

## முன்தேவைகள்

- **MCP திறன் கொண்ட Claude கிளையண்ட்.** உறுதியான, சோதிக்கப்பட்ட கட்டளைகளுடன் Claude Code-ஐ (கட்டளை வரி) இந்த வழிகாட்டி விளக்குகிறது; மேலும் ஆவணப்படுத்தப்பட்ட பொதுவான செயல்முறையுடன் Claude Desktop / claude.ai-ஐயும் (Custom Connectors வழியாக) உள்ளடக்குகிறது. Cursor அல்லது Windsurf போன்ற பிற MCP கிளையண்டுகளும் இதே சேவையகத்துடன் இணைகின்றன; அவற்றுக்கான விவரங்களை [Namefi-ல் உங்கள் AI முகவர் மூலம் ஒரு டொமைனைப் பதிவுசெய்வது எப்படி](/ta/blog/ai-agent-register/) என்ற கட்டுரையின் ஒவ்வொரு முகவருக்குமான பகுதிகளில் பார்க்கலாம். இணைப்புக் கட்டளைகள் மட்டும் தேவைப்பட்டால், சுருக்கமான [Namefi MCP விரைவுத் தொடக்கம்: Claude Code, Cursor & Windsurf](/ta/blog/mcp-quickstart/) வழிகாட்டியைப் பார்க்கவும்.
- **ஒரு Namefi API விசை**, [namefi.io/api-key](https://namefi.io/api-key)-ல் உருவாக்கப்பட்டது; *அல்லது* API விசையே இல்லாமல் ஒவ்வொரு பரிவர்த்தனைக்கும் பணம் செலுத்த விரும்பினால், ஒரு கிரிப்டோ [பணப்பை](/ta/glossary/wallet/) (முடிவுக்கு அருகிலுள்ள பணப்பைப் பகுதியைப் பார்க்கவும்).
- **நிதியளிக்கப்பட்ட NFSC இருப்பு**, Namefi-ன் உற்பத்திச் சூழலில் நீங்கள் பதிவுசெய்கிறீர்கள் எனில். NFSC (Namefi Service Credits) என்பது டொமைன் பதிவுக்கான கட்டணம் பிடிக்கப்படும் இருப்பு. உற்பத்திச் சூழலில் Namefi டாஷ்போர்டு வழியாக அதில் தொகையைச் சேர்ப்பதும், மேம்பாட்டுச் சூழல்களில் faucet முனைப்புள்ளியிலிருந்து இலவச சோதனைக் கிரெடிட்களைக் கோருவதும் பற்றி Namefi-ன் ஆவணங்கள் விளக்குகின்றன.

## படி 1: Namefi API விசையைப் பெறுங்கள்

[API விசை](https://namefi.io/api-key) தான் மிகவும் எளிய அங்கீகார வழி; இந்த வழிகாட்டி முழுவதும் இதையே பயன்படுத்துகிறது. பதிவு, DNS பதிவை உருவாக்குதல், புதுப்பித்தல், நீக்குதல் என ஒவ்வொரு செயல்பாட்டுக்கும் ஒரே header போதுமானது. விசையை உருவாக்குவதற்கு முன் நினைவில் கொள்ள வேண்டிய ஒரு விவரம்: **அதை உருவாக்கிய பணப்பையின் அனுமதிகளையே அந்த விசையும் பெறுகிறது.** உங்களிடம் ஏற்கெனவே உள்ள டொமைனின் DNS-ஐ நிர்வகிக்க விரும்பினால், அந்த டொமைனின் NFT-ஐ வைத்திருக்கும் பணப்பையிலிருந்து விசையை உருவாக்குங்கள் — வேறொரு பணப்பையிலிருந்து உருவாக்கப்பட்ட விசைக்கு, வேறு ஒருவர் [பதிவாளராக](/ta/glossary/registrant/) உள்ள டொமைனில் எழுதும் அணுகல் கிடைக்காது.

உருவாக்கப்பட்ட விசை `nfk_` முன்னொட்டுடன் தொடங்கும் சரமாக இருக்கும். ஒவ்வொரு எழுதும் செயல்பாட்டிலும் அதை `x-api-key` header ஆக அனுப்புவீர்கள்; கிடைக்கும்தன்மையைச் சரிபார்ப்பது போன்ற படிக்க மட்டும் பயன்படும் செயல்பாடுகளுக்கு அது தேவையே இல்லை.

## படி 2: Claude-ஐ Namefi MCP சேவையகத்துடன் இணைக்கவும்

ICANN அங்கீகாரம் பெற்ற [ரெஜிஸ்ட்ராரான](/ta/glossary/registrar/) Namefi, தனது முழு API பரப்பிற்கும் `https://api.namefi.io/mcp` என்ற ஒரே MCP சேவையகத்தை Streamable HTTP போக்குவரத்தில் இயக்குகிறது. அந்தச் சேவையகம் ஒவ்வொரு `/v-next` செயல்பாட்டையும் வகையிடப்பட்ட கருவியாக வழங்குகிறது — தேடல், பதிவு, DNS, டொமைன் அமைப்பு, outbound — மேலும் அதன் இருப்பும் இணைப்பு விவரங்களும் [namefi.io/.well-known/mcp/servers.json](https://namefi.io/.well-known/mcp/servers.json)-ல் கண்டறிதல் விவரிப்பாக வெளியிடப்பட்டுள்ளன. எனவே ஒரு மனிதர் URL-ஐ முதலில் ஒட்டாமலேயே முகவரால் அதை இயந்திரம் படிக்கக்கூடிய வடிவில் கண்டறிய முடியும்.

### Claude Code

Claude Code-ல் சேவையகத்தைச் சேர்க்க ஒரே கட்டளை போதும்:

```bash
claude mcp add --transport http namefi https://api.namefi.io/mcp --header "x-api-key: YOUR_KEY"
```

தனிப்பயன் அங்கீகார header உடன் தொலைநிலை HTTP MCP சேவையகத்தைச் சேர்ப்பதற்கான [Claude Code-ன் ஆவணப்படுத்தப்பட்ட தொடரமைப்புடன்](https://code.claude.com/docs/en/mcp) இது பொருந்துகிறது — பொதுவான வடிவம் `claude mcp add --transport http <name> <url> --header "<Header-Name>: <value>"`. இதை உங்கள் முனையத்தில் ஒருமுறை இயக்குங்கள் (படி 1-ல் பெற்ற விசையை `YOUR_KEY` என்பதற்குப் பதிலாக இடுங்கள்); பின்னர் Claude Code, சேவையகத்தை உங்கள் திட்டத்தின் அல்லது பயனரின் MCP அமைப்பில் எழுதும். இயல்பாக இந்தக் கட்டளை சேவையகத்தை தற்போதைய திட்டத்திற்காக மட்டும் பதிவுசெய்கிறது; எல்லா திட்டங்களிலும் கிடைக்க `--scope user` சேர்க்கவும். தொடக்கத்தில் கிடைக்கும்தன்மைத் தேடல் போன்ற படிக்க மட்டும் பயன்படும் கருவிகள் போதுமெனில், விசையை முற்றிலும் விட்டுவிட்டு பின்னர் சேர்க்கலாம்.

`claude mcp list` மூலம் இணைப்பை உறுதிசெய்யுங்கள்; அதில் `namefi` இணைக்கப்பட்டதாகத் தெரிய வேண்டும். Claude Code அமர்வுக்குள் `/mcp` இயக்கினால் Namefi சேவையகம் வழங்கும் கருவிகளின் எண்ணிக்கையைப் பார்க்கலாம்.

### Claude Desktop மற்றும் claude.ai

Claude Desktop மற்றும் claude.ai, [modelcontextprotocol.io-ல் ஆவணப்படுத்தப்பட்ட](https://modelcontextprotocol.io/docs/develop/connect-remote-servers) **Custom Connectors** வழியாக தொலைநிலை MCP சேவையகங்களுடன் இணைகின்றன: Settings-ஐத் திறந்து, Connectors பகுதிக்குச் சென்று, “Add custom connector” என்பதைத் தேர்ந்தெடுத்து, `https://api.namefi.io/mcp` என்ற சேவையக URL-ஐ உள்ளிடுங்கள். Add-ஐக் கிளிக் செய்ததும் அங்கீகாரத்தை நிறைவுசெய்யுமாறு செயல்முறை கேட்கும்; Anthropic-ன் ஆவணங்களின்படி, குறிப்பிட்ட சேவையகம் எதைத் தேவைப்படுத்துகிறது என்பதற்கேற்ப இந்தப் படியில் “பொதுவாக OAuth, API விசைகள் அல்லது பயனர்பெயர்/கடவுச்சொல் சேர்க்கைகள் இடம்பெறும்”; சேவையகம் கேட்கும் அறிவுறுத்தலை Claude காட்டும்.

<!-- குழுவுடன் உறுதிப்படுத்த வேண்டியது — x-api-key பாணி header-க்காக Claude Desktop-ன் Custom Connector அங்கீகாரத் திரை காட்டும் துல்லியமான புலம்; Anthropic-ன் பொது ஆவணங்கள் Namefi சேவையகத்தைத் தனியாகக் காட்டாமல் பொதுவான அங்கீகாரப் படியை மட்டுமே விவரிக்கின்றன --> உங்கள் Desktop connector அமைப்பில் விசையை உள்ளிடுவதற்கான தெளிவான இடம் தெரியவில்லை எனில், தற்போதைக்கு Claude Code தான் சரிபார்க்கப்பட்ட வழி; படிக்க மட்டும் பயன்படும் கருவிகள் (கிடைக்கும்தன்மைத் தேடல்) விசை இல்லாமலேயே connector வழியாகச் செயல்படும்.

## படி 3: உங்கள் NFSC இருப்பில் தொகை சேர்க்கவும்

டொமைன் பதிவு கட்டணச் செயல்பாடாகும்: பணம் செலுத்தும் பணப்பையில் NFSC (Namefi Service Credits) இருக்க வேண்டும். மேம்பாட்டு அல்லது சோதனைச் சூழலில், ஒரு faucet (`POST /v-next/user/faucet`, அல்லது SDK-ல் `client.user.requestNfscFaucet()`) இலவச சோதனைக் கிரெடிட்களை வழங்கும்; ஒவ்வொரு பணப்பைக்கும் வீத வரம்பு உண்டு. உற்பத்திச் சூழலில் Namefi டாஷ்போர்டு வழியாக NFSC இருப்பில் தொகை சேர்க்கப்படுகிறது. <!-- குழுவுடன் உறுதிப்படுத்த வேண்டியது — உற்பத்திச் சூழலில் தொகை சேர்ப்பதற்கான துல்லியமான செயல்முறை: ஏற்கப்படும் கட்டண முறைகள் என்ன, அரட்டையிலேயே நேரடியாக வாங்க முடியுமா அல்லது டாஷ்போர்டு UI வழியாக மட்டுமா --> எந்த நேரத்திலும் உங்கள் தற்போதைய இருப்பைச் சரிபார்க்கலாம் — இணைத்த பிறகு Claude-இடம் (“எனது Namefi இருப்பு என்ன?”) என்று கேட்பதன் மூலமோ, நேரடியாக `GET /v-next/balance` வழியாகவோ பார்க்கலாம்.

## படி 4: வாங்கும் உரையாடல்

MCP சேவையகம் இணைக்கப்பட்டு, இருப்பில் போதிய தொகை இருந்தால், மீதமுள்ள செயல்முறை இயல்பான மொழியிலேயே நடைபெறும். Namefi-ன் API ஆவணம் ஒவ்வொரு படிக்கும் குறிப்பிடும் அடிப்படைச் செயல்பாட்டுடன் பொருத்தப்பட்ட, குறிப்புரைகளுடனான உரையாடல் இப்படித்தான் இருக்கும்.

**1. ஒரு பெயரைக் கிடைக்குமா என்று சரிபார்க்க Claude-இடம் கேட்கிறீர்கள்.**

> “`example.com` பதிவுசெய்யக் கிடைக்கிறதா?”

Claude கிடைக்கும்தன்மைச் சரிபார்ப்பை அழைக்கும் (`checkAvailability` செயல்பாடு; நேரடியாக `GET /v-next/search/availability?domain=example.com` வழியாக அணுகலாம்; அங்கீகாரம் தேவையில்லை). பெயர் காலியாக உள்ளதா என்பதை அது தெரிவிக்கும்; ஒப்பிடுவதற்குப் பல பெயர்களைக் கொடுத்தால், மொத்தக் கிடைக்கும்தன்மை வடிவம் மூலம் அவற்றை ஒரே நேரத்தில் சரிபார்க்கவும் முடியும்.

**2. நீங்கள் உறுதிசெய்து பதிவுசெய்கிறீர்கள்.**

> “அதை ஓராண்டுக்குப் பதிவுசெய்து, `@`-ஐ 203.0.113.10-க்குச் சுட்டுமாறு DNS-ஐ அமைக்கவும்.”

Claude ஒரு பதிவுக் கோரிக்கையைச் சமர்ப்பிக்கும் (`registerDomain`, `POST /v-next/orders/register-domain`) — அல்லது DNS பதிவுகளையும் கேட்டிருந்தால், ஒருங்கிணைந்த `register-domain/records` வடிவத்தைப் பயன்படுத்தி, ஆர்டர் முடிந்தவுடன் நீங்கள் கேட்ட [A பதிவை](/ta/glossary/dns-record-types/) இடும். கோரிக்கை உடல் `normalizedDomainName` (சிற்றெழுத்தில், இறுதிப் புள்ளி இல்லாமல், `search/availability` பதிவுசெய்யக்கூடியது என்று தெரிவித்த எந்த [TLD](/ta/glossary/tld/) பெயரும்) மற்றும் `durationInYears` (0–10, இயல்புநிலை 1) ஆகியவற்றை ஏற்கிறது. விருப்பமான `nftReceivingWallet` டோக்கனாக்கத்தை கட்டுப்படுத்துகிறது — அதை விடுத்தால், உங்கள் API விசையுடன் இணைக்கப்பட்ட பணப்பைக்கு Base-ல் NFT ஆக டொமைன் பதிவுசெய்யப்படும். பதிவுக் கோரிக்கைகளில், `domainSetupOptions` ஒவ்வொரு டொமைனுக்குமான `autoRenew`, `dnssec` போன்ற மேலெழுதல்களை ஆதரிக்கிறது. பகிரப்பட்ட கோரிக்கை வகை `keepExistingNameservers` என்பதையும் வழங்குகிறது; ஆனால் Namefi அந்தப் புலத்தை **டொமைன் இறக்குமதி** ஆர்டர்களுக்கு மட்டுமே பயன்படுத்துகிறது, புதிய பதிவுகளில் அதைப் புறக்கணிக்கிறது. பதிவுசெய்யப்படாத பெயருக்கு பாதுகாத்து வைக்கக்கூடிய தற்போதைய [பெயர்ச் சேவையக](/ta/glossary/nameserver/) ஒப்படைப்பு எதுவும் இருக்காது.

**3. ஆர்டர் முடியும் வரை Claude நிலையைக் கேட்டுக்கொண்டிருக்கும்.**

பதிவு ஒத்திசைவின்றி நடைபெறும். ஆர்டர் இறுதி நிலையை அடையும் வரை Claude (அல்லது நிலையைப் பார்த்துக்கொண்டிருக்கும் நீங்கள்) `getOrder` (`GET /v-next/orders/{orderId}`) மூலம் நிலையைக் கேட்டுக்கொண்டிருக்கும்: `SUCCEEDED`, `FAILED`, `CANCELLED` அல்லது `PARTIALLY_COMPLETED`. வழக்கமான பதிவு சில கேட்பு சுற்றுகளிலேயே முடியும்; உங்களைச் சுழலும் குறியீட்டைப் பார்த்துக்கொண்டிருக்க விடாமல், அது முடிந்ததும் Claude தெரிவிக்கும்.

**4. எல்லா DNS பதிவுகளையும் முன்கூட்டியே அமைக்கவில்லை எனில், மேலும் பதிவுகளைக் கேட்கிறீர்கள்.**

> “மேலும், `www`-க்கு `cname.vercel-dns.com.` என்பதைச் சுட்டும் CNAME ஒன்றையும், `_verify` கீழ் இந்த token உடன் ஒரு TXT பதிவையும் சேர்க்கவும்.”

ஒவ்வொன்றுக்கும் Claude `createDnsRecord` (`POST /v-next/dns/records`) செயல்பாட்டை அழைக்கும். கேட்பதற்கு முன் தெரிந்திருக்க வேண்டிய இரண்டு வடிவமைப்பு விதிகள்: [CNAME](/ta/glossary/dns-record-types/) மற்றும் அதுபோன்ற பதிவு வகைகளுக்கான `rdata`, இறுதிப் புள்ளியுடன் (`cname.vercel-dns.com.`) முடிய வேண்டும்; ஆனால் `zoneName` — அதாவது டொமைன் — அப்படியிருக்கக் கூடாது. இவற்றைத் தலைகீழாக அமைப்பதே இந்தச் செயல்முறையில் validation பிழை ஏற்படுவதற்கான மிகவும் பொதுவான காரணம்.

**5. விருப்பமாக: தானியங்கி புதுப்பிப்பை இயக்குங்கள்.**

> “இந்த டொமைனுக்கு auto-renew-ஐ இயக்கவும்.”

`PUT /v-next/domain-config/auto-renew` வழியாக Claude [தானியங்கி புதுப்பிப்பை](/ta/glossary/domain-renewal/) மாற்றியமைக்கும். இது இயக்கப்பட்டால், உரிமையாளரின் பணப்பையில் கிடைக்கும் கட்டண முறைகளைப் பயன்படுத்தி காலாவதிக்கு முன் டொமைன் தானாகப் புதுப்பிக்கப்படும் — அதை இயக்குவதற்கு முன் தெரிந்திருக்க வேண்டிய விஷயம் இது; இது ஒரு தொடர்ச்சியான அங்கீகாரம், ஒருமுறை மட்டும் அளிக்கும் உறுதிப்படுத்தல் அல்ல.

## படி 5: அது resolve ஆகிறதா என்று உறுதிசெய்யுங்கள்

[DNS பரவல்](/ta/glossary/dns-propagation/) உடனடியாக நடைபெறாது; எனவே பதிவுகளைச் சரிபார்ப்பதற்கு முன் சில நிமிடங்கள் காத்திருங்கள். DNS வாசிப்புகளுக்கு அங்கீகாரம் தேவையில்லை. எனவே `GET /v-next/dns/records?zoneName=example.com` வழியாகவோ, பொது DNS lookup கருவி வழியாகவோ என்ன செயல்பாட்டில் உள்ளது என்பதை நீங்கள் (அல்லது Claude) உறுதிசெய்யலாம். டொமைனை ஒரு deployment தளத்துடன் இணைத்திருந்தால், அது கேட்ட TXT பதிவு உள்ளதா எனச் சரிபார்க்கும் அந்தத் தளத்தின் சொந்த டொமைன் உறுதிப்படுத்தல் படியையும் தனியாகச் செய்வது நல்லது.

## API விசைக்குப் பதிலாகப் பணப்பை மூலம் கட்டணம் செலுத்துதல்

மேலுள்ள அனைத்தும் API விசை வழியைப் பயன்படுத்துகின்றன. Namefi, [x402](/ta/glossary/x402/) நெறிமுறை வழியாக கிரிப்டோ பணப்பையைக் கொண்டு, Namefi கணக்கே இல்லாமல் டொமைனைப் பதிவுசெய்வதையும் ஆதரிக்கிறது: வாங்குபவரின் பணப்பை EIP-3009 அங்கீகாரத்தில் கையொப்பமிடும்; கட்டணம் இணைக்கப்படவில்லை எனில் விலையுடன் API `402 Payment Required` பதிலை வழங்கும்; செல்லுபடியாகும் கட்டணம் வந்ததும் பதிவை நிறைவேற்றும். இந்தச் செயல்முறைக்கு ஓர் அடிக்குறிப்பைவிடத் தனி வழிகாட்டியே பொருத்தமானது — முழு விவரங்களுக்கு [கிரிப்டோ பணப்பை மூலம் டொமைன்களுக்கு பணம் செலுத்துங்கள்: கணக்கு தேவையில்லை](/ta/blog/wallet-checkout/) அல்லது [Namefi-ல் உங்கள் AI முகவர் மூலம் ஒரு டொமைனைப் பதிவுசெய்வது எப்படி](/ta/blog/ai-agent-register/) என்ற கட்டுரையின் கட்டணப் பகுதியைப் பார்க்கவும்.

## அடிக்கடி கேட்கப்படும் கேள்விகள்

### இதைப் பயன்படுத்த Namefi-ன் REST API பற்றி நான் அறிந்திருக்க வேண்டுமா, அல்லது Claude-இடம் இயல்பான மொழியில் பேசினால் போதுமா?
மேலுள்ள முழுச் செயல்முறைக்கும் இயல்பான மொழியே போதுமானது — “இந்த டொமைன் கிடைக்கிறதா?”, “இதைப் பதிவுசெய்”, “இதை இந்த IP-க்குச் சுட்டு” என்பவை அனைத்தும் நேரடியான கோரிக்கைகளாகச் செயல்படும். Claude பின்னணியில் என்ன செய்கிறது என்பதை நீங்கள் உறுதிசெய்யவும், அரட்டைக்குப் பதிலாக நிரலாக்கினால் நேரடியாக அழைக்கவும் வசதியாகவே இந்த வழிகாட்டியில் முனைப்புள்ளிகளும் கோரிக்கைப் புலங்களும் ஆவணப்படுத்தப்பட்டுள்ளன.

### Claude வழியாகப் பதிவுசெய்வதற்கு Namefi இணையதளம் வழியாகப் பதிவுசெய்வதைவிட அதிகச் செலவாகுமா?
இந்த வழிகாட்டி விலை ஒப்பீட்டை எந்தத் திசையிலும் முன்வைக்கவில்லை. <!-- குழுவுடன் உறுதிப்படுத்த வேண்டியது — Namefi-ன் MCP/API விலை அதன் வழக்கமான பதிவுக் கட்டணத்துடன் பொருந்துகிறதா அல்லது வேறுபடுகிறதா --> எப்படியிருந்தாலும், கோரிக்கை உலாவி, நிரல் அல்லது MCP கருவி அழைப்பு ஆகியவற்றில் எதிலிருந்து வந்திருந்தாலும், பதிவு அதே NFSC இருப்பிலிருந்தே கட்டணத்தைப் பிடிக்கும்.

### இவ்வாறு பதிவுசெய்யும்போது என் டொமைன் தானாகவே NFT ஆக டோக்கனாக்கப்படுமா?
ஆம், இயல்பாக அப்படித்தான். பதிவுக் கோரிக்கையில் `nftReceivingWallet` குறிப்பிடப்படவில்லை எனில், உங்கள் API விசையுடன் இணைக்கப்பட்ட பணப்பைக்கு Base-ல் NFT ஆக டொமைன் பதிவுசெய்யப்படும். பதிவின்போதே அதை வேறொரு பணப்பை அல்லது சங்கிலிக்குத் திருப்பிவிடலாம்.

### Claude-ன் DNS பதிவுக் கோரிக்கையில் எழுத்துப்பிழை இருந்தால் என்ன நடக்கும் — அது என் டொமைனை அமைதியாகச் செயலிழக்கச் செய்யுமா?
DNS எழுதல்கள் செயல்படுத்தப்படுவதற்கு முன் Namefi-ன் validation வழியாகச் செல்கின்றன; தவறான `rdata` (எடுத்துக்காட்டாக, CNAME இலக்கில் இறுதிப் புள்ளி இல்லாதது) அமைதியாக ஏற்கப்படாமல் பிழையுடன் நிராகரிக்கப்படும் — மேலுள்ள சிக்கல் தீர்வு அட்டவணையைப் பார்க்கவும். இருந்தாலும், செயல்பாட்டில் உள்ள டொமைனின் DNS மாற்றங்களை மற்ற எந்த உள்கட்டமைப்பு மாற்றத்தையும் அணுகுவது போலவே அணுகுங்கள்: உறுதிசெய்வதற்கு முன் Claude சமர்ப்பிக்கவிருப்பதை மதிப்பாய்வு செய்யுங்கள்.

### Claude-க்குப் பதிலாக Cursor அல்லது Windsurf உடன் இதே MCP சேவையகத்தைப் பயன்படுத்தலாமா?
ஆம் — எந்தக் கிளையண்ட் இணைகிறது என்பதைக் கருத்தில் கொள்ளாமல் Namefi-ன் சேவையகம் அதே திறந்த MCP நெறிமுறையையே பேசுகிறது; எனவே சேவையகப் பக்கம் மாறாது. ஒவ்வொரு editor-க்கும் கிளையண்ட் இணைப்புக் கட்டளைகள் வேறுபடும்; [Namefi-ல் உங்கள் AI முகவர் மூலம் ஒரு டொமைனைப் பதிவுசெய்வது எப்படி](/ta/blog/ai-agent-register/) என்ற கட்டுரையின் ஒவ்வொரு கிளையண்டுக்குமான அமைப்புப் பகுதிகளையோ, சுருக்கமான [Namefi MCP விரைவுத் தொடக்கம்: Claude Code, Cursor & Windsurf](/ta/blog/mcp-quickstart/) வழிகாட்டியையோ பார்க்கவும்.

## உங்கள் அடுத்த டொமைனை ஓர் உரையாடலிலிருந்து வாங்குங்கள்

இது Namefi இன்று ஆதரிக்கும் துல்லியமான அமைப்பு; கற்பனை அல்ல. MCP சேவையகம் இணைக்கப்பட்டதும், ஒரு பெயரைத் தேடுவதிலிருந்து அதை பதிவுசெய்தல், DNS அமைத்தல், விருப்பமாக அதைப் பணப்பையில் வைத்திருக்கும் டோக்கனாக மாற்றுதல் வரை அனைத்தும் அரட்டையை விட்டு வெளியேறாமலேயே நடைபெறும். MCP சேவையகம் பதிவை மட்டுமல்லாமல் மேலும் பலவற்றை வழங்குகிறது — outbound lead-finding, மொத்த DNS செயல்பாடுகள், டொமைன் அமைப்பு — அமைப்பு முடிந்ததும் அனைத்தையும் இதே இணைப்பிலிருந்து கண்டறியலாம். முழுக் கருவிப் பட்டியலுக்கு [Namefi MCP சேவையகம்: AI முகவர்களுக்கான டொமைன் கருவிகள்](/ta/blog/namefi-mcp/) பார்க்கவும்.
