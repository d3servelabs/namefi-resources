ஒரு குறிப்பிட்ட நிறுவனத்தின் முகவர் மட்டுமல்லாமல், எந்த [AI முகவரையும்](/ta/glossary/ai-agent/) பயன்படுத்தி, [ICANN](/ta/glossary/icann/) அங்கீகாரம் பெற்ற [ரெஜிஸ்ட்ராரான](/ta/glossary/registrar/) [Namefi](https://namefi.io)-ல் உங்களுக்காக ஓர் உண்மையான டொமைனைப் பதிவுசெய்ய விரும்பினால், புக்மார்க் செய்துவைக்க வேண்டிய ஒரே பக்கம் இதுதான். எந்தக் கிளையண்டில் நீங்கள் கட்டளையை உள்ளிடுகிறீர்கள் என்பதைப் பொருட்படுத்தாமல் மாறாத அடிப்படைச் செயல்முறைகளை முதலில் இது விளக்குகிறது. அதன் பிறகு, இன்று மக்கள் உண்மையில் பயன்படுத்தும் Claude Desktop, Claude Code, OpenAI Codex, Cursor, Windsurf, Gemini CLI ஆகிய ஆறு முகவர்களுக்கும் தனித்தனியாகச் சரிபார்க்கப்பட்ட துல்லியமான அமைப்புப் படிகளை வழங்குகிறது. உங்கள் முகவர் இந்தப் பட்டியலில் இல்லையெனில், HTTP கோரிக்கை அனுப்பக்கூடிய எதனுடனும் செயல்படும் நேரடி REST வழிமுறையுடன் இந்த வழிகாட்டி முடிகிறது. அந்த நோக்கத்திற்காகவே Namefi-ன் முழு API பரப்பும் எளிய உரையாக வெளியிடப்பட்டுள்ளது.

இந்த வழிகாட்டியை Namefi குழுவே எழுதி பராமரிக்கிறது. எனவே ஒவ்வொரு படியிலும் Namefi சார்ந்த தகவல் நேரடி முதன்மை ஆதாரத்திலிருந்து வருகிறது: முகவர்களுக்காக [namefi.io/llms.txt](https://namefi.io/llms.txt) மற்றும் [docs.namefi.io](https://docs.namefi.io)-ல் நாங்கள் வெளியிடும் அதே API-ஐ மனிதர்கள் புரிந்துகொள்ளும் வகையில் இது விளக்குகிறது. இந்த வழிகாட்டி வெளியான தேதியில், ஒவ்வொரு முகவர் வழங்குநரின் அமைப்பும் அந்த வழங்குநரின் அப்போதைய சொந்த ஆவணங்களுடன் ஒப்பிட்டுச் சரிபார்க்கப்பட்டது. வழங்குநரின் ஆவணத்தில் தெளிவான பதில் இல்லாத இடங்களில் ஊகத்தால் நிரப்பாமல், அது வெளிப்படையாகக் குறிக்கப்பட்டுள்ளது.

நீங்கள் Claude-ஐப் பயன்படுத்தப் போவது ஏற்கெனவே தெரிந்திருந்தால், உண்மையான உரையாடல் பதிவுடன் கூடிய முழுமையான விளக்கத்துக்கு [Claude மூலம் டொமைன் வாங்குதல்: Namefi MCP படிப்படியான வழிகாட்டி](/ta/blog/claude-mcp-domains/) இங்குள்ள சுருக்கமான Claude பகுதிகளைவிட ஆழமாகச் செல்கிறது. இந்தப் பக்கம் மையம்; அந்தக் கட்டுரையும், இதில் ஆங்காங்கே உள்ள மற்ற இணைப்புகளும் அதிலிருந்து விரியும் கிளைகள்.

## “AI முகவர் மூலம் ஒரு டொமைனைப் பதிவுசெய்தல்” என்பதன் உண்மையான பொருள்

நீங்கள் படிவத்தை நிரப்பாமலேயே உங்கள் சார்பாக ஒரு முகவர் டொமைனைப் பதிவுசெய்ய இரண்டு நிபந்தனைகள் பூர்த்தியாக வேண்டும். முதலில், Namefi-ன் API-ஐ *கண்டறிந்து அழைப்பதற்கான* வழி முகவருக்கு வேண்டும். அதுதான் [Model Context Protocol](https://modelcontextprotocol.io) (MCP): ஓர் AI கிளையண்ட் வெளிப்புறக் கருவிச் சேவையகத்துடன் இணைந்து, அழைக்கக்கூடிய செயல்பாடுகளின் வரையறுக்கப்பட்ட பட்டியலைக் காண உதவும் திறந்த தரநிலை. உரையாடல் முகவராக இல்லாமல் நிரலாக்கப்பட்ட முகவராக இருந்தால், எளிய HTTP கோரிக்கையையும் பயன்படுத்தலாம். இரண்டாவதாக, முகவருக்குச் *செலவிடுவதற்கான அங்கீகாரம்* வேண்டும் — நிதியளிக்கப்பட்ட இருப்புடன் இணைக்கப்பட்ட API விசை அல்லது உடனடியாகப் பணம் செலுத்துவதற்குக் கையொப்பமிடக்கூடிய கிரிப்டோ [பணப்பை](/ta/glossary/wallet/). இந்த வழிகாட்டியில் உள்ள அனைத்தும் இந்த இரண்டில் ஏதாவது ஒரு பகுதிக்குச் சேர்ந்தவை.

Namefi தனது முழு API-க்கும் `https://api.namefi.io/mcp` என்ற ஒரே MCP சேவையகத்தை Streamable HTTP போக்குவரத்து வழியாக இயக்குகிறது. இந்தப் பக்கத்தைப் படிக்காமலேயே ஒரு முகவரோ அதை அமைக்கும் நபரோ அந்தச் சேவையகத்தைக் கண்டறிய முடியும்: [namefi.io/.well-known/mcp/servers.json](https://namefi.io/.well-known/mcp/servers.json)-ல் இயந்திரம் படிக்கக்கூடிய விவரிப்பை வெளியிட்டுள்ளோம். அதில் சேவையகம் `namefi-api` என்றும் அதன் போக்குவரத்து `streamable-http` என்றும் குறிப்பிடப்பட்டுள்ளது. கீழே உள்ள ஒவ்வொரு கிளையண்டும் அதே URL-க்குத்தான் இணைகிறது; ஒவ்வொரு கிளையண்டின் அமைப்புக் கோப்போ கட்டளை வரியோ அதை எப்படிச் சுட்டிக்காட்டச் சொல்கிறது என்பதே வேறுபாடு.

## எல்லாவற்றுக்கும் பொதுவான ஐந்து படிகள்

கீழே வரும் ஒவ்வொரு முகவருக்குமான பகுதியின் அடிப்படையிலும் இந்த வரிசையே உள்ளது. இதை இங்கே புரிந்துகொண்டால், தனிப்பட்ட முகவர் வழிமுறைகள் அனைத்தும் “இந்தக் குறிப்பிட்ட கருவியில் படி 2-ஐ எப்படிச் செய்வது?” என்பதன் விடைகளே.

1. **அணுகல் சான்றுகளைப் பெறுங்கள்.** [API விசையை](https://namefi.io/api-key) உருவாக்குங்கள் — `nfk_` என்ற முன்னொட்டுடன் தொடங்கும் இந்தச் சரம் பதிவு, DNS பதிவுகளை உருவாக்குதல், புதுப்பித்தல், நீக்குதல் என எல்லா செயல்பாடுகளுக்கும் பயன்படும். அதை உருவாக்கிய பணப்பையின் அனுமதிகளையே விசை பெறுகிறது. எனவே டொமைனின் உரிமையாளராக இருக்க வேண்டிய பணப்பையிலிருந்தே அதை உருவாக்குங்கள். Namefi API விசையை வைத்திருக்கவே விரும்பவில்லை என்றால், கீழே உள்ள பணப்பைக் கட்டண வழிக்குச் செல்லுங்கள் — அதற்கு கணக்கு தேவையில்லை.
2. **உங்கள் முகவரை MCP சேவையகத்துடன் இணைக்கவும்.** உங்கள் விசையை ஏந்திய `x-api-key` header உடன் கிளையண்டை `https://api.namefi.io/mcp` முகவரிக்குச் சுட்டுங்கள். துல்லியமான தொடரமைப்பு ஒவ்வொரு கிளையண்டுக்கும் வேறுபடும் — கீழே உங்கள் முகவருக்கான பகுதியைப் பாருங்கள்.
3. **தேடி விலையைப் பாருங்கள்.** ஒரு பெயர் கிடைக்கிறதா என்று இயல்பான மொழியில் கேளுங்கள். இது `checkAvailability` செயல்பாட்டை (`GET /v-next/search/availability?domain=…`) அழைக்கும்; இதற்கு எந்த அங்கீகாரமும் தேவையில்லை. பல பெயர்களை ஒரே நேரத்தில் பரிசீலிக்க அதன் மொத்தச் சரிபார்ப்பு வடிவத்தையும் பயன்படுத்தலாம்.
4. **பதிவுசெய்து, பின்னர் நிலையைக் கேளுங்கள்.** நீங்கள் உறுதிப்படுத்தியதும், முகவர் `registerDomain` (`POST /v-next/orders/register-domain`) கோரிக்கையைச் சமர்ப்பிக்கும். அதே அழைப்பில் DNS-ஐயும் அமைக்க வேண்டுமெனில், ஒருங்கிணைந்த `register-domain/records` வடிவத்தைப் பயன்படுத்தும். பதிவு ஒத்திசைவின்றி நடைபெறும் — கோரிக்கை உடலில் `normalizedDomainName` மற்றும் `durationInYears` இடம்பெறும். `register-domain/records` முனைப்புள்ளி கூடுதலாக `records` அணியை ஏற்றுக்கொள்ளும்; அதில் ஒவ்வொரு பதிவுக்கும் `name`, `type`, `rdata`, `ttl` இருக்கும். இதனால் ஆர்டர் முடிந்தவுடன் DNS எழுதப்படும். ஆர்டர் `SUCCEEDED`, `FAILED`, `CANCELLED` அல்லது `PARTIALLY_COMPLETED` என்ற இறுதி நிலைகளில் ஒன்றை அடையும் வரை, முகவரோ நீங்களோ `getOrder` (`GET /v-next/orders/{orderId}`) மூலம் நிலையைக் கேட்டுக்கொண்டிருக்க வேண்டும்.
5. **DNS-ஐ அமைத்துச் சரிபார்க்கவும்.** `createDnsRecord` (`POST /v-next/dns/records`) மூலம் [DNS பதிவுகளை](/ta/glossary/dns-record-types/) சேர்க்கவோ மாற்றவோ செய்யுங்கள்; தேவைப்பட்டால் [பெயர் சேவையக](/ta/glossary/nameserver/) மட்டத்திலான ஒப்படைப்பைச் சுட்டுங்கள்; டொமைன் அணுகக்கூடியதாக உள்ளதா என்று உறுதிப்படுத்துவதற்கு முன் [DNS பரவலுக்கு](/ta/glossary/dns-propagation/) சில நிமிடங்கள் கொடுங்கள்.

பதிவுக் கோரிக்கை ஒவ்வொரு டொமைனுக்குமான மேலெழுதல்களுடன் `domainSetupOptions` பொருளையும் ஏற்றுக்கொள்கிறது — `autoPark`, `autoEns`, `autoRenew`, `dnssec`, `keepExistingNameservers`. இதில் கடைசி விருப்பம், டொமைனின் தற்போதைய பெயர் சேவையக ஒப்படைப்பை வேறொரு இடத்திற்குத் திருப்பாமல் அப்படியே விடுமாறு Namefi-க்குக் கூறுகிறது; பதிவு செய்தவுடன் டொமைன் ஏற்கெனவே உள்ள வேறொரு இடத்திலேயே தொடர்ந்து இயங்க வேண்டும் என்றால் இது பயனுள்ளதாக இருக்கும். விருப்பமான `nftReceivingWallet` புலம் டொமைனின் உரிமை டோக்கன் எந்தப் பணப்பைக்குச் செல்கிறது என்பதைக் கட்டுப்படுத்துகிறது. அதை விடுத்தால், உங்கள் API விசையுடன் இணைக்கப்பட்ட பணப்பைக்கு Base-ல் NFT ஆக டொமைன் பதிவுசெய்யப்படும்.

> “Namefi-ல் `example.com` பதிவுசெய்யக் கிடைக்கிறதா என்று சரிபார்த்து, அதைக் கண்டறிய நீங்கள் எந்தக் கருவி அல்லது செயல்பாட்டை அழைத்தீர்கள் என்று சொல்லுங்கள். இன்னும் எதையும் பதிவுசெய்ய வேண்டாம்.”

இது படிக்க மட்டும் செய்யும் அழைப்பு — `checkAvailability`-க்கு அங்கீகாரம் தேவையில்லை. எனவே நிதி எதையும் சேர்ப்பதற்கு முன்பே புதிதாக இணைக்கப்பட்ட முகவரில் இதை பாதுகாப்பாக இயக்கலாம். இணைப்பும் கருவிப் பட்டியலும் இயங்குகின்றனவா என்பதை இது உடனே தெரிவிக்கும்.

Claude Desktop மற்றும் claude.ai ஆகியவை **Custom Connectors** வழியாகத் தொலைநிலை MCP சேவையகங்களுடன் இணைகின்றன. Settings-ஐத் திறந்து, Connectors பகுதிக்குச் சென்று, “Add custom connector” என்பதைத் தேர்ந்தெடுத்து, சேவையக URL ஆக `https://api.namefi.io/mcp`-ஐ உள்ளிடுங்கள். Add என்பதைக் கிளிக் செய்த பிறகு, அங்கீகாரத்தை முடிக்குமாறு Claude கேட்கும். இந்தப் படியில் பொதுவாக “OAuth, API keys, or username/password combinations” இடம்பெறும் என்றும், இணைக்கப்பட்ட சேவையகம் எதை வேண்டுமென்று கேட்கிறதோ அதற்கேற்பத் துல்லியமான கேள்வி அமையும் என்றும் Anthropic-ன் ஆவணம் விளக்குகிறது.

## Claude Code

தனிப்பயன் header உடன் தொலைநிலை HTTP MCP சேவையகத்தைச் சேர்ப்பதற்கான துல்லியமான பொதுத் தொடரமைப்பை Claude Code-ன் சொந்த ஆவணம் வழங்குகிறது:

```bash
claude mcp add --transport http namefi https://api.namefi.io/mcp --header "x-api-key: YOUR_KEY"
```

உங்கள் உண்மையான விசையை மாற்றி, இதை terminal-ல் ஒருமுறை இயக்குங்கள். இயல்பாக இது சேவையகத்தை **local** scope-ல் எழுதும் — உங்கள் தற்போதைய திட்டத்தில், உங்களுக்கு மட்டும் கிடைக்கும் (பழைய Claude Code பதிப்புகள் இந்த scope-ஐ “project” என்று அழைத்தன). உங்கள் கணினியில் உள்ள எல்லாத் திட்டங்களிலும் இணைப்பு கிடைக்க `--scope user` சேர்க்கவும்; திட்டத்தில் உள்ள அனைவருடனும் commit செய்யப்பட்ட `.mcp.json` கோப்பு வழியாகப் பகிர `--scope project` சேர்க்கவும். இணைப்பை `claude mcp list` மூலம் உறுதிப்படுத்துங்கள்; ஒரு session-க்குள் நேரடி கருவிகளின் எண்ணிக்கையை `/mcp` மூலம் பாருங்கள்.

## OpenAI Codex CLI

Codex CLI, MCP அமைப்பை இயல்பாக `~/.codex/config.toml` என்ற TOML கோப்பில் சேமிக்கிறது; நம்பகமான திட்டங்களுக்கு திட்ட அளவிலான `.codex/config.toml`-ஐயும் பயன்படுத்தலாம். ஒவ்வொரு சேவையகத்திற்கும் தனி அட்டவணை இருக்கும். எந்த விசைகள் உள்ளன என்பதிலிருந்து போக்குவரத்து வகை தீர்மானிக்கப்படுகிறது: `command` விசை இருந்தால் அது உள்ளூர் stdio சேவையகம்; `url` விசை இருந்தால் அது Streamable HTTP. அட்டவணைப் பெயர் underscore உடன் `mcp_servers` ஆகவே இருக்க வேண்டும் என்று Codex ஆவணம் குறிப்பாகச் சொல்கிறது — `mcp-servers` அல்லது அதைப் போன்ற வடிவங்கள் எந்த எச்சரிக்கையும் இன்றி புறக்கணிக்கப்படும்.

```toml
# ~/.codex/config.toml
[mcp_servers.namefi]
url = "https://api.namefi.io/mcp"
env_http_headers = { "x-api-key" = "NAMEFI_API_KEY" }
```

இந்த வடிவம் விசையைக் கோப்பில் எழுதாமல் `NAMEFI_API_KEY` என்ற சூழல் மாறியிலிருந்து எடுக்கிறது — Codex-ஐ இயக்குவதற்கு முன் அதை உங்கள் shell-ல் அமைக்கவும். அதை நேரடியாக எழுத விரும்பினால் (commit ஆகக்கூடிய கோப்புக்கு இது பரிந்துரைக்கப்படவில்லை), அதற்குச் சமமான நிலையான வடிவம் `http_headers = { "x-api-key" = "YOUR_KEY" }`. `Authorization: Bearer …` வகை அங்கீகாரத்துக்காக `bearer_token_env_var` புலத்தையும் Codex ஆவணம் குறிப்பிடுகிறது. ஆனால் Namefi-ன் `x-api-key` header-க்கு bearer-க்கென உள்ள புலமல்லாமல், பொதுப் பயன்பாட்டு `http_headers` / `env_http_headers` புலங்களையே பயன்படுத்த வேண்டும்.

## Cursor

Cursor, MCP சேவையக வரையறைகளை `mcp.json`-லிருந்து படிக்கிறது — உங்கள் repo root-ல் திட்ட அளவிலான `.cursor/mcp.json` அல்லது எல்லா இடங்களுக்கும் பொருந்தும் `~/.cursor/mcp.json`. தொலைநிலைச் சேவையக வடிவத்தை Cursor-ன் ஆவணம் நேரடியாக வழங்குகிறது. இதில் header அடிப்படையிலான அங்கீகாரமும், விசையை கோப்பில் சேமிக்கத் தேவையில்லாத சூழல்-மாறி இடைச்செருகலும் அடங்கும்:

```json
{
  "mcpServers": {
    "namefi": {
      "url": "https://api.namefi.io/mcp",
      "headers": {
        "x-api-key": "${env:NAMEFI_API_KEY}"
      }
    }
  }
}
```

இணைப்பு ஏற்படும் நேரத்தில் அந்தச் சூழல் மாறியில் உள்ள மதிப்பாக `${env:NAMEFI_API_KEY}` மாறும். இதே அமைப்பின் சுருக்கமான வடிவத்துக்கு [Namefi MCP விரைவுத் தொடக்கம்: Claude Code, Cursor & Windsurf](/ta/blog/mcp-quickstart/) பார்க்கவும்.

## Windsurf (Cascade)

Windsurf-ன் MCP ஒருங்கிணைப்பு, தயாரிப்புக்குள் **Cascade** என்று அழைக்கப்படுகிறது. அது தனது சேவையகப் பட்டியலை `~/.codeium/windsurf/mcp_config.json`-லிருந்து படிக்கிறது. தொலைநிலை HTTP சேவையகங்கள் `command` அல்லாமல் `serverUrl` புலத்தையும், Cursor போலவே `headers` பொருள் மற்றும் `${env:VAR}` இடைச்செருகலையும் பயன்படுத்துகின்றன:

```json
{
  "mcpServers": {
    "namefi": {
      "serverUrl": "https://api.namefi.io/mcp",
      "headers": {
        "x-api-key": "${env:NAMEFI_API_KEY}"
      }
    }
  }
}
```

தெளிவாகக் குறிப்பிட வேண்டிய ஒன்று: இந்த வழிகாட்டி வெளியான தேதியில், `docs.windsurf.com/windsurf/cascade/mcp` என்பது `docs.devin.ai/desktop/cascade/mcp`-க்குத் திருப்பிவிடப்படுகிறது. Windsurf-ன் ஆவணம் இப்போது Cognition-ன் Devin தயாரிப்பு ஆவண domain-ன் கீழ் உள்ளது; அந்தப் பக்கமே “Windsurf”, “Cascade”, “Devin Desktop” ஆகிய அனைத்தையும் குறிப்பிடுகிறது. மேலே உள்ள அமைப்பு வடிவமே அந்தத் தற்போதைய பக்கத்தில் ஆவணப்படுத்தப்பட்டுள்ளது. பழைய Windsurf பதிப்பைப் பயன்படுத்தினால், புலப் பெயர்கள் பொருந்த வேண்டும்; இருந்தாலும், உங்கள் பதிப்பின் பயன்பாட்டுக்குள் உள்ள உதவி இணைப்பு சுட்டும் ஆவண URL-ஐச் சரிபார்க்கவும்.

## Gemini CLI

Gemini CLI, MCP சேவையகங்களை `settings.json`-லிருந்து படிக்கிறது — பயனர் அளவில் `~/.gemini/settings.json` அல்லது அந்தத் திட்டத்துக்குள் மட்டும் பொருந்தும் திட்ட அளவிலான `.gemini/settings.json`. தொலைநிலைச் சேவையக வடிவம் `url` என்பதற்குப் பதில் `httpUrl`-ஐப் பயன்படுத்துகிறது:

```json
{
  "mcpServers": {
    "namefi": {
      "httpUrl": "https://api.namefi.io/mcp",
      "headers": {
        "x-api-key": "YOUR_KEY"
      }
    }
  }
}
```

குறிப்பிட்ட கருவி அழைப்புக்கு வழக்கத்தைவிட அதிக நேரம் தேவைப்பட்டால் `timeout` புலத்தையும் Gemini CLI ஆவணம் குறிப்பிடுகிறது (மில்லிவிநாடிகளில்; இயல்புநிலை 600,000). பதிவு நிலையைக் கேட்க இதன் தேவை இருக்காது; கிளையண்ட் ஒவ்வொரு தனி அழைப்புக்காக மட்டுமே காத்திருக்கிறது, முழு polling சுழற்சிக்காக அல்ல.

## வேறு எந்த MCP ஆதரவுள்ள முகவரும்

உங்கள் முகவர் MCP-ஐ ஆதரித்தாலும் மேலே உள்ள ஆறில் ஒன்றாக இல்லையெனில், எந்தக் கிளையண்ட் இணைந்தாலும் சேவையகப் பகுதி ஒரே மாதிரியாகத்தான் இருக்கும்: தனிப்பயன் header ஆக `x-api-key: YOUR_KEY` உடன் Streamable HTTP வழியாக `https://api.namefi.io/mcp`-க்குச் சுட்டுங்கள். குறிப்பிட்ட அமைப்புக் கோப்பு அல்லது கட்டளைத் தொடரமைப்புக்கு உங்கள் கிளையண்டின் சொந்த ஆவணத்தைப் பாருங்கள். மனிதர் கையால் நகலெடுத்து ஒட்டாமல் ஒரு முகவரோ அதை அமைக்கும் நபரோ சேவையகத்தின் URL, போக்குவரத்து, அங்கீகாரத் தேவைகளை கண்டறியவே [namefi.io/.well-known/mcp/servers.json](https://namefi.io/.well-known/mcp/servers.json) விவரிப்பு உள்ளது.

உங்கள் கிளையண்ட் தொலைநிலை HTTP அல்லது SSE-ஐ நேரடியாக ஆதரிக்காமல் **உள்ளூர் (stdio) MCP சேவையகங்களை** மட்டும் ஆதரித்தால், ஒரு வழி இருக்கிறது: சமூகத்தால் பராமரிக்கப்படும் `mcp-remote` package, தொலைநிலை Streamable HTTP சேவையகத்தை உங்கள் கிளையண்ட் வழக்கம்போல் தொடங்கக்கூடிய உள்ளூர் செயல்முறையாக இணைத்து, நீங்கள் அமைக்கும் header-களை முன்னனுப்பும். இது Namefi வெளியிட்ட வழி அல்ல; மூன்றாம் தரப்பு இணைப்பானதால், Namefi-ன் சொந்த ஆவணங்களுடன் இதை இந்த வழிகாட்டி சரிபார்க்க முடியாது. உங்கள் கிளையண்ட் உண்மையாகவே தொலைநிலை HTTP-ஐ நேரடியாக ஆதரிக்கவில்லை என்றால் மட்டும் இதை மாற்றுவழியாகக் கருதுங்கள்; இயல்புநிலை வழியாக அல்ல. <!-- TODO: verify — an exact mcp-remote invocation for Namefi's server if a client without native Streamable HTTP support needs it -->

## MCP இல்லையா? நேரடி REST வழி

மேலே விவரிக்கப்பட்ட ஒவ்வொரு செயல்பாடும் எளிய HTTPS முனைப்புள்ளியாகவும் கிடைக்கிறது. அவை [namefi.io/llms.txt](https://namefi.io/llms.txt)-ல் ஒவ்வொரு முனைப்புள்ளியாகவும், [docs.namefi.io](https://docs.namefi.io)-ல் முழுமையாகவும் ஆவணப்படுத்தப்பட்டுள்ளன. HTTP அழைப்புகளைச் செய்யக்கூடிய ஆனால் MCP பேசாத முகவர் கட்டமைப்பு — தனிப்பயன் நிரல், வேறொரு முகவர் runtime, CI வேலை — இதே செயல்முறையை நேரடியாக இயக்கலாம்:

```bash
# 1. Check availability (no auth required)
curl "https://api.namefi.io/v-next/search/availability?domain=example.com"

# 2. Register (requires x-api-key)
curl -X POST "https://api.namefi.io/v-next/orders/register-domain" \
  -H "x-api-key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"normalizedDomainName": "example.com", "durationInYears": 1}'

# 3. Poll the order until it reaches a terminal status
curl "https://api.namefi.io/v-next/orders/{orderId}" \
  -H "x-api-key: YOUR_KEY"
```

llms.txt என்பது எளிய உரை மரபு — வடிவமைக்கப்பட்ட ஆவணப் பக்கங்களைத் தேடிப் படிக்காமல், ஓர் API என்ன செய்கிறது என்பதை AI முகவர் கண்டறிய ஒரு தளம் தனது root-ல் வெளியிடும் இயந்திரம் படிக்கக்கூடிய அட்டவணை. மேலே உள்ள சுருக்கத்துக்குப் பதில் முழுப் பதிப்பை விரும்பினால், Namefi-ன் கோப்பு [namefi.io/llms.txt](https://namefi.io/llms.txt)-ல் நேரடியாகப் படிக்கும் அளவுக்குச் சுருக்கமாக உள்ளது. இந்த மரபைப் பற்றி மேலும் அறிய [டொமைன்களுக்கான llms.txt: எந்த AI முகவரும் படிக்கக்கூடிய API](/ta/blog/llms-txt/) பார்க்கவும்.

## கட்டணம் செலுத்துதல்: API விசையா பணப்பை செக்-அவுட்டா?

மேலே உள்ள அனைத்துப் பகுதிகளும், நிதியளிக்கப்பட்ட NFSC (Namefi Service Credit) இருப்பிலிருந்து கட்டணம் எடுக்கப்படும் API விசையைப் பயன்படுத்துவதாகக் கருதுகின்றன. அதை எந்த நேரத்திலும் `GET /v-next/balance` மூலம் சரிபார்க்கலாம் (`x-api-key` தேவை). மேம்பாட்டு சூழல்களில் faucet முனைப்புள்ளி வழியாகவும், production-ல் Namefi dashboard வழியாகவும் இருப்பை நிரப்பலாம். <!-- TODO: confirm with team — the exact production NFSC top-up flow: accepted payment methods, and whether it's purchasable through chat/API or only the dashboard UI -->

Namefi கணக்கே இல்லாமல் கிரிப்டோ பணப்பை மூலம் டொமைனைப் பதிவுசெய்யவும் Namefi ஆதரவளிக்கிறது. [x402](/ta/glossary/x402/) நெறிமுறை வழியாக, முகவரின் பணப்பை EIP-3009 அங்கீகாரத்தில் கையொப்பமிடுகிறது; கட்டணம் இன்னும் இணைக்கப்படவில்லை என்றால் விலையைக் காட்டும் HTTP 402 பதிலை API தருகிறது; செல்லுபடியாகும் கையொப்பமிட்ட கட்டணம் வந்ததும் பதிவு நிறைவேறும் — பொதுவாக USDC போன்ற [நிலையான நாணயத்தில்](/ta/glossary/stablecoin/). இதனுடன் தொடர்புடைய MPP (Machine Payable Protocol) challenge-response வடிவமும், இந்த இரு சுருக்குவழிகளில் எதையும் பயன்படுத்தாத பணப்பைகளுக்கான கைமுறை EIP-712 கையொப்ப வழியும் உள்ளன. இந்த வழிகாட்டி குறிப்பிடும் முகவர்களுக்கு இந்தப் பணப்பை-முதல் வழி முக்கியமானது: இது கணக்கை உருவாக்கும் படியை முற்றிலும் நீக்குவதால், தன்னாட்சி செயல்முறை API விசையை வைத்திருக்கவோ கசியவிடவோ வேண்டியதில்லை. முழுமையான செயல்முறைக்கு [கிரிப்டோ பணப்பை மூலம் டொமைன்களுக்குப் பணம் செலுத்துங்கள்: கணக்கு தேவையில்லை](/ta/blog/wallet-checkout/) பார்க்கவும்.

## முகவருக்குக் கொள்முதல் அதிகாரம் வழங்குவதற்கு முன் பாதுகாப்புக் கட்டுப்பாடுகள்

ஒரு டொமைனைப் பதிவுசெய்யக்கூடிய முகவரால் பணத்தைச் செலவிடவும், இயங்கிக்கொண்டிருக்கும் தளத்தின் DNS-ஐ மாற்றவும் முடியும். எனவே சில முடிவுகளை இயல்புநிலைக்கு விட்டுவிடாமல் கவனமாக எடுக்க வேண்டும்:

- **API விசையைத் தேவையான குறைந்தபட்ச பணப்பைக்குள் மட்டுப்படுத்துங்கள்.** விசை அதை உருவாக்கிய பணப்பையின் அனுமதிகளைப் பெறுகிறது. புதிய பதிவுகளின் உரிமையாளராக இருக்க வேண்டிய பணப்பையிலிருந்து அதை உருவாக்குங்கள்; முகவரின் விசை வெளிப்பட்டால் பாதிக்கக் கூடாத சொத்துகள் உள்ள பணப்பையிலிருந்து உருவாக்க வேண்டாம்.
- **முகவர் செலவிடக்கூடிய தொகைக்கு உச்சவரம்பு வையுங்கள்.** NFSC இருப்பே ஒரு செலவு உச்சவரம்பாகும். மனிதக் கண்காணிப்பின்றி முகவர் பயன்படுத்துவதில் உங்களுக்கு ஏற்ற அளவை மட்டும் அதில் சேருங்கள்; பெரிய நிரந்தர இருப்பை வைத்திருக்க வேண்டாம்.
- **எந்த இடத்தில் மனித ஒப்புதல் தேவை என்பதை முடிவுசெய்யுங்கள்.** கிடைப்புநிலைத் தேடல் போன்ற படிக்க மட்டும் செய்யும் செயல்பாடுகளுக்கு அங்கீகாரம் தேவையில்லை; ஆபத்தும் இல்லை. ஓர் அழைப்பு `registerDomain`-ஐச் சமர்ப்பிக்கும், தானியங்கி புதுப்பிப்பை மாற்றும் அல்லது ஏற்கெனவே இணையப் போக்குவரத்தை வழங்கும் டொமைனில் DNS பதிவை எழுதும் தருணமே, முகவர் தன்னிச்சையாகத் தொடராமல் வெளிப்படையான உறுதிப்படுத்தலைக் கோர வேண்டிய எல்லை.
- **DNS மாற்றங்களை உறுதிப்படுத்துவதற்கு முன் மறுஆய்வு செய்யுங்கள்,** மற்ற எந்த உள்கட்டமைப்பு மாற்றத்தையும் பரிசீலிப்பதைப் போல. Namefi-ன் சரிபார்ப்பு தவறான பதிவுகளை மௌனமாக ஏற்காமல் நிராகரிக்கும் (கீழே உள்ள சிக்கல் தீர்வு அட்டவணையைப் பார்க்கவும்). ஆனால் சரிபார்ப்பு வடிவமைப்புத் தவறுகளை மட்டுமே கண்டறியும்; தொடரமைப்பில் சரியாக இருந்தாலும் தவறான மதிப்பைக் கண்டறியாது.

[AI முகவர்களை மையமாகக் கொண்ட டொமைன் ரெஜிஸ்ட்ரார் என்றால் என்ன?](/ta/blog/agent-native/) என்பது Namefi உட்பட எந்த ரெஜிஸ்ட்ராரின் முகவர் சார்ந்த சேவையையும் மதிப்பிடுவதற்கான விரிவான சரிபார்ப்புப் பட்டியலை வழங்குகிறது — கண்டறியும் வசதி, இயந்திரம் படிக்கக்கூடிய பிழைகள், மனிதர் கிரெடிட் கார்டை வைத்திருப்பதாகக் கருதாத கட்டண வழிகள் ஆகியவை இதில் அடங்கும்.

## அடிக்கடி கேட்கப்படும் கேள்விகள்

### நான் ஒரு முகவரை மட்டும் தேர்ந்தெடுத்து அதிலேயே தொடர வேண்டுமா?

தேவையில்லை. எந்தக் கிளையண்ட் இணைந்தாலும் MCP சேவையகமும் ஒவ்வொரு REST முனைப்புள்ளியும் ஒன்றுதான். இன்று Claude Code-க்கும் நாளை Cursor-க்கும், அதே API விசை மற்றும் அதே NFSC இருப்பைப் பயன்படுத்தி அமைப்பை இயக்கலாம்; இடமாற்றப் படி எதுவும் இல்லை.

### ஒரு டொமைனைப் பதிவுசெய்ய இந்த முகவர்களில் எது “சிறந்தது”?

இந்தப் பணிக்கான திறனில் அர்த்தமுள்ள வேறுபாடு இல்லை; ஒவ்வொரு கிளையண்டும் சேவையகத்தில் உள்ள அதே செயல்பாடுகளையே அழைக்கிறது. வேறுபாடுகள் அந்தந்த கிளையண்டின் MCP அமைப்புத் தொடரமைப்பில் மட்டுமே உள்ளன. அதனால்தான் இந்த வழிகாட்டி ஒவ்வொன்றுக்கும் தனிப் பகுதியையும் ஒரே சோதனைக் கட்டளையையும் வழங்குகிறது — ஒவ்வொரு கிளையண்டிலும் ஒருமுறை இயக்கி, உரையாடல் பதிவுகளை நீங்களே ஒப்பிடுங்கள்.

### என் முகவர் MCP-ஐ ஆதரிக்கவில்லை என்றால் என்ன செய்வது?

மேலே உள்ள நேரடி REST வழியைப் பயன்படுத்துங்கள். MCP கருவி அழைப்பு சென்றடையும் ஒவ்வொரு செயல்பாடும் ஆவணப்படுத்தப்பட்ட HTTPS முனைப்புள்ளியாகவும் உள்ளது. உலாவி இல்லாமல் முகவரோ அதை அமைக்கும் நபரோ படிக்கக்கூடிய எளிய உரை நுழைவுப் புள்ளியாகவே `namefi.io/llms.txt` வடிவமைக்கப்பட்டுள்ளது.

### இவ்வாறு பதிவுசெய்யும்போது என் டொமைன் தானாகவே டோக்கனைஸ் செய்யப்படுமா?

ஆம், இயல்பாகச் செய்யப்படும். பதிவுக் கோரிக்கையில் `nftReceivingWallet`-ஐக் குறிப்பிடவில்லை என்றால், உங்கள் API விசையுடன் இணைக்கப்பட்ட பணப்பைக்கு Base-ல் NFT ஆக டொமைன் பதிவுசெய்யப்படும். பதிவு செய்யும் நேரத்திலேயே வேறொரு பணப்பைக்குத் திருப்பலாம்.

### நான் API விசையை வைத்திருக்காமலேயே ஒரு முகவரால் டொமைனைப் பதிவுசெய்ய முடியுமா?

ஆம் — பணப்பைக் கையொப்பமிட்ட x402 செக்-அவுட் வழிக்கு Namefi கணக்கோ API விசையோ தேவையில்லை; நிதியளிக்கப்பட்ட பணப்பை மட்டும் போதும். மேலே உள்ள கட்டணப் பகுதி அதன் அடிப்படைகளை விளக்குகிறது. முழுமையான வழிமுறைக்கு [கிரிப்டோ பணப்பை மூலம் டொமைன்களுக்குப் பணம் செலுத்துங்கள்: கணக்கு தேவையில்லை](/ta/blog/wallet-checkout/) பார்க்கவும்.
