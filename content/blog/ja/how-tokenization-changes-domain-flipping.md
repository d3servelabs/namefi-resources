[ドメインフリッピング](/ja/blog/domain-flipping/)の作業のほとんどは、ドメイン名そのものとは無関係だ。名前を探し出し、評価し、保護し、買い手を見つける——そして誰もが嫌がる段階にたどり着く。資産を実際に移転し、どちらか一方が損をせずに代金を受け取る段階だ。この決済ステップは遅く、手作業で、見知らぬ者同士の信頼に依存している。トークン化は、その構造を書き換える変化だ。

ドメインをオンチェーンに移しても、価値のない名前が良くなるわけでも、良い名前が安くなるわけでもない。変わるのは取引の*仕組み*だ——何を買っているかをどう確認するか、どう保有するか、どう移転するか、そして資金決済がどう行われるか。この記事では、フリッピングのライフサイクルにおいてトークン化が実際に作業を変える4つのポイント——取得、保管、移転、転売——を順に見ていく。基本概念をまだ知らない方は[トークン化ドメインとは何か](/ja/blog/what-are-tokenized-domains/)から始めるとよい。より深いトレーダー向けの解説は、クラスターの柱である[オンチェーンドメインフリッピング](/ja/blog/onchain-domain-flipping/)を参照してほしい。

## まず「オンチェーン」が実際に意味すること

「ブロックチェーンドメイン」と一括りにされる3種類のものは、同じ資産ではない。この区別は正確に理解する必要がある。

`vitalik.eth` のような [ENS](/ja/glossary/ens/) 名や `brand.crypto` のような [Unstoppable 系](/ja/blog/ens-vs-unstoppable-vs-tokenized-dns/)の名前は完全にオンチェーンに存在し、[ICANN](/ja/blog/what-are-tokenized-domains/) のルートシステム外にある。リゾルバーやブリッジなしには通常のブラウザでは解決されない。一方、**トークン化ドメイン**とは、あらゆるブラウザで機能する正真正銘の ICANN ドメイン——`.com`、`.xyz`、`.io` など——であり、その所有権が [NFT](/ja/glossary/nft/) として[ウォレット](/ja/glossary/wallet/)上にも表現されているものだ。[DNS](/ja/glossary/dns/) レコードとオンチェーントークンは同期が維持されるため、ドメインはこれまでどおり解決し続けながら、所有権がウォレットネイティブになる。これらのカテゴリの違いは[トークン化ドメイン vs Web3 ドメイン](/ja/blog/tokenized-domain-vs-web3-domain/)で詳しく説明しており、本記事全体の前提となる区別でもある。フリッピングが変わると言うとき、それはオンチェーン所有権レイヤーを持つ*本物の*ドメインのフリッピングを指す——並行ネームスペースの売買ではない。

この仕組みを支えるトークン規格の多くは [ERC-721](/ja/glossary/erc-721/) だ。オリジナルの仕様によれば、スマートコントラクト内で [NFT のための標準 API の実装を可能にする](https://eips.ethereum.org/EIPS/eip-721#:~:text=allows%20for%20the%20implementation%20of%20a%20standard%20API%20for%20NFTs%20within%20smart%20contracts) Ethereum インターフェースである。この「標準 API」こそが話の核心だ。トークン化ドメインが他の NFT と同じインターフェースを持つことで、既存のすべてのウォレット、マーケットプレイス、[スマートコントラクト](/ja/glossary/smart-contract/)が、カスタム統合なしにドメインを扱えるようになる。

## 取得：本当に検証できる名前を買う

![虫眼鏡がドメイン NFT トークンを保有するウォレットを映し出し、ブロックの公開台帳と透明なプロベナンス履歴が周囲に広がる編集イラスト](../../assets/how-tokenization-changes-domain-flipping-01-verify.jpg)

レジストラの流通市場では、購入対象の確認が面倒だ。マーケットプレイスのリスティング、プライバシー保護で隠れている可能性のある WHOIS レコード、そして「本当にドメインを管理していて、引き渡す」という売り手の言葉を信じるしかない。実際に所有しているとわかるのは、[クロスレジストラ移転](/ja/blog/how-tokenized-marketplaces-replace-escrow/)が数日後にクリアされてからだ。

オンチェーンでは、所有権は公開された事実だ。ドメインの NFT は誰でも読めるアドレスに存在し、発行したスマートコントラクトは監査可能で、移転履歴はブロックエクスプローラーで確認できる。1ドルを使う前に、どのウォレットが名前を保有しているか、どのコントラクトが管理しているか、過去に移転されたか何かにラップされたかを正確に確認できる。これはデューデリジェンスの実質的な向上だ——旧来の流通市場では自分では実行できないプロベナンスチェックが可能になる。まだ保管を取得していない資産の価格設定に最も役立ち、オンチェーンのプロベナンスは根拠のある評価値への追加情報となる。

正直な注意点として：*トークン*の検証は容易だが、*その下にある名前*も検証しなければならない。トークン化 `.com` は、それが反映している DNS ドメインの質次第だ。更新状況、[ICANN](/ja/glossary/icann/) のポリシーリスク、商標リスクは、登記書類がオンチェーンにあるからといって消えるわけではない。トークン化は所有権を読みやすくする。名前を合法的にフリップできるかどうかは別問題だ。

## 保管：資産を自分で保有する

これがすべての構造的変化の根幹だ。従来のモデルでは、ドメインを本当の意味で「保有」しているわけではない——ドメインを保有するレジストラの*アカウント*を保有しているにすぎない。これが[カストディアル所有](/ja/glossary/custodial-ownership/)だ。アカウントがロック、停止、または消失すれば、支払った金額にかかわらず名前も失われる。

トークン化ドメインは自分のウォレットに保管される。秘密鍵を保有するということは、資産を保有するということだ。これはクリプト資産をポータブルにしたセルフカストディモデルをドメインに適用したものだ——そしてこれには両面がある点を、フリッパーは過小評価しがちだ。セルフカストディはレジストラを単一障害点から排除するが、代わりに*自分自身*が単一障害点になる。鍵を失えば、パスワードをリセットするサポート窓口はない。

意味のある規模のポートフォリオを保有する人にとって、これはウォレットセキュリティをフリッピングの中核スキルとして扱うべき理由だ。後回しにすべきものではない。資産の移転に複数の鍵を必要とする[マルチシグウォレット](/ja/glossary/multi-sig/)は標準的なツールだが、[マルチシグウォレットは本当にセキュリティを改善するか](/ja/blog/do-multisig-wallets-actually-improve-security/)で説明しているとおり、それはトレードオフであり魔法の盾ではない。セルフカストディでは回復も自分の責任になるため、災害が起きる前に選択肢を知っておくことは非交渉的だ。鍵を失った場合に何ができるかについては、[ウォレット紛失後のトークン化ドメイン回復](/ja/blog/recovering-a-tokenized-domain-after-wallet-loss/)を参照してほしい。

## 移転：1週間ではなく数分で

![旧来のレジストラ移転を示すカレンダーの消された日付と錠前、対してドメイン NFT が1つの確認済みブロックで2つのウォレット間を移動するオンチェーン移転の速さを対比した編集イラスト](../../assets/how-tokenization-changes-domain-flipping-02-transfer.jpg)

ここがレジストラの世界との対比が最も際立つ点であり、フリッピングの摩擦のほとんどが実際に存在する場所だ。

旧来の方法でドメインの所有者を変更するには、実際の待機期間が組み込まれた移転ポリシーに従わなければならない。gTLD ドメインを登録したり新しいレジストラへ移転したりすると、ICANN のルールによってロックがかかる。レジストラは、特定の登録者変更後60日間、[他のレジストラへの移転を阻止するロックを課さなければならない](https://support.dnsimple.com/articles/icann-60-day-lock-registrant-change/#:~:text=any%20transfer%20to%20another%20registrar%20for%20sixty%20%2860%29%20days)。通常のレジストラ間移転でも、認証コード、メール確認、複数日のクリアリングウィンドウが必要だ。これは悪意があるわけではなく、ハイジャックに対抗するために存在する。しかしそれは摩擦であり、摩擦はスピードに依存するフリッピングを殺す。

オンチェーンの移転は1つのトランザクションだ。トークンが1つのウォレットから別のウォレットに移動し、ブロックで確認される。プラットフォームは DNS 側のレコードを同期して名前の解決が途切れないようにする。ENS も自身の名前について同じ点を指摘している——ユーザーはレジストリと対話して[他の ERC721 トークンと同様に](https://docs.ens.domains/registry/eth#:~:text=just%20like%20with%20any%20other%20ERC721%20token)名前を移転できる——そしてトークン化 ICANN ドメインもこの特性を継承する。フリッパーにとって「移転はトランザクション」とは、取引が合意されたのと同じセッション内でクローズできることを意味する。レジストラ移転を1週間見守る必要はない。

## 転売：エスクローをアトミック決済に置き換える

![コインとドメイン NFT トークンが1つのループ内で同時に交換されるアトミックスワップのイラスト。エスクローエージェントは不要として脇に除外されている編集イラスト](../../assets/how-tokenization-changes-domain-flipping-03-atomic.jpg)

トークン化がフリッピングについて変える最も重要なことは、資金決済の仕組みだ。

あらゆるドメイン売買における古典的な膠着状態は信頼の順序問題だ。売り手は代金を受け取る前に移転しないし、買い手は名前を受け取る前に支払わない。従来の解決策は[エスクロー](/ja/glossary/escrow/)だ——中立的な第三者が資金を預かり、移転が完了したら解放し、このギャップを埋めるための手数料（通常数パーセント）を受け取る。機能するが、遅く、すべての取引でコストがかかる。

オンチェーンでは、このギャップは機械的に解消できる。支払いと資産移転は[アトミック転送](/ja/glossary/atomic-transfer/)によって同じトランザクション内で行われる——買い手の資金*と*ドメイン NFT の両方が移動するか、何も移動しないかのどちらかだ。どちらかの当事者がリスクにさらされる窓口がないため、エスクローエージェントが橋渡しするものが何もない。詳細なメカニズムは[トークン化マーケットプレイスがエスクローを代替する方法](/ja/blog/how-tokenized-marketplaces-replace-escrow/)で解説しているが、フリッパーにとってのポイントはシンプルだ——すべての売却から手数料、遅延、そしてカウンターパーティが1つずつ排除される。

トークン化ドメインは標準的な NFT であるため、既存のインフラにリスティングすることもできる。[NFT として売る](/ja/blog/selling-domains-as-nfts/)ことで、ドメイン専用のマーケットプレイスと並んで、[最大級の NFT マーケットプレイスの1つに成長した](https://en.wikipedia.org/wiki/OpenSea#:~:text=one%20of%20the%20largest%20NFT%20marketplaces) OpenSea のような汎用マーケットプレイスにも出品できる。これらの場所のトレードオフはリスティング前によく検討する価値がある——[オンチェーンドメインマーケットプレイス比較](/ja/blog/onchain-domain-marketplaces-compared/)がその場所だ。実際の結果として得られるのは、より広い[流動性](/ja/glossary/domain-trading/)の表面積だ——1つの資産が複数の場所にリスティングでき、仲介者なしに決済される。

## プログラマブルな所有権：従来の仕組みに類似物がない部分

上記のすべてには、トークン化によってより速く、またはより安くなるレジストラ世界のアナログが存在する。最後のこれは違う。

ドメインが[スマートコントラクト](/ja/glossary/smart-contract/)資産であるため、所有権はプログラム可能になる。名前をローンの担保に使うことができ、コードによってルールが執行されるオンチェーンオークションで売却したり、複数の保有者に[分割](/ja/glossary/domain-trading/)したり、自動的に実行される条件でリースしたりできる。これらのパターンは従来の流通市場には存在しない——従来、ドメインはレジストラのデータベース上のエントリーで、購入、売却、またはどこかへの指定しかできなかった。単純な安く買って高く売る取引を超えて考えるフリッパーにとって、プログラマビリティは以前は弁護士や特注契約を用意できる人だけに利用可能だったファイナンスや構造化オプションを開く。

これはまた、採用曲線が最も初期の部分でもあるため、エキゾチックなユースケースは成熟したものではなく新興のものとして扱うべきだ。今日から利用可能な確実な成果は最初の4つだ——検証可能な取得、セルフカストディ、高速移転、エスクロー不要の決済。

## 変わらないこと

トークン化が過大評価されることもあるため、限界についてはっきり言う価値がある。フリッピングの難しい部分は依然として難しい。買う価値のある名前を探し出し、誠実に評価し、商標の罠を避け——そして何より——買い手を見つけなければならない。誰も欲しがらないトークン化ドメインは、誰も欲しがらないレジストラ保有のドメインと全く同じように売れない。[3,000万ドル](https://www.sidn.nl/en/news-and-blogs/voice-com-sold-for-usd-30-million#:~:text=blockchain%20provider%20Block.one%20paid%2030%20million%20US%20dollars%20for%20the%20domain%20name%20voice.com)という注目すべき `Voice.com` の売却は、それが決済されたインフラではなく、その名前への需要によるものだった。トークン化は需要を作り出さない。需要がすでに支えている取引から摩擦を取り除くのだ。

すでに `.com` を保有していて、実際の違いを体感したいなら、最もクリーンな入口は管理しているドメインをトークン化して、新しいインフラを通じて1回の売却を試してみることだ——手順については[.com のトークン化方法](/ja/blog/how-to-tokenize-your-com/)を参照し、どこで行うかを決める際は[ドメイントークン化プラットフォームの選び方](/ja/blog/choosing-a-domain-tokenization-platform/)を確認してほしい。[Namefi](https://namefi.io) のようなプラットフォームは DNS レイヤーを常に完全に機能させたまま保つため、名前はドメインとして機能し続けながら、上記で説明したオンチェーンの仕組みを得ることができる。
