अब तीन [रजिस्ट्रार](/hi/glossary/registrar/) ऐसे हैं जो चेकआउट फ़ॉर्म भरने का काम किसी इंसान के अलावा किसी और को भी करने देते हैं। Cloudflare ने अप्रैल 2026 में एक बीटा API शुरू किया, जिससे [AI एजेंट](/hi/glossary/ai-agent/) बिना ब्राउज़र सेशन के डोमेन रजिस्टर कर सकता है। Name.com ने इसी विचार के इर्द-गिर्द अपना API दोबारा बनाया और खुद को पहला AI-नेटिव डोमेन प्लेटफ़ॉर्म कहा। Namefi ने Model Context Protocol (MCP) सर्वर और वॉलेट-हस्ताक्षरित चेकआउट बनाया, जो अकाउंट बनाने के चरण को पूरी तरह हटा देता है। तीनों का लक्ष्य एक ही बदलाव है: डोमेन रजिस्ट्रेशन को ब्राउज़र में किसी व्यक्ति द्वारा किए जाने वाले काम से API कॉल के जरिए एजेंट द्वारा किए जाने वाले काम में बदलना।

फिर भी ये अलग-अलग लोगो वाले एक ही उत्पाद नहीं हैं। हर कंपनी ने कीमत, “एजेंट-नेटिव” होने की वास्तविक आवश्यकताओं और खरीदार द्वारा भुगतान क्षमता साबित करने के तरीके पर अलग दाँव लगाया है। यह तीनों की फ़ीचर-दर-फ़ीचर तुलना है। इसमें वे पहलू भी शामिल हैं जहाँ Cloudflare की कीमत को हराना वास्तव में कठिन है और जहाँ Name.com की बाज़ार स्थिति उसके अब तक उपलब्ध कराए गए उत्पाद से आगे है।

## “एजेंट-नेटिव” होने के लिए वास्तव में क्या चाहिए

API होना और एजेंट के उपयोग योग्य होना एक ही बात नहीं है। अधिकांश रजिस्ट्रार वर्षों से प्रोग्रामेटिक रजिस्ट्रेशन उपलब्ध कराते आए हैं, लेकिन वे इंटरफ़ेस ऐसे रीसेलर और डेवलपर के लिए बनाए गए थे जो दस्तावेज़ पढ़ते हैं। वे किसी ऐसे स्वायत्त प्रोसेस के लिए नहीं बने थे जिसे खुद पता लगाना हो कि क्या संभव है, किसी इंसान द्वारा पासवर्ड टाइप किए बिना प्रमाणित होना हो और किसी इंसान द्वारा पढ़े बिना त्रुटि संदेश को समझना हो। “API वाले” रजिस्ट्रार और एजेंट-नेटिव रजिस्ट्रार के बीच अंतर की पूरी चेकलिस्ट [एजेंट-नेटिव डोमेन रजिस्ट्रार क्या है?](/hi/blog/agent-native/) में दी गई है। संक्षेप में, इसके मुख्य तत्व हैं खोज-योग्यता (क्या एजेंट खुद API ढूँढ सकता है), मशीन-पठनीय प्रतिक्रियाएँ और ऐसी भुगतान प्रक्रिया जो यह न माने कि किसी इंसान के हाथ में क्रेडिट कार्ड है। नीचे दिए गए तीनों रजिस्ट्रार अलग-अलग स्तर तक इस मानक पर खरे उतरते हैं।

## Cloudflare Registrar API: लागत मूल्य, बीटा और पहले से आपके एडिटर में

Cloudflare का Registrar API कंपनी की “Agents Week” घोषणाओं के दौरान 15 अप्रैल 2026 को बीटा में आया। लॉन्च पर एक उद्योग रिपोर्ट के अनुसार, यह API [AI एजेंट को बिना किसी ब्राउज़र इंटरैक्शन या मैन्युअल मंज़ूरी के डोमेन की उपलब्धता खोजने, कीमत जाँचने और प्रोग्रामेटिक तरीके से रजिस्ट्रेशन पूरा करने देता है](https://webhosting.today/2026/04/22/ai-agents-can-now-register-domains-no-human-required/#:~:text=lets%20an%20AI%20agent%20search%20for%20domain%20availability%2C%20check%20pricing%2C%20and%20complete%20registration%20programmatically%20without%20any%20browser%20interaction%20or%20manual%20approval)। सामान्य डोमेन का रजिस्ट्रेशन कुछ सेकंड के भीतर सिंक्रोनस रूप से पूरा हो जाता है। API को [Cursor और Claude Code जैसे MCP समर्थित कोड एडिटर](https://webhosting.today/2026/04/22/ai-agents-can-now-register-domains-no-human-required/#:~:text=code%20editors%20with%20MCP%20support%20such%20as%20Cursor%20and%20Claude%20Code) के भीतर इस्तेमाल करने के लिए बनाया गया है। डेवलपर जिस टूल में प्रोजेक्ट बना रहा है, उसे छोड़े बिना उसी प्रोजेक्ट के लिए डोमेन रजिस्टर कर सकता है।

Cloudflare की पेशकश का सबसे मजबूत पक्ष कीमत है। विश्वसनीय तुलना के लिए इस वास्तविक लाभ को स्वीकार करना ज़रूरी है: Cloudflare [.ai डोमेन का रजिस्ट्रेशन और नवीनीकरण होलसेल कीमतों पर, बिना किसी अतिरिक्त मार्कअप के देता है](https://www.cloudflare.com/application-services/products/registrar/buy-ai-domains/#:~:text=Cloudflare%20offers%20.ai%20domain%20registrations%20and%20renewals%20at%20wholesale%20prices%2C%20with%20no%20additional%20markups)। हर रजिस्टर्ड डोमेन के साथ [मुफ़्त DNSSEC, मुफ़्त SSL, टू-फ़ैक्टर ऑथेंटिकेशन और डिफ़ॉल्ट रूप से सक्षम डोमेन लॉक](https://www.cloudflare.com/application-services/products/registrar/buy-ai-domains/#:~:text=free%20DNSSEC%2C%20free%20SSL%2C%20two-factor%20authentication%2C%20and%20a%20domain%20lock%20enabled%20by%20default) मिलता है, साथ ही [मुफ़्त WHOIS रिडैक्शन](https://www.cloudflare.com/application-services/products/registrar/buy-ai-domains/#:~:text=every%20.ai%20domain%20comes%20with%20free%20WHOIS%20redaction) भी मिलता है। यानी उस [WHOIS गोपनीयता](/hi/glossary/whois-privacy/) सुरक्षा के लिए कोई अतिरिक्त शुल्क नहीं, जिसे दूसरे रजिस्ट्रार ऐड-ऑन के रूप में बेचते हैं। रजिस्ट्रार की एक अलग समीक्षा भी स्वतंत्र रूप से इस मूल्य मॉडल की पुष्टि करती है: Cloudflare की [लागत-मूल्य नीति में आप केवल उतना चुकाते हैं जितना Cloudflare चुकाता है; रजिस्ट्रेशन या नवीनीकरण पर कोई मार्कअप नहीं होता](https://www.hostinger.com/tutorials/best-domain-registrars/#:~:text=At-cost%20pricing%20charges%20you%20only%20what%20Cloudflare%20pays%2C%20with%20no%20markup%20at%20registration%20or%20renewal)। यदि कीमत निर्णायक कारक है और आपको “इसे रजिस्टर करें और सुरक्षित रूप से लॉक करें” से आगे कुछ नहीं चाहिए, तो Cloudflare को हराना कठिन है।

सीमा इसके दायरे में है। बीटा में खोज, कीमत जाँच और रजिस्ट्रेशन शामिल हैं। [Cloudflare ने कहा है कि लाइफ़साइकल प्रबंधन विकास में है और इसे 2026 में बाद में जारी करने की योजना है](https://webhosting.today/2026/04/22/ai-agents-can-now-register-domains-no-human-required/#:~:text=Cloudflare%20has%20stated%20that%20lifecycle%20management%20is%20in%20development%20and%20is%20planned%20for%20release%20later%20in%202026)। इसका अर्थ है कि ट्रांसफर, नवीनीकरण और संपर्क विवरण अपडेट अभी एजेंट के लिए उपलब्ध API का हिस्सा नहीं हैं। क्रिप्टो भुगतान का कोई विकल्प नहीं है और टोकनाइज़्ड स्वामित्व भी नहीं है। Cloudflare के जरिए रजिस्टर्ड डोमेन एक पारंपरिक रजिस्ट्रार अकाउंट की संपत्ति है, ऐसी संपत्ति नहीं जिसे वॉलेट सीधे रख सके।

## Name.com का AI-नेटिव API: प्राकृतिक भाषा से काम करने वाले कोड तक

Name.com की पेशकश Cloudflare से अलग है। कीमत को प्राथमिकता देने के बजाय, Name.com ने [नए name.com API के लॉन्च के इर्द-गिर्द अपना डेवलपर API दोबारा बनाया, जिसे वह एजेंटिक AI के युग के लिए डोमेन को आधुनिक बनाने वाला अपना AI-नेटिव प्लेटफ़ॉर्म कहता है](https://www.name.com/blog/the-first-ai-native-domain-platform#:~:text=the%20launch%20of%20the%20new%20name.com%20API%2C%20our%20AI-native%20platform%20that%20modernizes%20domains%20for%20the%20age%20of%20agentic%20AI)। यह [Model Context Protocol (MCP) और OpenAPI विनिर्देश पर बना है, जो AI एजेंट को डोमेन ऑपरेशनों से सीधे इंटरैक्ट करने देते हैं](https://www.name.com/blog/the-first-ai-native-domain-platform#:~:text=supported%20by%20modern%20standards%20like%20Model%20Context%20Protocol%20%28MCP%29%20and%20OpenAPI%20specification%2C%20which%20enable%20AI%20agents%20to%20interact%20directly%20with%20domain)। कंपनी इसे साफ़ तौर पर एडिटर के भीतर चलने वाले वर्कफ़्लो के रूप में भी पेश करती है। उसके अनुसार, डेवलपर [MCP समर्थन की मदद से Claude और Cursor जैसे AI टूल को साधारण प्रॉम्प्ट देकर डोमेन ऑपरेशन संभाल सकते हैं](https://www.name.com/blog/the-first-ai-native-domain-platform#:~:text=Leverage%20AI%20tools%20like%20Claude%20and%20Cursor%20to%20handle%20domain%20operations%20through%20simple%20prompts%2C%20thanks%20to%20MCP%20support)।

Name.com की घोषणा में सबसे स्पष्ट अंतर प्राकृतिक भाषा से कोड बनाने का ढाँचा है। एजेंट द्वारा निश्चित एंडपॉइंट का एक सेट कॉल करने के बजाय, प्रस्ताव यह है कि आप एजेंट से कहें, “मेरे ऐप में डोमेन रजिस्ट्रेशन जोड़ो,” और एजेंट API दस्तावेज़ का उपयोग करके इंटीग्रेशन कोड खुद लिखे। Name.com ने “दुनिया इसी दिशा में जा रही है” वाले तर्क को अपने ग्राहक शोध से समर्थन दिया है। कंपनी के अनुसार, [91% उत्तरदाता मानते हैं कि अगले दो वर्षों में AI एजेंट उनके डोमेन प्रबंधन का कम से कम कुछ हिस्सा संभालेंगे](https://www.name.com/blog/the-first-ai-native-domain-platform#:~:text=a%20remarkable%2091%25%20of%20respondents%20envision%20AI%20agents%20handling%20at%20least%20some%20of%20their%20domain%20management%20in%20the%20next%20two%20years)। यह आँकड़ा किसी तीसरे पक्ष के बजाय सीधे Name.com की अपनी घोषणा से आता है, इसलिए इसे स्वतंत्र सर्वेक्षण नहीं बल्कि कंपनी द्वारा रिपोर्ट की गई बाज़ार धारणा मानें।

दो बातों को स्पष्ट रूप से बताना ज़रूरी है। पहली, Name.com का ब्लॉग पोस्ट पोज़िशनिंग और विज़न का लेख है। वह Cloudflare और Namefi के दस्तावेज़ों जैसी मदवार क्षमता तालिका प्रकाशित नहीं करता, इसलिए नीचे मैट्रिक्स की कई प्रविष्टियाँ परीक्षित विनिर्देश के बजाय घोषणा के दावों को दर्शाती हैं। दूसरी, कीमत के बारे में Name.com की अपनी पोस्ट रीसेलर-पक्ष की सुविधा—[अपने मार्कअप खुद तय करने की क्षमता](https://www.name.com/blog/the-first-ai-native-domain-platform#:~:text=the%20ability%20to%20set%20your%20own%20markups)—की बात करती है। यह रीसेलर-पार्टनर फ़ीचर है, अंतिम उपयोगकर्ताओं के लिए Cloudflare जैसा लागत-मूल्य का वादा नहीं। घोषणा में क्रिप्टो भुगतान का रास्ता या टोकनाइज़्ड स्वामित्व भी नहीं बताया गया है।

## Namefi: MCP सर्वर, वॉलेट चेकआउट और टोकनाइज़्ड स्वामित्व

Namefi का दृष्टिकोण एक अलग धारणा से शुरू होता है: खरीदार शायद ब्राउज़र सेशन या क्रेडिट कार्ड वाला इंसान न हो और कार्रवाई करने से पहले Namefi अकाउंट भी न बनाना चाहे। Namefi के अपने मशीन-पठनीय API दस्तावेज़ के अनुसार—जो उसके उत्पाद संबंधी दावों के लिए एकमात्र प्रामाणिक स्रोत है—Namefi Streamable HTTP ट्रांसपोर्ट पर `https://api.namefi.io/mcp` में MCP सर्वर चलाता है। यह “हर `/v-next` ऑपरेशन को टाइप्ड टूल (खोज, रजिस्ट्रेशन, DNS, डोमेन कॉन्फ़िगरेशन, आउटबाउंड) के रूप में” उपलब्ध कराता है, जिसे `https://namefi.io/.well-known/mcp/servers.json` पर खोजा जा सकता है। Claude Code के लिए एक पंक्ति की सेटअप कमांड भी दस्तावेज़ित है (`claude mcp add --transport http namefi https://api.namefi.io/mcp --header "x-api-key: YOUR_KEY"`)। REST API का प्रमाणीकरण डोमेन के मालिक वॉलेट से जुड़ा `x-api-key` हेडर उपयोग करता है, जबकि केवल पढ़ने वाले टूल को किसी कुंजी की आवश्यकता नहीं होती।

विशिष्ट अंतर भुगतान है। Namefi एक [x402](https://x402.org) भुगतान प्रवाह का दस्तावेज़ देता है, जिससे एजेंट पहले Namefi अकाउंट बनाए बिना स्टेबलकॉइन USDC से डोमेन खरीद सकता है। खरीदार का वॉलेट EIP-3009 `transferWithAuthorization` पर हस्ताक्षर करता है; यदि भुगतान संलग्न नहीं है तो API कीमत के साथ `402 Payment Required` प्रतिक्रिया देता है; और वैध भुगतान हेडर आने पर रजिस्ट्रेशन का निपटान करता है। एक अलग Machine Payable Protocol (MPP) प्रवाह भी इसी तरह चुनौती और हस्ताक्षर का तरीका देता है। Cloudflare और Name.com में से किसी ने भी तुलनीय प्रक्रिया का दस्तावेज़ नहीं दिया है। इस तुलना में यही सबसे स्पष्ट अंतर है। इस चेकआउट प्रवाह के शुरू से अंत तक काम करने का तरीका जानने के लिए [क्रिप्टो वॉलेट से डोमेन का भुगतान करें: अकाउंट की आवश्यकता नहीं](/hi/blog/wallet-checkout/) देखें।

Namefi डोमेन को [NFT](/hi/glossary/nft/) के रूप में भी रजिस्टर करता है—ऐसे [टोकनाइज़्ड डोमेन](/hi/glossary/tokenized-domain/) जिनका स्वामित्व केवल रजिस्ट्रार के आंतरिक डेटाबेस में दर्ज होने के बजाय ऑन-चेन सत्यापित होता है। इसके DNS विकल्पों में ऑटो-[ENS](/hi/glossary/ens/) रिकॉर्ड और [DNSSEC](/hi/glossary/dnssec/) शामिल हैं। साथ ही एकल और बैच में पूर्ण CRUD DNS रिकॉर्ड प्रबंधन, ऑटो-रिन्यूअल, डोमेन पार्किंग और फ़ॉरवर्डिंग भी उपलब्ध हैं। Namefi का llms.txt किसी घोषित मूल्य नीति को प्रकाशित नहीं करता। Cloudflare के समान “लागत-मूल्य” का कोई दावा नहीं है और इस लेख के लिए देखे गए दस्तावेज़ों में सार्वजनिक मूल्य सूची भी दिखाई नहीं देती। इसलिए कीमत में Cloudflare के बराबर होने का अनुमान लगाने के बजाय namefi.io पर मौजूदा कीमत सीधे जाँचें। <!-- TODO: confirm with team — Namefi's published pricing/markup policy relative to registry cost -->

## किस स्थिति में कौन बेहतर है

यदि कीमत और सरलता निर्णायक कारक हैं और आपको नाम रजिस्टर करके सुरक्षित रूप से लॉक करने से आगे कुछ नहीं चाहिए, तो **Cloudflare** चुनें। इसकी लागत-मूल्य नीति और डिफ़ॉल्ट रूप से शामिल सुरक्षा सुविधाएँ (DNSSEC, WHOIS रिडैक्शन, टू-फ़ैक्टर ऑथेंटिकेशन) वास्तव में उन सुरक्षा उपायों से बेहतर हैं जिनके लिए अधिकांश स्थापित कंपनियाँ शुल्क लेती हैं। यदि आप पहले से Cursor या Claude Code के भीतर Cloudflare के स्टैक पर काम कर रहे हैं, तो वर्कफ़्लो निर्बाध है। स्पष्ट समझौता इसका दायरा है: DNS प्रबंधन नहीं, नवीनीकरण स्वचालन नहीं और अभी क्रिप्टो या टोकनाइज़्ड विकल्प भी नहीं, क्योंकि बीटा केवल रजिस्ट्रेशन तक सीमित है।

यदि आप ऐसा एजेंट चाहते हैं जो निश्चित API को कॉल करने के बजाय आपके लिए इंटीग्रेशन कोड लिखे, या आप पहले से Name.com रीसेलर हैं और आधुनिक MCP-संगत प्लेटफ़ॉर्म पर मार्कअप में लचीलापन चाहते हैं, तो **Name.com** चुनें। वास्तव में उपलब्ध फ़ीचर और रोडमैप के बीच अंतर पर इसके दस्तावेज़ Cloudflare या Namefi की तुलना में कम विस्तृत हैं। इसलिए मार्केटिंग के दावों के सामने वास्तविक API दायरे को परखने के लिए समय रखें।

यदि खरीदार सच में एजेंट-प्रथम है, तो **Namefi** चुनें: किसी इंसानी अकाउंट की आवश्यकता नहीं, सहेजे गए कार्ड के बजाय वॉलेट हस्ताक्षर से अधिकृत भुगतान और ऐसा स्वामित्व जिसे रजिस्ट्रार डेटाबेस की एक पंक्ति के बजाय ऑन-चेन, ट्रांसफर किए जा सकने वाले टोकन के रूप में दर्शाना हो। MCP सर्वर, पूर्ण DNS नियंत्रण, ऑटो-ENS और वॉलेट-नेटिव चेकआउट का यह संयोजन फिलहाल Cloudflare के बीटा या Name.com की घोषणा में उपलब्ध नहीं है। समझौता यह है कि Namefi ने Cloudflare जैसी लागत-मूल्य प्रतिबद्धता प्रकाशित नहीं की है। इसलिए यदि होलसेल कीमत आपकी सर्वोच्च प्राथमिकता है, तो यह मानने से पहले कि Namefi Cloudflare से सस्ता है, Namefi की मौजूदा कीमत सीधे सत्यापित करें।

कई टीमें अंततः एक से अधिक विकल्प इस्तेमाल करेंगी: जिस इंफ्रास्ट्रक्चर को वे पहले से Cloudflare या Name.com पर चलाती हैं, उसके सामने वाले डोमेन के लिए उन्हीं का उपयोग; और ऐसी किसी भी चीज़ के लिए Namefi जैसे वॉलेट-नेटिव रजिस्ट्रार का उपयोग जिसे ऑन-चेन रखना और लेन-देन करना हो। यह मार्केटप्लेस में ट्रेड होने वाला नाम हो सकता है या किसी व्यक्ति के अकाउंट के बजाय एजेंट के अपने वॉलेट के स्वामित्व वाला नाम। जब [रजिस्ट्रेंट](/hi/glossary/registrant/) इंसान के बजाय एजेंट हो, तब “स्वामित्व” का अर्थ क्या है, यह अपने आप में गहरे लेख का विषय है। देखें: [क्या AI एजेंट किसी डोमेन का मालिक हो सकता है? WHOIS, कस्टडी और टोकन](/hi/blog/agent-own-domain/)।

## अक्सर पूछे जाने वाले प्रश्न

### AI एजेंट के उपयोग के लिए कौन-सा रजिस्ट्रार सबसे सस्ता है?

इन तीनों में केवल Cloudflare स्पष्ट रूप से लागत-मूल्य और बिना मार्कअप की प्रतिबद्धता प्रकाशित करता है, जिसकी पुष्टि रजिस्ट्रार की एक स्वतंत्र समीक्षा भी करती है। Name.com की घोषणा अंतिम उपयोगकर्ताओं के लिए लागत-मूल्य का वादा करने के बजाय रीसेलर को मार्कअप में लचीलापन देने की बात करती है। Namefi ने अपने API दस्तावेज़ में मूल्य नीति प्रकाशित नहीं की है। इसलिए हर प्लेटफ़ॉर्म की मौजूदा कीमत जाँचे बिना फिलहाल सीधी मूल्य तुलना संभव नहीं है।

### क्या इनमें से कोई एजेंट को इंसान के क्रेडिट कार्ड के बिना भुगतान करने देता है?

दस्तावेज़ित क्रिप्टो-नेटिव भुगतान प्रवाह केवल Namefi के पास है। एजेंट का वॉलेट Namefi अकाउंट बनाए बिना x402 प्रोटोकॉल के जरिए USDC में भुगतान कर सकता है, या अलग Machine Payable Protocol की चुनौती और हस्ताक्षर प्रक्रिया इस्तेमाल कर सकता है। Cloudflare का बीटा और Name.com का API, दोनों किसी तुलनीय अकाउंट-रहित भुगतान प्रक्रिया का दस्तावेज़ नहीं देते।

### क्या इन API से केवल डोमेन रजिस्टर करने के बजाय DNS रिकॉर्ड भी प्रबंधित किए जा सकते हैं?

Namefi के दस्तावेज़ पूर्ण DNS रिकॉर्ड CRUD को कवर करते हैं, जिसमें बैच में बनाना/अपडेट करना/हटाना और पार्किंग, फ़ॉरवर्डिंग, ऑटो-ENS तथा Vercel anycast रिकॉर्ड के विकल्प शामिल हैं। इस लेख के समय Cloudflare Registrar API का बीटा केवल रजिस्ट्रेशन के लिए है। लाइफ़साइकल और रजिस्ट्रेशन के बाद का प्रबंधन, DNS सहित, बाद में जारी करने की योजना है। Name.com की घोषणा DNS प्रबंधन क्षमताओं का मदवार विवरण नहीं देती।

### क्या Cloudflare का Registrar API अभी सामान्य रूप से उपलब्ध है?

नहीं। यह Cloudflare की “Agents Week” के दौरान 15 अप्रैल 2026 को बीटा में आया। Cloudflare ने कहा है कि व्यापक लाइफ़साइकल प्रबंधन—ट्रांसफर, नवीनीकरण और संपर्क अपडेट—अब भी विकास में है और 2026 में बाद के लिए नियोजित है। बीटा चरण की क्षमताएँ बदल सकती हैं, इसलिए प्रोडक्शन में उन पर निर्भर होने से पहले दोबारा सत्यापित करें।

### “एजेंट-नेटिव” का क्या अर्थ है और क्या तीनों इस श्रेणी में आते हैं?

एजेंट-नेटिव का अर्थ है कि एजेंट API खोज सके, प्रमाणित हो सके और किसी इंसान द्वारा ब्राउज़र फ़ॉर्म भरे बिना खरीद पूरी कर सके। पूरी चेकलिस्ट के लिए [एजेंट-नेटिव डोमेन रजिस्ट्रार क्या है?](/hi/blog/agent-native/) देखें। यहाँ तीनों रजिस्ट्रार बुनियादी मानक—प्रोग्रामेटिक खोज से खरीद तक और MCP या MCP से सटे टूल—पूरा करते हैं। लेकिन रजिस्ट्रेशन से आगे एजेंट-नेटिव डिज़ाइन कितनी दूर जाता है, इसमें बड़ा अंतर है: DNS, नवीनीकरण, भुगतान विधि और स्वामित्व मॉडल।
