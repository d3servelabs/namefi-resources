La **transferencia entre registradores** es el proceso de mover un dominio de un [registrador](/es/glossary/registrar/) a otro, que tradicionalmente requiere códigos de autenticación, períodos de espera y coordinación entre [registradores](/es/glossary/registrar/). Este proceso puede tardar días o semanas y puede implicar tarifas de ambos [registradores](/es/glossary/registrar/). Con los dominios [tokenizados](/es/glossary/tokenize/), si bien el registro [DNS](/es/glossary/dns/) subyacente aún puede requerir procesos de transferencia tradicionales, el token de propiedad puede transferirse instantáneamente entre [billeteras](/es/glossary/wallet/). Esta separación permite el comercio inmediato y los cambios de propiedad a nivel de token, mientras que la gestión de [DNS](/es/glossary/dns/) puede manejarse por separado. Los modelos avanzados de tokenización incluso podrían automatizar las transferencias entre registradores a través de [contratos inteligentes](/es/glossary/smart-contract/), reduciendo significativamente la complejidad y el tiempo necesarios para mover dominios entre diferentes proveedores de servicios.
