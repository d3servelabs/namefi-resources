**x402** es un protocolo abierto que revive el código de estado **HTTP 402 "Pago Requerido"** — largo tiempo dormido — como una forma estructurada para que servidores, agentes y clientes soliciten y paguen por pagos en cadena de manera integrada con las interacciones HTTP normales. Cuando un recurso está protegido por x402, una respuesta `402` contiene el precio, los activos aceptados (típicamente [stablecoins](/es/glossary/stablecoin/)), la dirección del destinatario y la URL del facilitador; el cliente paga [en cadena](/es/glossary/on-chain/) y reintenta la solicitud con prueba de pago. x402 es relevante para los [dominios tokenizados](/es/blog/what-are-tokenized-domains/) porque permite que los agentes y aplicaciones que usan un dominio como identidad cobren pagos de forma limpia — y porque la [billetera](/es/glossary/wallet/) controladora de un dominio tokenizado funciona también como punto de pago. Consulta [Route 402](/es/blog/route402-x402-facilitator-router/) para la explicación más detallada. Sitio oficial: [x402.org](https://www.x402.org/).
