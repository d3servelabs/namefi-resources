A **reserve price** is the confidential minimum sale price set by a domain seller, below which no [auction](/en/glossary/auction/) bid or negotiated offer will result in a completed transaction. In a domain auction, if bidding closes below the reserve, the name is not awarded to the highest bidder — it simply goes unsold. Most major auction platforms (GoDaddy Auctions, Sedo, NameJet) allow sellers to set a hidden reserve, so bidders see the current bid but not whether it has met the floor. A reserve that is set too high relative to market value leads to repeated no-sales and erodes credibility; a reserve set too low risks leaving money on the table. In private [make-offer](/en/glossary/make-offer/) listings, the reserve functions as the seller's internal walk-away number during negotiation, often communicated to a [domain broker](/en/glossary/domain-broker/) but not disclosed to the buyer.
