Un **titulaire de domaine** est la personne ou l'organisation qui détient les droits sur un domaine enregistré — le *titulaire de nom enregistré* figurant au registre. Le titulaire contracte avec un [registraire](/fr/glossary/registrar/), qui travaille à son tour avec le [registre](/fr/glossary/registry/) selon les règles de l'[ICANN](/fr/glossary/icann/) ; le titulaire est ce qui se rapproche le plus d'un « propriétaire » dans le système traditionnel, bien que ces droits résident dans un compte de registraire plutôt que dans quelque chose que vous détenez directement. C'est précisément ce que la tokenisation change : lorsqu'un domaine est tokenisé, le contrôle passe dans un [portefeuille](/fr/glossary/wallet/) sous la forme d'une couche de propriété [on-chain](/fr/glossary/on-chain/), de sorte que les droits du titulaire deviennent un actif transférable que vous gérez vous-même plutôt qu'une ligne dans la base de données d'un registraire.
