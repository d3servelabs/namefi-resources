**.site** ドメインは、インターネット上でもっとも直感的な拡張子の一つです。「site」という単語は英語で「ウェブサイト」を意味し、多くの言語でも広く認識されているため、.site アドレスを見ればそのページが何であるかが一目でわかります。ICANN の[新 gTLD](/ja/glossary/new-gtld/) プログラムのもとで開始された[ジェネリックトップレベルドメイン](/ja/glossary/gtld/)（gTLD）であり、資格制限なく世界中の誰でも登録できます。

理想のドメイン名の .com がすでに取得済みのファウンダー、インディー開発者、ブランドオーナーにとって、.site は短くわかりやすいアドレスを実現する実用的な代替手段です。本ページでは、運営者、利用状況、SEO およびメール到達性のプロファイル、登録前に検討すべき点を詳しく解説します。

## .site の概要

| 項目 | 詳細 |
| --- | --- |
| TLD 種別 | ジェネリックトップレベルドメイン（新 gTLD） |
| レジストリ運営者 | Radix（Radix Technologies Inc. SEZC） |
| 開始年 | 2015 年 |
| IDN サポート | あり |
| DNSSEC | 対応 |
| 登録制限 | なし — 資格証明書、事業実態、現地拠点不要 |
| 最適な用途 | ウェブサイト、ポートフォリオ、ランディングページ、小規模事業、マイクロサイト |

## .site とは？

.site は、[ICANN の新 gTLD プログラム](https://newgtlds.icann.org/)の一環として 2015 年に DNS ルートへ委任されたジェネリックトップレベルドメインです。このプログラムは 2012 年のラウンドで始まり、ネームスペースに数百の新しい拡張子を追加しました。[.io](/ja/tld/io/) や .de のような国コード TLD とは異なり、.site に地理的な意味はなく、完全にジェネリックでグローバルを対象としています。

この違いは国際的なプロジェクトにとって重要です。[Google Search Central](https://developers.google.com/search/docs/crawling-indexing/managing-multi-regional-sites) によると、.site のような新 gTLD はデフォルトでグローバルとして扱われ、[ccTLD](/ja/glossary/cctld/) のように特定の国に地理的に限定されることはありません。[IANA の .site ルートゾーンレコード](https://www.iana.org/domains/root/db/site.html)には、現在の運営者として Radix が記載され、委任履歴も確認できます。

## .site の歴史

.site は 2015 年に委任され、同年に一般登録が開始されました。現在はレジストリとして Radix が運営しており、同社は .online、.store、.tech、[.fun](/ja/tld/fun/) など単語ベースの新 gTLD のポートフォリオを保有しています。「site」はウェブ全般に通じる普遍的な言葉であるため、特定のニッチに依存することなく、開始当初から安定した需要を集めました。

普及は順調に進み、2019 年 2 月にはレジストリが登録数 100 万件の突破を報告しています。以来、.site は新 gTLD の中でも登録数の多い拡張子の一つであり続けています。Radix の報告によれば、.site 登録の大多数は中小企業によるもので、電子商取引と情報技術分野への集中が見られます。これは「これはウェブサイトです」というシンプルなコンセプトと一致するプロファイルです。

## .site の活用方法

.site は意図的にジェネリックな設計のため、実際の用途も特定の業種に限らず幅広いです。

- **中小企業・地域ビジネスのサイト** — .com が取得済みの場合に、サービス名と組み合わせたわかりやすいアドレスとして活用。
- **ポートフォリオ・個人ページ** — `firstnamelastname.site` のパターンでデザイナー、ライター、フリーランサーが利用。
- **ランディングページ・マイクロサイト** — URL が「ウェブサイト」として伝わればよい、単一目的のキャンペーン、製品、イベントページ。
- **ドキュメント・プロジェクトサイト** — オープンソースツールや個人プロジェクトの公開ページを立ち上げる開発者向け。
- **ブランド防衛目的の登録** — 主要ドメインと並んで[商標](/ja/glossary/trademark/)の .site バージョンを確保する企業。

**向いていないケース:** 視聴者が .com を期待する大企業のフラッグシップサイト、または特定の国を意識したサイト（そのような場合は ccTLD が適切です）。テックブランドやアプリ系製品を構築する場合は、[.app](/ja/tld/app)、[.dev](/ja/tld/dev)、[.tech](/ja/tld/tech) のようなより専門的な拡張子の方がカテゴリをより明確に伝えられます。

## .site を使用している主なサイト

.site は有名ブランドのフラッグシップではなく、独立系ビジネス、クリエイター、個人プロジェクトによる利用が圧倒的に多いです。その強みはごく少数の有名事例ではなく、幅広い採用にあります。Radix の報告によれば、.site 登録の約 70% は電子商取引と IT 分野の中小企業によるもので、まさにこの拡張子が命名された「日常のウェブサイト、ストア、ポートフォリオ」を体現しています。オーナーが変わりうる特定の著名事例を挙げるより、「このアドレスはウェブサイトです」とひとめで伝わるサイトが無数に存在しているというのが正直な姿です。

## .site と他のドメインの比較

| 拡張子 | 種別 | 意味 | 主な用途 |
| --- | --- | --- | --- |
| [.site](/ja/tld/site) | 新 gTLD | 文字どおり「ウェブサイト」 | 汎用ウェブサイト、中小企業、ポートフォリオ |
| [.com](/ja/tld/com) | レガシー gTLD | 商業 / デフォルト | 最も認知度の高いデフォルト拡張子 |
| [.online](/ja/tld/online) | 新 gTLD | オンラインであること | .com に代わる幅広いブランダブルな選択肢 |
| [.store](/ja/tld/store) | 新 gTLD | 店舗 | 電子商取引・小売に特化 |

最高の知名度が重要でドメイン名が手頃なら[.com](/ja/tld/com) を選んでください。.com が取得できない場合に短くわかりやすいアドレスを求めるなら .site が有力です。プロジェクトが明確に「ショップ」であれば [.store](/ja/tld/store) を、別の幅広い汎用オプションと比較したいなら [.online](/ja/tld/online) を検討してください。

## .site を選ぶ理由

- **直感的に理解できる** — 「site」はウェブサイトを意味する言葉として世界中で通じ、説明不要です。
- **取得しやすい** — .com ではとっくに消えてしまった短い 1〜2 語の名前が、.site ではまだ登録可能なケースが多いです。
- **制限なし** — 誰でも登録可能で、書類手続き、資格、現地拠点は不要です。
- **地理的にニュートラル** — ジェネリック gTLD として国のバイアスがなく、国際的な対象者に適しています。
- **標準規格対応** — レジストリが DNSSEC と国際化ドメイン名（IDN）をサポートしています。

## 検討すべき点

.site がすべてのプロジェクトに最適というわけではありません。低価格なオープン新 gTLD であるため、保守的な層には「本物の」サイトは .com にあるという先入観が残っており、.site アドレスは老舗感に欠けると受け取られることがあります。低コストなオープン拡張子と同様に一部のスパムや悪用事例があり、まったく知らないドメインからのメールを受け取るフィルターやユーザーが新しいドメインを慎重に扱うケースも稀にあります。また、新 gTLD 全般で一般的な初年度プロモーション価格と更新価格が異なる場合があるため、更新費用の予算を確保しておく必要があります。これらは致命的な欠点ではありませんが、拡張子の明確さや取得しやすさと合わせて検討する価値があります。

## .site ドメインは誰が登録できますか？

**登録制限: なし。** .site には資格要件がありません。[.law](/ja/tld/law/) や [.cpa](/ja/tld/cpa/) のような資格証明の壁もなく、会員要件も現地拠点要件もありません。世界中の個人・法人が先着順で利用可能な .site 名を登録できます。

新 gTLD の標準的なセーフガードは引き続き適用されます。開始時にはサンライズ期間が設けられ、商標権者は[商標クリアリングハウス](/ja/glossary/tmch/)を通じて一致する名前を優先的に取得できました。通常の予約名・プレミアム名リストも適用されます。ドメイン名は標準的な文字数ルールに従い、レジストリは非ラテン文字スクリプト向けの IDN もサポートしています。運用面では、DNSSEC がサポートされ、[WHOIS プライバシー](/ja/glossary/whois-privacy/)はレジストラを通じて一般的に利用でき、移管・更新・救済猶予期間の処理は ICANN の標準的な gTLD ライフサイクルに従います。権威ある規則は [.site の ICANN レジストリ契約](https://www.icann.org/en/registry-agreements/details/site)と [Radix レジストリポリシー](https://radix.website/policies)に記載されています。

## .site の価格と価値

このページでは具体的な価格は記載しませんが、価格の仕組みを理解しておくことが重要です。多くの新 gTLD と同様に、.site には標準ティアと**プレミアムティア**があります。短い名前、一般的な名前、高需要のキーワード名はプレミアムに分類され、価格が高く設定されており、更新価格も高いまま続くことが多いです。標準の名前ははるかに手頃です。

**初年度価格と更新価格は異なる場合があります** — レジストラでは初年度プロモーションが一般的ですが、更新時には標準価格が適用されます。主なコスト要因は、プレミアム名かどうか、登録期間、プライバシー保護などのオプションです。長期利用を考えているドメインについては、初年度の特別価格だけでなく、必ず更新価格も確認してください。

## 評判とメール到達性

.site は、プレミアムや「テック系」というよりも、明確で実用的、コスト効率の高い拡張子として広く認識されています。低価格でオープンであるため、多くの低コスト新 gTLD と同様に一部のスパマーに利用される事例があり、特に保守的なメールフィルターは未知の新 gTLD の送信者を老舗 .com 送信者より慎重にスコアリングすることがあります。

実際には、これは十分に対処可能な問題です。到達性を左右するのは、拡張子そのものよりもドメインの設定と送信履歴のほうがはるかに大きいです。**SPF、DKIM、DMARC** を正しく設定し、新しい送信ドメインを段階的に育てて、リストを清潔に保ち、送信者レピュテーションを監視すれば、適切に運用された .site ドメインは確実に受信トレイに届きます。拡張子がメール到達性の根本的な障壁になることはありません。

## ブランディングと命名のヒント

.site の魅力はその直接性にあります。説明的な言葉と拡張子を組み合わせてアドレス全体が一つのフレーズとして読めるようにしましょう（たとえば、業種やサービス名 + `.site`）。これにより、.com の同等ドメインが取得できない場合に最大の効果を発揮できます。「site」という単語は明確ですが、その前に置く部分も同様に、短くて声に出しても伝わりやすいものにしてください。ハイフンや数字はできるだけ避けましょう。記憶しづらく、聞き間違えられやすいためです。また、「site」は英語で「cite（引用する）」や「sight（視覚）」と同音のため、音声や放送でフルドメインを伝える際は混乱を避けるためにはっきりと発音してください。

## よくある質問

### .site ドメインは誰でも登録できますか？

はい。.site 拡張子は、資格要件のないオープンなジェネリックトップレベルドメインです。資格証明書、商標、事業実態、現地拠点のいずれも不要で、世界中の個人・法人が先着順で利用可能な .site 名を登録できます。

### .site ドメインは SEO に影響しますか？

いいえ。Google は .site を .com を含む他のジェネリックトップレベルドメインと同様に扱います。拡張子自体に順位の優位性も不利もなく、ランキングを決めるのはコンテンツの品質、リンク、ユーザー体験です。新 gTLD が Google 検索で軽視されることはありません。

### .site ドメインはどんな人に向いていますか？

理想のドメイン名の .com が取得済みや高額な場合に、短くわかりやすい URL を求める方に最適です。小規模事業者、ポートフォリオサイト、ランディングページ、個人プロジェクト、キャンペーン用マイクロサイトに向いています。「site」という単語がウェブサイトを直接指す言葉であるため、広く理解され、覚えやすいという利点があります。

### .site はスパムと関連していますか？

.site を含む低コストのオープン新 gTLD にはある程度の悪用事例があるため、一部の厳しいスパムフィルターは未知の新 gTLD 送信者をより慎重にスコアリングすることがあります。SPF、DKIM、DMARC を適切に設定し、送信レピュテーションを段階的に育てれば、安定したメール到達性を確保できます。
