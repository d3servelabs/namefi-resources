Toggle Visual Hierarchy

Exploring toggle styles that communicate state importance — from subtle defaults to attention-grabbing warnings

📋

Design Requirements

Problem Current toggles treat all states equally. Users don't recognize when a setting is in a risky state (e.g., Auto Renew OFF) vs. a safe default state.
Goals Create a visual hierarchy where safe/default states are calm and subtle, while risky states immediately draw attention without being overwhelming.
Constraints Must work in dark theme. Toggle size should remain consistent (44-64px width). Must be accessible and clearly indicate ON/OFF state.
Success Criteria Users can instantly identify which domains need attention. Safe states don't create visual noise. Warning states are noticeable but not alarming.

✨ Recommended Approach

Our proposed solution: Use "Red OFF Warning" toggle that shows red when OFF (danger) and green when ON (safe). Try toggling below to see the interaction.

Auto Renew: OFF (Risky) ⚠️ Needs Attention
Auto Renew: ON (Safe) ✓ Protected

The toggle is red when OFF to immediately signal risk, and green when ON to indicate safety. This inverts the typical pattern where color only appears when enabled, ensuring dangerous states are never invisible.

💡 Recommendations

For Critical Settings (Auto Renew, Transfer Lock)

  • Red OFF Warning — Shows danger when disabled, safety when enabled
  • Use the large size variant for emphasis
  • Consider pulsing animation for domains expiring soon

For Safe Default States

  • Ghost / Whisper — Ultra subtle, doesn't draw attention
  • Muted / Soft — Visible but calm
  • Avoid prominent green when ON is the expected state

For Standard Settings

  • Subtle or Muted — Balanced visibility
  • Use consistent style across similar settings

General Guidelines

  • Reserve prominent/glowing styles for states needing action
  • Default states should be calm and quiet
  • Use text labels (ON/OFF) for critical toggles
